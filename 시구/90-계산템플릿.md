## 90-계산 템플릿 + 예제

> [!summary] 계산 파트는 여기만 반복해도 점수 효율이 큼
> - 서브넷/CIDR  
> - 지연(전송/전파/처리/큐잉)  
> - Shannon/SNR(dB)  
> - 해밍코드(r 계산/오류 위치)  
> - CRC(검출 흐름)

### 90.1 서브넷/CIDR “4단계 템플릿”
1) `/n`이면 **호스트 비트 = 32-n**  
2) **호스트 수 = 2^(호스트비트) - 2**(네트워크/브로드캐스트 제외)  
3) 서브넷 블록 크기 = 해당 옥텟에서 `256 - 마스크값`  
4) 네트워크/브로드캐스트/유효호스트 범위 산출

```text
예제1) 192.168.10.0/27
호스트비트=5 → 호스트수=2^5-2=30
블록크기=32 → 네트워크: .0 .32 .64 .96 ...
첫 서브넷: 192.168.10.0 ~ .31 (브로드캐스트 .31)
유효호스트: .1 ~ .30
```

```text
예제2) “서브넷당 5호스트가 필요” (최소 호스트 수 조건)
필요 호스트 ≥ 5 → (2^h - 2) ≥ 5
h=3이면 2^3-2=6 OK → 호스트비트=3 → 프리픽스=/29
```

> [!warning] 서브넷 함정
> - /31, /32 같은 특수는 일반 규칙과 다르지만(옵션), 시험에서 보통 /24~ /30 중심.
> - “최대 서브넷 수” vs “서브넷당 호스트 수” 문제를 먼저 구분.

### 90.2 지연 템플릿
- 전송지연: `L/R` (L=비트 길이, R=bps)  
- 전파지연: `d/s` (d=거리, s=전파속도 ≈ 2×10^8 m/s(광섬유 근사))  
- 처리지연/큐잉지연: 문제 조건대로 합산  
- end-to-end: 각 링크/노드 합산

```text
예제) 1,500Byte 패킷, 10Mbps 링크 1개
L=1500*8=12,000bit
전송지연=12,000 / 10,000,000 = 0.0012s = 1.2ms
```

> [!warning] 지연 함정
> - Mbps(10^6) vs Mi(2^20) 섞지 말기(대부분 Mbps로 줌)
> - Byte→bit 변환(×8) 누락이 1등 실수
> - RTT(왕복)인지 편도인지 확인

### 90.3 Shannon + SNR(dB)
- `C = B * log2(1 + S/N)`
- `SNR(dB)=10*log10(S/N)` → `S/N = 10^(SNR(dB)/10)`

```text
예제) B=3kHz, SNR=30dB
S/N = 10^(30/10)=10^3=1000
C = 3000 * log2(1+1000) ≈ 3000 * log2(1001)
log2(1024)=10이므로 log2(1001)≈9.97
C≈ 3000*9.97 ≈ 29.9 kbps
```

### 90.4 해밍코드 템플릿(단일 오류 정정)
1) 데이터 비트 m이 주어지면 `2^r ≥ m + r + 1`로 r 결정  
2) 전체 길이 n = m + r  
3) 패리티 위치: 1,2,4,8,... (2의 거듭제곱)  
4) 각 패리티는 “자기 비트가 커버하는 위치” XOR(또는 짝/홀)로 계산  
5) 수신 시 신드롬(패리티 체크 결과)로 오류 위치 결정

```text
예제) m=4라면 r?
r=3이면 2^3=8 ≥ 4+3+1=8 OK → r=3, n=7
패리티 위치: 1,2,4 / 데이터는 3,5,6,7에 채움
```

> [!warning] 해밍 함정
> - r만 묻는 문제 / “오류 위치 찾기” 문제 구분
> - 짝/홀 패리티 조건을 문제에서 반드시 확인

### 90.5 CRC(개념 템플릿)
- 데이터 비트열을 다항식으로 보고 생성다항식 G(x)로 나눠 **나머지(R)**를 붙여 전송
- 수신 측에서 같은 G(x)로 나눠 나머지가 0이면(이상적) “오류 없음”으로 판단

> [!warning] CRC 함정
> - “정정”이 아니라 “검출”이 핵심
> - 생성다항식 비트표현(최고차항 포함) 실수
