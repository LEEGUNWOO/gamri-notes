# ⭐ 정보보호개론 

## 📘정보보호 기본

### 📌정보보호의 개념

정보자산을 공개/노출, 변조/수정, 지체/재난 등의 위협으로부터 보호하여 정보의 기밀성, 무결성, 가용성을 확보 정보의 수집, 가공, 저장, 전송 중에 정보의 훼손, 변조, 유출 등을 방지하기 위한 관리적, 기술적 수단

### **📌 정보보호의 목적**

#### **✔️ 정보보호 주요 목적**

| 구분      | 설명                                                                            |
| ------- | ----------------------------------------------------------------------------- |
| **기밀성** | **Confidentiality**. 비인가된 개인·단체·프로세스 등으로부터 중요한 정보를 보호하는 것, 즉 노출되지 않는다는 확신을 제공 |
|         | 위협: 스니핑, 네트워크 모니터링, 엿보기                                                       |
|         | 대응책: 암호화, 접근통제                                                                |
| **무결성** | **Integrity**. 정보가 의도적 또는 비의도적으로 변조되지 않음을 보장하는 특성                             |
|         | 위협: 트로이목마, 바이러스, 해킹, 파일 변조                                                    |
|         | 대응책: 인증, 디지털 서명, 접근통제                                                         |
| **가용성** | **Availability**. 인가를 받은 사용자가 정보나 서비스를 시기 적절하게 접근/사용할 수 있는 것                  |
|         | 위협: DOS 공격, 통신방해, 자연재해                                                        |
|         | 대응책: 이중화, Fault Tolerant 시스템, 백업/DRS                                          |

#### ✔️ 정보보호 기타 목적

| 구분        | 설명                                                           |
| --------- | ------------------------------------------------------------ |
| **인증성**   | **Authenticity**. 통신중인 상대편이 자신이 믿고 있는 그 사람이 맞는지를 **확인시키는 것** |
|           | 수신한 메시지의 송신자가 자신이 믿고 있는 그 사람이 맞는지를 **확인시키는 것** (메시지 출처 인증)   |
|           | 구현: ID/PW, 인증서                                               |
| **부인방지**  | **Non-Repudiation.** 작성한 메시지를 작성하지 않았다고 부인할 수 없게 하는 특성       |
|           | 구현: 전자서명                                                     |
| **책임추적성** | **Accountability**. 주체의 행동, 활동을 기록하여 차후 추적 가능케 하는 특성         |
|           | 구현: 식별, 인증, 권한부여, 접근통제, 감사                                   |

**CIA 트리아드 암기 포인트**

기밀성·무결성·가용성 = **CIA** / 기타 목 적 = **인부책** (인증성·부인방지·책임추적성)
