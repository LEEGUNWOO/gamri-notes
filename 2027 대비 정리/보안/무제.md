|---|---|---|

|**스니핑**|•데이터 가로채기|•네트워크 패킷 스니핑, Wi-Fi 스니핑|
|**스트리핑**|•암호화 제거 (HTTPS → HTTP)|•SSL Stripping|

|목적(보안요소)|계층|공격 유형|대표 기법|핵심 포인트|
|---|---|---|---|---|
|**가용성**|네트워크|서비스 거부 공격 (DoS/DDoS)|SYN Flood, Smurf, DNS Amplification|자원 고갈·증폭|
|**기밀성**|네트워크|스니핑 (Sniffing)|Packet Sniffing|평문 탈취|

|**인증**|응용|무차별 대입 (Brute Force)|Password Guessing|계정 탈취|

|**정찰**|사전단계|풋프린팅 (Footprinting)|Whois, Port Scan|정보 수집|
|**무선**|네트워크|이블 트윈 (Evil Twin)|AP 위장|Wi-Fi 탈취|

## 📘 DoS / DDoS 공격 기법 한 장 정리

|계열|공격 기법|원리|핵심 키워드|
|---|---|---|---|
|**TCP**|SYN Flood (SYN 플러딩)|•3-way handshake 미완성|•Half-open|
|**ICMP**|Smurf (스머프 공격)|•ICMP Broadcast 증폭|•Amplification|
|**UDP**|Fraggle (프래글 공격)|•UDP Broadcast 증폭|•Smurf의 UDP 버전|
|**IP**|Land (랜드 공격)|•출발지=목적지 IP|•Loop 유발|
|**IP**|Teardrop (티어드롭)|•Fragment 겹침|•Overlapping|
|**IP**|Bonk (봉크)|•시퀀스번호 같은 값으로 반복|•재조립 오류|
|**IP**|Boink (보잉크)|•시퀀스번호 중간에 비정상으로 섞어보냄<br>•Bonk 변형|•변형형|
|**ICMP**|Ping of Death|•비정상 큰 패킷 전송|•구형 취약점|
|**응용**|Slowloris|•연결 유지로 자원 고갈|•동시 연결 고갈|
|**응용**|HTTP Flood|•정상 요청 위장|•탐지 어려움|
|**증폭**|DNS Amplification|•작은 요청→큰 응답|•Reflection|
|**증폭**|NTP Amplification|•monlist 악용|•대규모 증폭|

## 📘 블루투스(Bluetooth) 공격 기법 총정리

|공격 기법|공격 유형|핵심 목표|특징 / 시험 키워드|
|---|---|---|---|
|**블루프린팅 (BluePrinting)**|정찰|장치 정보 수집|•MAC, 서비스, 버전 탐색|
|**블루재킹 (Bluejacking)**|메시지 전송|사용자 장난|•vCard 전송, 비교적 무해|
|**블루스나핑 (BlueSnarfing)**|정보 탈취|저장 데이터 접근|•연락처·파일 탈취|
|**블루버깅 (BlueBugging)**|기기 제어|원격 제어권 획득|•통화·문자 발신 가능|
|**블루스매킹 (Bluesmack)**|서비스 거부 (DoS)|시스템 마비|•L2CAP Ping Flood|
|**카 위스퍼러 (Car Whisperer)**|도청|오디오 탈취|•기본 PIN 취약|
|**블루본 (BlueBorne)**|원격 코드 실행 (RCE)|시스템 장악|•페어링 없이 감염|
|**KNOB 공격**|암호 약화|키 길이 축소|•암호 강도 다운그레이드|
|**BIAS 공격**|인증 우회|기기 사칭|•재연결 인증 취약|

## 📘 AI 공격 기법 전체 정리

|단계|공격 기법|핵심 목표|방어 방법|시험 키워드|
|---|---|---|---|---|
|**학습 단계**|데이터 오염 공격 (Poisoning Attack)|•모델 성능 저하|•데이터 검증<br>•이상 탐지 후 제거<br>•강건한 학습 기법 사용|•학습 데이터 변조|
|**학습 단계**|백도어 공격 (Backdoor Attack)|•특정 조건 오작동|•트리거 탐지 및 제거<br>•강건한 모델 훈련|•특정 조건에서만 오동작|
|**추론 단계**|회피 공격 (Evasion Attack)|•탐지 우회<br>•잘못 분류하도록 유도|•적대적 훈련<br>•입력 검증<br>•검출 기법 사용|•적대적 예제<br>•소음 왜곡|
|**추론 단계**|모델 역공격 (Model Inversion Attack)|•학습 데이터 복원|•출력 제한<br>•모델 구조 변경<br>•데이터 익명화|•결과 분석 → 학습 데이터 추출|
|**추론 단계**|모델 추출 공격 (Model Extraction Attack)|•모델 복제|•API 요청 횟수 제한<br>•모델 응답 제한<br>•API 인증 강화|•모델 구조 추정|
|**추론 단계**|멤버십 추론 공격 (Membership Inference Attack)|•개인정보 노출|•학습 데이터 공개 금지<br>•예측 결과 비공개<br>•민감 정보 포함 금지|•학습 포함 여부 판단|
