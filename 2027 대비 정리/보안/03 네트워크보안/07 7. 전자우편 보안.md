## 📘 7. 전자우편 보안

### 📌 PEM / PGP 비교

| 구분      | PEM                                                                             | PGP                                                                            |
| ------- | ------------------------------------------------------------------------------- | ------------------------------------------------------------------------------ |
| 개발자     | •IETF                                                                           | •Phil Zimmermann                                                               |
| 키 인증 방식 | •중앙 집중된 키 인증                                                                    | **•분산화된 키 인증 (Web of Trust)**                                                  |
| 특징      | •인터넷 표준(안)<br>•익명의 메시지 불허용<br>•구현의 어려움, 높은 보안성 (군사·금융계 등)<br>•이론 중심, 많이 사용되지 않음 | •응용 프로그램<br>•익명의 메시지 허용<br>•구현의 용이성<br>•PEM에 비해 낮은 보안성<br>•실세계 사용 중심, 현재 많이 사용 |

### 📌 PGP 세부

| 항목                | 내용                                                                                                                                                        |
| ----------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 공개키 알고리즘          | •RSA, Elgamal, DSS, ECC, DH 등                                                                                                                             |
| 비밀키 알고리즘          | •IDEA, 3DES, CAST-128, Blowfish 등                                                                                                                         |
| 해시 알고리즘           | •MD5, SHA-1, MD2, RIPE-MD/160 등                                                                                                                           |
| **신뢰망(Web of Trust)** | **•중앙인증 시스템 없이 분산 인증 수행<br>•"내가 신뢰하는 사용자가 신뢰하는 상대방은 나도 신뢰한다"**                                                                                                |
| **KeyRing**           | **•자신의 개인키와 다른 사용자의 공개키를 관리하기 위해 사용되는 저장공간<br>•Public/Private 2개의 열쇠고리 존재<br>•Public key ring: 다른 사용자의 공개키·서명정보를 관리<br>•Private key ring: 자신의 개인키를 암호화하여 보관** |

### 📌 PGP 암호화 절차 (전자봉투쓰는 하이브리드 암호화 체계)

|단계|내용|
|---|---|
|①|•원본 메시지 압축|
|②|•세션키(대칭키) 생성<br>•세션키로 압축된 메시지 대칭키 암호화|
|③|•수신자 공개키로 세션키 암호화|
|④|•암호화된 세션키 + 압축·암호화 메시지 결합|
|⑤|•바이너리를 텍스트로 변환 후 송신|

### 📌 S/MIME

| 항목        | 내용                                                                                                                                                                      |
| --------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| MIME 개념   | •SMTP는 ASCII 문자로된 RFC822 형식의 메시지만 전송 가능<br>•MIME은 한국어 같은 2바이트 다중언어, 실행파일, 그림파일 같은 이진파일을 SMTP로 전송될 수 있도록 ASCII 코드로 변환하는 코드변환 방식                                          |
| S/MIME 개념 | •MIME 데이터를 안전하게 송수신하는 방법을 제공<br>•인터넷의 MIME 표준에 의거하여 전자 메시지에 인증, 메시지 무결성, 송신처의 부인방지(전자서명 이용), 프라이버시와 데이터 보안(암호 이용)과 같은 암호학적 보안 서비스 제공<br>•PGP와 달리 사용되는 공개키는 X.509 인증서 사용 |

### 📌 S/MIME 보안서비스·알고리즘

|보안서비스|보안 메커니즘|암호 알고리즘|
|---|---|---|
|메시지 기밀성|•암호화|•메시지암호화: 3DES, RC2/40<br>•키분배: Diffie-Helman, RSA|
|메시지 무결성|•해쉬함수|•SHA-1, MD5|
|사용자 인증|•인증서|•X.509 v3 인증서|
|송신 부인방지|•전자서명|•DSS, RSA|

### 📌 S/MIME 제공 서비스

|제공 서비스|쉬운 설명|
|---|---|
|Enveloped Data|내용을 **암호화**한 것. 수신자 공개키로 봉투 씌운 것 (전자봉투)|
|Signed Data|내용 + 서명 **둘 다 base64 인코딩** → S/MIME 없으면 **깨져 보임**|
|Clear-signed Data|내용은 **평문 그대로**, 서명만 base64 → S/MIME 없어도 **내용은 읽힘**|
|Signed and Enveloped Data|**암호화 + 서명 동시에** 적용|
※ base64: SMTP가 7bit ASCII만 지원하므로 바이너리 데이터를 ASCII 텍스트로 변환하는 부호화 (암호화 아님)
