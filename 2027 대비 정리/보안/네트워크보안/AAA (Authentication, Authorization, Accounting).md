## 📘  AAA (Authentication, Authorization, Accounting)

### 📌 개념 및 기능

| 구분             | 한글   | 내용                                                                         |
| -------------- | ---- | -------------------------------------------------------------------------- |
| **개념**         |      | •네트워크 환경에서 가입자에 대한 안전하고 신뢰성 있는 인증·권한검증·과금 기능을 체계적으로 제공하는 정보 보호 기술          |
| Authentication | 인증   | •인증 요구자 당사자의 증명 (id/password/생체인식/Smart Card 등) **(※ 네가 누구인지 확인 (신원 확인))** |
| Authorization  | 인가   | •사용자 요구작업의 허용여부 결정 **(※ 네가 뭘 할 수 있는지 확인 (권한 부여))**                         |
| Accounting     | 계정관리 | •사용자 사용내역의 Audit **(※ 네가 뭘 했는지 기록 (로그/감사))**                               |
### 📌 주요 인증 프로토콜

| 프로토콜     | 설명                                                                                                                |
| -------- | ----------------------------------------------------------------------------------------------------------------- |
| RADIUS   | •UDP 기반, C/S(단방향), <br>•가장 널리 쓰임 (오래된 표준)<br>•보안/Fail Over 기능 약함 (장애 시 자동전환 약함, 암호화 미흡)<br>•과금 시 패킷 손실 가능(UDP 특성) |
| TACACS+  | •TCP 기반, CISCO, C/S<br>•인증을 위한 사용자 Profile(사용자 이름, 암호) 확인<br>•보안강화, CISCO NW기기 인증에 주로 사용                          |
| Diameter | •TCP/SCTP 기반, P2P(양방향)<br>•보안강화(IPSEC/TLS), 뛰어난 확장성<br>•Mobile IP(Roaming, 도메인간 이동성), Fail Over, 메시지 처리기능 향상      |
