## 📘 6. SSO / EAM / IAM

### 📌 개념 비교

| 구분  | 정의                                                                                                                | 비고       |
| --- | ----------------------------------------------------------------------------------------------------------------- | -------- |
| SSO | •다수의 시스템에 단일 Entity를 이용한 자동인증을 가능하게 하는 아키텍처                                                                       | 인증만      |
| EAM | •SSO에 사용자의 인증을 관리하고<br>•애플리케이션이나 데이터에 대한 사용자 접근을 결정하는 기업 내 정책을 구현하는 단일화된 메커니즘을 제공하는 솔루션<br>•SSO + 접근제어(메뉴/기능별 권한) | 인증+인가    |
| IAM | •SSO과 권한 관리 및 계정관리 기능을 제공<br>•EAM의 확장, EAM + Provisioning(계정 생성/수정/삭제 **자동화** *EAM은 수동관리)                         | 인증+인가+계정 |

### 📌 참고: OAuth

| 항목  | 내용                                                                                                                                                                                                              |
| --- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 정의  | •OpenAPI 환경에서 이기종 App 혹은 사이트 간 인증과 권한 확인을 위해 Token 발급 메커니즘                                                                                                                                                      |
| 예시  | •Twitter, Facebook 계정으로 다른 게임사이트 접속                                                                                                                                                                             |

**※ Access Token: 유효기간 짧음 (수십분~수시간). Resource Server 접근에 직접 사용. 만료 시 사용자 인지 없이 자동 갱신.**  
**※ Refresh Token: 유효기간 김 (수일~수개월). Authorization Server에서만 사용. Access Token 재발급 전용. 만료 시 사용자 재로그인 필요.**
