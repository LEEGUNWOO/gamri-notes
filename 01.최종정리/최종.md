# 📘 Git 주요 명령어 정리표

| 생성       | 원격 저장소 복제          | `git clone [url]`                   |

| 확인       | 커밋 이력 보기           | `git log`                           |

**내용이론 (동기가 생기는 이론)**

| 이론      | 학자                   | 핵심 내용                                                                       |
| ------- | -------------------- | --------------------------------------------------------------------------- |
| 욕구단계설   | 매슬로우 (Maslow)        | 생리→안전→사회→존경→자아실현 5단계. 하위 욕구 충족 후 상위로 이동. 단계 건너뜀 불가                          |
| ERG이론   | 앨더퍼 (Alderfer)       | 존재(Existence)·관계(Relatedness)·성장(Growth) 3단계. 매슬로우와 달리 **좌절-퇴행 가능**         |
| 동기-위생이론 | 허즈버그 (Herzberg)      | 동기요인(성취·인정·성장)은 만족 유발. 위생요인(급여·환경·정책)은 불만족 예방만. **만족↔불만족이 별개의 축**           |
| X·Y이론   | 맥그리거 (McGregor)      | X이론: 인간은 게으르고 강제해야 함. Y이론: 인간은 자율적이고 책임감 있음. **맥그리거는 Y이론 지지**               |
| 성취동기이론  | 맥클리랜드 (McClelland)   | 성취(nAch)·권력(nPow)·친교(nAff) 3욕구. 욕구는 **학습·경험으로 형성**. 탁월한 관리자 = nPow↑ nAff↓   |
| 자기결정이론  | 다니엘 핑크 (Daniel Pink) | 자율성(Autonomy)·숙련(Mastery)·목적(Purpose) 3요소. 외재적 보상보다 **내재적 동기**가 창의적 업무에 효과적 |

**RSA 암호화:**

```
암호문 = 평문^e mod n
```

**복호화:**

```
평문 = 암호문^d mod n
```

**키 구성:**

|키|구성|공개 여부|
|---|---|---|
|공개키|**(e, n)**|공개|
|개인키|**(d, n)**|비밀|

**n = p × q** (두 큰 소수의 곱)

디피헬만이랑 비슷하게 **"큰 수 오른쪽"** 으로 기억하면 돼!

```
평문^e mod n
```

e로 올리고 n으로 나누는 거야 👍

**배깅 Bagging(Bootstrap Aggregating)** 

```
■ 정의 = 데이터 랜덤 샘플링 후 독립적으로 학습 → 평균/다수결

■ 구조 = 복원추출 → 여러 모델 병렬 학습 → 평균/투표

■ 특징 = 병렬 처리 / 분산(Variance) 감소 / 과적합 방지

■ 대표 = Random Forest

■ 함정 = 순차 아님
```

**부스팅 Boosting**

```
■ 정의 = 이전 모델 오답을 보완하며 순차적으로 학습

■ 구조 = 모델1 → 틀린 거 가중치↑ → 모델2 → 반복

■ 특징 = 순차 처리 / 편향(Bias) 감소 / 정확도 높음

■ 대표 = XGBoost, AdaBoost

■ 함정 = 병렬 아님 / 과적합 위험
```

**스태킹 Stacking**

```
■ 정의 = 여러 모델 결과를 또 다른 모델이 학습해서 최종 결과

■ 특징 = 가장 복잡함

■ 함정 = Bagging/Boosting 아님
```

**Random Forest:**

Bagging + 의사결정트리 조합

```
■ 정의 = 배깅 + 결정트리 + 랜덤 feature 선택

■ 구조 = 복원추출 → 트리마다 변수 랜덤 선택 → 수백개 트리 → 다수결/평균

■ 특징 = 과적합 감소 / 변수 중요도 파악 가능 / 안정적 성능

■ 함정 = Boosting 아님 (Bagging 계열)
```

랜덤 포레스트 특징:

- 과적합에 강함
- 변수 중요도 파악 가능
- 단일 트리보다 정확도 높음
- 해석은 어려움 (블랙박스)

## 2025년 80번 — 최대 TPS 계산

**정답: ④ 72.727**
### 풀이

**각 업무의 사용률(Utilization) 계산**

- A업무 사용률 = 10/40 = **0.25**
- B업무 사용률 = 10/400 = **0.025**

**전체 사용률**

0.25 + 0.025 = **0.275**

**최대 TPS**

전체 요청 = 10 + 10 = 20개/초

최대 TPS = 20 / 0.275 = **72.727**

### 공식

> **최대 TPS = 총 요청수 / 총 사용률**

총 사용률 = Σ(각 업무 요청수 / 각 업무 최대TPS)

# 시간으로 구하기

| 항목                | 값                        |
| ----------------- | ------------------------ |
| BAC               | 100만원                    |
| 사업 기간             | 40주                      |
| ==**AT (실제경과)**== | ==**20주**==              |
| 완료율               | 40%                      |
| AC                | 60만원                     |
| EV                | 100 × 40% = **40만원**     |
| PV                | 100 × (20/40) = **50만원** |

**풀이**

PV rate = 100만원 ÷ 40주 = **2.5만원/주**

ES = EV ÷ PV rate = 40 ÷ 2.5 = **16주** → "40만원어치 일은 원래 16주면 끝났어야 했다"

SPI(ES) = ES ÷ AT = 16 ÷ 20 = **0.8** → 20주 걸려서 할 일을 16주치밖에 못했으니 일정 80%밖에 못따라감

 ES=16주, SPI=0.8

**핵심 공식**

- ES = EV / PV rate
- SPI(ES) = ES / AT (1보다 작으면 일정 지연)

**SCTP (Stream Control Transmission Protocol)**

|구분|내용|
|---|---|
|정의|TCP + UDP 장점 결합한 전송계층 프로토콜|
|연결 방식|4-way 핸드셰이크 (TCP 3-way보다 보안 강화)|
|특징|멀티호밍, 멀티스트리밍 지원|

**4-way 핸드셰이크**

| 순서  | 방향    | 메시지         | 역할                 |
| --- | ----- | ----------- | ------------------ |
| 1   | 클라→서버 | INIT        | 연결 요청              |
| 2   | 클라←서버 | INIT ACK    | 응답 + **쿠키 전송**     |
| 3   | 클라→서버 | COOKIE ECHO | **쿠키 돌려줌** (신원 확인) |
| 4   | 클라←서버 | COOKIE ACK  | 연결 완료              |

**TCP vs SCTP 비교**

| 구분              | TCP   | SCTP                  |
| --------------- | ----- | --------------------- |
| 핸드셰이크           | 3-way | **4-way**             |
| 멀티호밍            | 미지원   | **지원** (여러 IP 동시 사용)  |
| 멀티스트리밍          | 미지원   | **지원** (여러 스트림 동시 전송) |
| HOL 블로킹         | 발생    | 스트림별 독립으로 감소          |
| SYN Flooding 방어 | 취약    | 쿠키 메커니즘으로 방어          |

**쿠키 메커니즘이 핵심**: TCP는 **SYN Flooding** 공격에 취약한데 SCTP는 COOKIE ECHO/ACK 단계로 방어함. **디도스도 방어함**

| **준지도학습 (Semi-Supervised Learning)**  | **소량의 라벨 데이터 + 대량의 비라벨 데이터**를 함께 사용              | Pseudo-Labeling, Consistency Regularization, Mean Teacher | 라벨 부족한 이미지/텍스트 분류            |

# 보안 솔루션

| 솔루션  | 핵심 한 줄                                                                       | 키워드                    |
| ---- | ---------------------------------------------------------------------------- | ---------------------- |
| ESM  | IDS·방화벽·VPN 등 각종 보안제품 **통합 관리(SW) 통합관제**                                     | 지능형 종합 보안관리            |
| UTM  | 방화벽·IPS·VPN을 **하나의 장비**에 **통합(HW)**                                          | 단일장비 통합, 성능이슈 주의       |
| NAC  | 단말기 보안정책 준수 여부·인증 검사 후 **네트워크 접속 통제**                                        | 단말 접근제어                |
| MDM  | 모바일 기기 보호·관리·감시, BYOD 환경 보안                                                  | 모바일 관리                 |
| IPS  | 침입 **탐지 + 능동적 차단**                                                           | IDS(탐지만) vs IPS(탐지+차단) |
| FDS  | 거래 패턴 학습으로 **이상거래 탐지**                                                       | 금융 사기탐지, 빅데이터          |
| SIEM | **ESM** + 통합로그 분석, 네트워크·보안장비 정보 수집·분석(**이상징후 자동 탐지 AI/머신러닝 기반 > SOAR 진화 중**) | 보안정보·이벤트 통합관리          |
| DLP  | 단말기에서 데이터 **외부 유출 감시·차단**                                                    | 데이터 유출방지               |

**시험 포인트**

- ESM vs SIEM 혼동 주의 → ESM은 보안제품 **통합관리**, SIEM은 **로그·이벤트 분석** 중심
- IDS vs IPS → IDS는 탐지만, IPS는 **능동 차단**까지
- NAC vs MDM → NAC은 **네트워크 접근**, MDM은 **모바일 기기** 관리
- UTM 단점 → **단일장비라 성능 이슈** 발생 가능

**SOAR (Security Orchestration, Automation and Response)**

보안 운영을 **자동화**하는 플랫폼이야.

**핵심 3요소:**

|요소|설명|
|---|---|
|**Orchestration**|여러 보안 도구 통합·연동|
|**Automation**|반복적인 보안 업무 자동화|
|**Response**|위협 탐지 시 자동 대응|

**SIEM이랑 차이:**

- SIEM → 로그 수집·분석·탐지
- SOAR → 탐지 후 **자동 대응**까지

SIEM이 탐지하면 SOAR가 자동으로 대응하는 구조로 같이 쓰는 경우 많아.

## IP 클래스 정리

| 클래스   | 범위          | 용도           |
| ----- | ----------- | ------------ |
| A     | 0~127       | 대형 네트워크      |
| B     | 128~191     | 중형 네트워크      |
| C     | 192~223     | 소형 네트워크      |
| D     | 224~239     | 멀티캐스트        |
| **E** | **240~255** | **실험/미래 예약** |
0 128 192 224 240

```
특수 목적 주소들:

0.0.0.0      → 모든 주소 (기본 라우트)
127.0.0.1    → 루프백 (localhost)
255.255.255.255 → 브로드캐스트
```
