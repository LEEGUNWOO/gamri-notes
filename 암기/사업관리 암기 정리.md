# 📘 EVM 진척 측정 방법 정리

## 1️⃣ 고정비율법 (Fixed Formula Method)

```text
작업 시작 시 일정 비율 인정
작업 완료 시 나머지 비율 인정
중간 진행률은 반영하지 않음
```

### 📌 대표 유형

- 50-50 Rule
    
- 20-80 Rule
    
- 0-100 Rule (시작 0%, 완료 100%)
    

### 📌 특징

- 진척 측정이 단순
    
- 단기 작업에 적합
    
- 과대 진척 인식 가능성 있음
    

---

## 2️⃣ 시작-종료법 (0-100 Rule)

```text
시작 시 0%
완료 시 100%
```

- 가장 보수적
    
- 완료 전까지 EV = 0
    

---

## 3️⃣ 가중 마일스톤 (Weighted Milestone)

```text
중간 마일스톤마다 가중치 부여
각 단계 완료 시 진척 인정
```

- 대규모·복잡 작업에 적합
    
- 통제력 우수
    

---

## 4️⃣ 달성률 방식 (Percent Complete)

```text
실제 작업 진행률을 그대로 반영
```

- 주관 개입 가능성
    
- 관리 성숙도 필요
    

---

## 5️⃣ 노력비례법 (Level of Effort, LOE)

```text
시간 경과에 비례해 진척 인정
산출물 없는 지원업무에 사용
```

---

# 🎯 시험용 구분 키워드

```text
정해진 % → 고정비율
중간 단계 여러 개 → 가중 마일스톤
완료 시 100 → 0-100
실제 진행률 → 달성률
지원업무 → LOE
```

# 📘 프로젝트 생애주기 4유형 비교표 (반복 vs 점증 명확화)

| 구분        | 예측형 (Predictive) | 반복적 (Iterative) | 점증적 (Incremental) | 애자일 (Agile)   |
| --------- | ---------------- | --------------- | ----------------- | ------------- |
| 기본 개념     | 처음에 전체 계획 수립     | 같은 기능을 반복 개선    | 기능을 하나씩 추가        | 반복 + 점증 결합    |
| 무엇이 변하는가  | 거의 변하지 않음        | **품질·해결책이 변함**  | **기능 범위가 늘어남**    | 품질 + 기능 모두 유연 |
| 결과물 제공    | 마지막에 전체 제공       | 개선 중인 결과        | 단계별 완성 기능         | 짧은 주기마다 완성 기능 |
| 피드백 역할    | 제한적              | 핵심 요소           | 선택적               | 필수            |
| 핵심 문장     | 계획 중심            | 같은 걸 고친다        | 다른 걸 더한다          | 적응 + 협업       |
| 시험 구분 키워드 | 순차적, 전통적         | 수정, 정제, 개선      | 점진적 제공, 단계적 기능    | 스프린트, 협업      |

---

## 2️⃣ CPM 핵심 계산식

```text
EF = ES + Duration
LS = LF - Duration
Float = LS - ES = LF - EF
```

✔ Float(여유시간) = 0 → Critical Path

|약어|풀네임|의미|계산식|
|---|---|---|---|
|ES|Early Start|가장 빨리 시작할 수 있는 시간|이전 작업 EF 중 최대값|
|EF|Early Finish|가장 빨리 끝날 수 있는 시간|ES + 기간|
|LS|Late Start|늦어도 시작해야 하는 시간|LF - 기간|
|LF|Late Finish|늦어도 끝나야 하는 시간|다음 작업 LS 중 최소값|

---

## 3️⃣ PERT 핵심 계산식

| 공식 종류           | 계산식              | 특징                                                     |
| --------------- | ---------------- | ------------------------------------------------------ |
| **삼각분포**        | ((O + M + P)/3)  | 단순평균, 모든 값에 동일 가중치, 빠르고 간단함                            |
| **PERT (베타분포)** | ((O + 4M + P)/6) | M에 높은 가중치, 현실적 예측에 유리                                  |
| **표준편차** (PERT) | ((P - O)/6)      | 예측의 불확실성 정도 측정                                         |
| 분산 (Variance)   | ((P - O)/6)²     | 분산은 최소값(O)과 최대값(P)을 사용하여 작업 시간의 **변동성**을 측정하는 데 사용됩니다. |

✔ O 최적시간 M 가능성있는시간 P 비관적시간
✔ 베타분포 가정  
✔ 3점 추정
### 🔥 **정규분포에서 표준편차에 따른 확률 값**

| **시그마 (σ)**   | **범위** | **확률 값 (%)**  |
| ------------- | ------ | ------------- |
| **1시그마 (1σ)** | μ ± 1σ | **68.23%**    |
| **2시그마 (2σ)** | μ ± 2σ | **95.5%**     |
| **3시그마 (3σ)** | μ ± 3σ | **99.7%**     |
| **4시그마 (4σ)** | μ ± 4σ | **99.9937%**  |
| **5시그마 (5σ)** | μ ± 5σ | **99.99994%** |
|**6시그마 (6σ)**|μ ± 6σ|**99.9999998%**|

# 📘 EVM 초압축 이해 + 계산 정리

---

## 1️⃣ 기본 개념 (3개만 기억)

|항목|의미|한 줄 이해|
|---|---|---|
|PV|계획가치|원래 이만큼 해야 함|
|EV|획득가치|실제로 번 성과|
|AC|실제비용|실제로 쓴 돈|

---
## 2️⃣ EVM 전체 공식

| 항목  | 공식        | 기준  | 해석        |
| --- | --------- | --- | --------- |
| SV  | EV - PV   | 0   | +면 일정 빠름  |
| CV  | EV - AC   | 0   | +면 비용 절감  |
| SPI | EV / PV   | 1   | 1↑ 일정 양호  |
| CPI | EV / AC   | 1   | 1↑ 비용 양호  |
| EAC | BAC / CPI | -   | 완료 예상 총비용 |
| ETC | EAC - AC  | -   | 남은 필요 비용  |

---

# 💣 절대 안 틀리는 공식

```text
EV - PV = 일정
EV - AC = 비용
SPI/CPI는 1 기준
```

---

## 3️⃣ BAC (Budget At Completion) 정리

|항목|의미|한 줄 이해|
|---|---|---|
|BAC|총예산|프로젝트 전체 승인 예산|

---

## 4️⃣  BAC 관련 계산식

| 구분  | 공식          | 의미           |
| --- | ----------- | ------------ |
| EV  | BAC × 실제진척률 | 실제 성과 가치     |
| PV  | BAC × 계획진척률 | 계획상 가치       |
| EAC | BAC / CPI   | 완료 시 예상 총비용  |
| ETC | EAC - AC    | 앞으로 더 필요한 비용 |

# 🎯 BAC 기억 포인트

```text
BAC = 전체 예산
EV/PV는 BAC에서 계산
EAC는 BAC와 CPI로 계산
```

---

# 🔥 시험장에서 자주 나오는 유형

1️⃣ BAC와 %만 줌 → EV, PV 계산  
2️⃣ CPI 줌 → EAC 계산  
3️⃣ CV/SV 해석 문제

---

---
## 5️⃣값이 안 주어지면 직접 계산

## 1) EV 역산 (가장 중요)

|주어진 값|EV 공식|
|---|---|
|BAC & 실제%|**EV = BAC × 실제%**|
|PV & SPI|**EV = SPI × PV**|
|PV & SV|**EV = SV + PV**|
|AC & CPI|**EV = CPI × AC**|
|AC & CV|**EV = CV + AC**|

---

## 2) PV 역산

|주어진 값|PV 공식|
|---|---|
|BAC & 계획%|**PV = BAC × 계획%**|
|EV & SPI|**PV = EV / SPI**|
|EV & SV|**PV = EV - SV**|

---

## 3) AC 역산

|주어진 값|AC 공식|
|---|---|
|(직접 제시)|**AC = 실제 사용비용**|
|EV & CPI|**AC = EV / CPI**|
|EV & CV|**AC = EV - CV**|

# 📘 의사소통 채널 공식

##  1️⃣ 의사소통 채널 수 (Communication Channels)

#      🔥  n(n−1)/2

| 항목  | 의미                     |
| --- | ---------------------- |
| n   | 이해관계자 또는 팀원 수          |
| 공식  | nC2 (2명씩 조합)           |
| 적용  | 모든 사람이 서로 직접 소통 가능한 경우 |

예) 10명 이라면 10 * (10-1) /2

# 🎯  같은 공식!

| 상황            | 의미        | 공식       |
| ------------- | --------- | -------- |
| 프로젝트 의사소통     | 사람-사람 연결  | n(n-1)/2 |
| 완전 연결 그래프 p2p | 노드-노드 연결  | n(n-1)/2 |
| 대칭키 관리        | 사용자-사용자 키 | n(n-1)/2 |

# 📘 리더십 이론 비교표

| 구분       | 이론명                   | 핵심 개념                   | 리더 유형/특징                |
| -------- | --------------------- | ----------------------- | ----------------------- |

| 행동이론     | 오하이오/미시간 연구           | 리더 행동 유형 구분             | 과업 중심 / 관계 중심           |

| 상황이론     | 피들러(Fiedler)          | 상황 호의성에 따라 효과적 리더 유형 다름 | 과업형 / 관계형               |
| 경로-목표 이론 | 하우스(House)            | 목표 달성 경로를 제시            | 지시형 / 지원형 / 참여형 / 성취지향형 |

| 서번트 리더십  | Servant               | 구성원 봉사 중심               | 지원, 배려                  |
| 카리스마 리더십 | Charismatic           | 리더 개인 매력 중심             | 강한 영향력                  |

# 📘 상황적 리더십 정확 구조

성숙도 = 능력 + 의지

성숙도 낮음 → 능력도 없고 의지도 낮음  
👉 **지시형 (Telling)**  
= 구체적으로 “이렇게 해” 알려줘야 함

성숙도 조금 상승 → 능력 부족하지만 의지는 있음  
👉 **설득형 (Selling / Coaching)**  
= 가르치면서 왜 해야 하는지 설명

성숙도 더 상승 → 능력 있음, 자신감 부족  
👉 **참여형 (Participating / Supporting)**  
= 의사결정에 참여시켜 동기 강화

성숙도 높음 → 능력도 의지도 충분  
👉 **위임형 (Delegating)**  
= 맡기고 간섭 최소

✅ **지시형(Directing/Telling)** → **설득형(Coaching/Selling)** → **참여형(Supporting/Participating)** → **위임형(Delegating)**

# 📘 인도측정(Delivery Measurements)

> “얼마나 빠르고 안정적으로 가치를 전달하고 있는가?”를 보는 지표


| 구분    | 지표                      | 의미                  | 핵심 포인트      |
| ----- | ----------------------- | ------------------- | ----------- |
| 시간 기반 | **주기 시간 (Cycle Time)**  | 작업 시작 → 완료까지 걸린 시간  | 짧을수록 효율 ↑   |
| 시간 기반 | **리드 타임 (Lead Time)**   | 요청 접수 → 완료까지 전체 시간  | 고객 관점 시간    |
| 흐름 기반 | **대기열 크기 (Queue Size)** | 처리 대기 중인 작업 수       | 많으면 병목 가능성  |
| 흐름 기반 | **WIP (진행 중 작업)**       | 현재 수행 중인 작업 수       | 제한이 핵심      |
| 흐름 기반 | **배치 크기 (Batch Size)**  | 한 번에 처리하는 작업 묶음 수   | 작을수록 빠른 피드백 |
| 속도 기반 | **속도 (Velocity)**       | 스프린트당 완료한 스토리 포인트 합 | 예측 가능성 판단   |
| 시각화   | **번다운 차트**              | 남은 작업량 감소 추이        | 일정 예측       |
| 시각화   | **번업 차트**               | 완료된 작업량 증가 추이       | 범위 변화 확인    |
| 시각화   | **누적 흐름도(CFD)**         | 작업 상태별 흐름 시각화       | 병목 구간 확인    |

#  📘 리스크 관리 프로세스 순서 (PMBOK 흐름)

1. **리스크 관리 계획 수립** (Plan Risk Management)
    
2. **리스크 식별** (Identify Risks)
    
3. **정성적 리스크 분석** (Perform Qualitative Risk Analysis)
    
4. **정량적 리스크 분석** (Perform Quantitative Risk Analysis)
    
5. **리스크 대응 계획 수립** (Plan Risk Responses)
    
6. **리스크 대응 실행** (Implement Risk Responses)
    
7. **리스크 모니터링** (Monitor Risks)
    

# 📘 리스크 대응 전략 (위협 ↔ 기회 대응 구조)

|위협 (Threat)|↔|기회 (Opportunity)|의미|
|---|---|---|---|
|**회피 (Avoid)**|↔|**활용 (Exploit)**|반드시 없애거나 / 반드시 실현시키거나|
|**완화 (Mitigate)**|↔|**증대 (Enhance)**|영향 줄이거나 / 가능성 키우거나|
|**전가 (Transfer)**|↔|**공유 (Share)**|제3자에게 넘기거나 / 제3자와 나누거나|
|**수용 (Accept)**|↔|**수용 (Accept)**|그냥 받아들임|

---

# 🎯 이렇게 보면 이해가 쉽다

- 회피 ↔ 활용 → “강하게 개입”
    
- 완화 ↔ 증대 → “조절”
    
- 전가 ↔ 공유 → “제3자 활용”
    
- 수용 ↔ 수용 → “그냥 둠”

# 📘 연동 기획 (Rolling Wave Planning)

## 🔥 정의

> 단기 작업은 상세 계획,  
> 장기 작업은 개략 계획 후 **점진적으로 구체화**하는 기법

## 📌 적용 상황

- 프로젝트 초기에 모든 세부사항을 알 수 없음
    
- 불확실성이 높음
    
- 점진적 상세화 필요
    

## 📊 특징

|구분|내용|
|---|---|
|단기 계획|상세하게 수립|
|장기 계획|개략적으로 수립|
|시간 경과|점차 구체화|
|적용 영역|일정관리 (계획 수립)|

## 🎯 시험 포인트

- 일정 계획 기법
    
- “가까운 것은 상세, 먼 것은 거칠게”
    
- 애자일/적응형 환경과도 연결 가능

# 📘 MBO (목표에 의한 관리, Management by Objectives)

## 🔥 정의

> 상사와 부하가 함께 목표를 설정하고  
> 그 달성 여부로 성과를 평가하는 관리 방식


## 📌 핵심 특징

|항목|내용|
|---|---|
|목표 설정|구체적·측정 가능|
|참여|구성원이 목표 설정에 참여|
|기간 명시|달성 기한 명확|
|피드백|성과 평가 및 환류|

## 🎯 시험 포인트

- 성과 중심 관리
    
- 상하 합의 기반
    
- 자율 책임 강조
    
- **다면평가는 MBO 자체 개념 아님**
    

## 🧠 한 줄 암기

> 목표 합의 + 기간 명확 + 성과 평가 = MBO

# 🧠형상관리 4대 활동 (시험 핵심)

|구분|영어|핵심 의미|
|---|---|---|
|형상식별|Configuration Identification|형상항목(CI) 식별, 베이스라인 설정|
|형상통제|Configuration Control|변경 요청 평가·승인·통제|
|형상상태기록|Configuration Status Accounting|변경 이력 기록 및 보고|
|형상검증|Configuration Verification / Audit|제품이 명세와 일치하는지 확인|

이건 실제로 네가 질문했던 문제와도 같다.

예:

- 베이스라인 설정 → **식별**
    
- 변경 승인 → **통제**
    
- 변경 이력 기록 → **상태기록**
    
- 명세와 일치 여부 확인 → **검증**
