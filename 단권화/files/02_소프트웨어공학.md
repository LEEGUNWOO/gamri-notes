# 📘 카나리 배포 (Canary Deployment)

|항목|내용|
|---|---|
|정의|일부 사용자에게만 신규 버전 먼저 배포 후 점진 확대|
|목적|장애 영향 최소화 + 실제 사용자 검증|
|흐름|소수 배포 → 모니터링 → 점진 확대 → 문제 시 롤백|
|핵심|트래픽 분할 + 실시간 모니터링|
|장점|안정성 높음, 빠른 검증|
|단점|구현/운영 복잡|

# 📘 RUP (Rational Unified Process)

|항목|내용|
|---|---|
|정의|객체지향 기반의 반복·점진적 소프트웨어 개발 프로세스|
|특징|Iterative(반복) + Incremental(점진) + Use Case 중심 + Architecture 중심|
|목적|위험을 초기에 제거하고 점진적으로 시스템 완성|

📌 4단계 구조

|단계|영문|핵심|
|---|---|---|
|착수|Inception|범위/비즈니스 타당성 정의|
|정교화|Elaboration|아키텍처 확립 + 핵심 위험 제거|
|구축|Construction|기능 구현|
|전이|Transition|배포/운영|

# 📘 응집도와 결합도

좋다. **둘 다 “높은 → 낮은” 기준으로 통일해서 준다.**

📌 응집도 (Cohesion) — 높을수록 좋음

|순위 (높→낮)|종류|영문|의미|
|---|---|---|---|
|1|기능적|Functional|하나의 기능만 수행|
|2|순차적|Sequential|출력 → 다음 입력|
|3|통신적|Communicational|같은 데이터 사용|
|4|절차적|Procedural|순서만 관련|
|5|시간적|Temporal|같은 시점 수행|
|6|논리적|Logical|조건 분기 묶음|
|7|우연적|Coincidental|관련 없음|

📌 한줄  
기순통절시논우

📌 결합도 (Coupling) — 낮을수록 좋음 (그래서 높은 → 나쁜)

|순위 (높→낮)|종류|영문|의미|
|---|---|---|---|
|1|내용 결합|Content|내부 구현 공유|
|2|공통 결합|Common|전역 변수 공유|
|3|외부 결합|External|외부 환경 공유|
|4|제어 결합|Control|제어값 전달|
|5|스탬프 결합|Stamp|구조체 일부 전달|
|6|자료 결합|Data|데이터만 전달|

📌 한줄  
내공외제스자
