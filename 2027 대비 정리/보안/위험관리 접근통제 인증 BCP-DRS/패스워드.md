## 📘 패스워드

### 📌 패스워드 개념

널리 사용되는 인증 방식이나, 안전하지 않은 인증 방식이다.

### 📌 패스워드 정책

•문자 조합, 대문자, 소문자, 숫자, 특수문자, **사용자**는 쉽게 기억할 수 있고 **해커**는 추측 어려움 •평문 저장보다는 해쉬값으로 저장해야 함(Shadow PW File) •시스템은 실패한 로그인 시도 횟수를 제한해야 함

### 📌 패스워드 인증방식의 취약성

•추측이 쉬움(잘 알려진 단어를 사용) •유출이 쉬움(사회공학적 방법, 엿보기) •모니터링이 쉬움(주로 패스워드는 평문으로 전달됨)

### 📌 패스워드 공격기법

•무차별 공격(Brute Force Attack), 사전공격(Dictionary Attack), Birth Day Attack, 트로이목마, 패스워드 파일에 대한 직접 접근, 사회공학적 방법(헬프 데스크로 가장 등)

### 📌 패스워드 공격의 대응

•패스워드 발생기 사용, 로그인 실패 횟수 제한, 주기적 패스워드 변경, 사전공격/무차별 공격 탐지를 위한 IDS 사용, 보안인식교육, OTP 등의 향상된 패스워드 방법 사용

### 📌 Password Encryption(Shadow Password File)

•패스워드 자체를 시스템에 저장하는 것이 아니라, 패스워드의 ==MD(Message Digest, 해시값)== 값을 저장 •시스템의 관리자 권한을 획득해도 시스템 사용자들의 패스워드를 알아낼 수 없음
