# 📘 암호 구조
# 🔥 1️⃣ 암호 공격 전체 구조

암호 공격은 크게 3가지 축으로 나뉜다.

```
① 암호해석 공격 (Cryptanalysis)
② 공격 모델 분류 (Attacker Model)
③ 부채널 공격 (Side Channel Attack)
```

---

# 2️⃣ 암호해석 공격 (Cryptanalysis)

👉 알고리즘의 수학적 구조를 공격

|한글|영어|핵심|
|---|---|---|
|무차별 대입 공격|Brute Force Attack|키 전수조사|
|기지 평문 공격|Known Plaintext Attack|평문 일부 알고 있음|
|선택 평문 공격|Chosen Plaintext Attack (CPA)|원하는 평문 암호화 가능|
|선택 암호문 공격|Chosen Ciphertext Attack (CCA)|원하는 암호문 복호화 가능|
|차분 공격|Differential Cryptanalysis|입력 차이 분석|
|선형 공격|Linear Cryptanalysis|선형 근사|

📌 포인트  
CPA / CCA는 **부채널이 아니라 암호 모델 공격**이다.

---

# 3️⃣ 공격 모델 (Attacker Capability Model)

이건 “공격자가 무엇을 할 수 있느냐” 기준.

|한글|영어|
|---|---|
|암호문 단독 공격|Ciphertext-Only Attack|
|기지 평문 공격|Known Plaintext Attack|
|선택 평문 공격|Chosen Plaintext Attack|
|선택 암호문 공격|Chosen Ciphertext Attack|

시험에서 자주 섞어서 낸다.

---

# 4️⃣ 부채널 공격 (Side Channel Attack)

👉 수학이 아니라 **물리적 특성**을 이용

| 한글        | 영어                              | 분석 대상        |
| --------- | ------------------------------- | ------------ |
| 전력 분석 공격  | Power Analysis Attack           | 소비 전력        |
| 타이밍 공격    | Timing Attack                   | 연산 시간 비밀키 추론 |
| 전자기 분석 공격 | Electromagnetic Analysis Attack | 전자파          |
| 캐시 공격     | Cache Attack                    | 캐시 접근 패턴     |
| 오류 주입 공격  | Fault Injection Attack          | 고의 오류 유도     |

📌 특징

- 암호 수학 안 깨도 됨
    
- 하드웨어 보안에서 중요
    
- 스마트카드, HSM 문제에서 자주 나옴
    


## 📘 서비스 ↔ 보안기술 한 장 정리

| 서비스(기본 프로토콜)          | 평문 여부    | 보안 기술 (한글 / 영어)                                                            | 적용 계층   | 무엇을 보호하는가             | 시험 포인트              |
| --------------------- | -------- | -------------------------------------------------------------------------- | ------- | --------------------- | ------------------- |
| 웹 (HTTP)              | 평문       | **HTTPS (HTTP Secure)** = **TLS (전송계층보안, Transport Layer Security) , SSL** | 전송계층    | 기밀성, 무결성, 서버 인증       | 웹·전자결제는 TLS 기반      |
| 원격 접속 (Telnet)        | 평문       | **SSH (보안 셸, Secure Shell)**                                               | 응용계층    | 암호화 통신, 사용자 인증        | Telnet 대체는 SSH      |
| 전자우편 (SMTP/POP3/IMAP) | 평문       | **PGP (프리티 굿 프라이버시, Pretty Good Privacy)** / **S/MIME (보안 다목적 인터넷 메일 확장)** | 응용계층    | 메일 내용 암호화, 전자서명(부인방지) | 메일 보안 = PGP/S-MIME  |
| 파일 전송 (FTP)           | 평문       | **SFTP (SSH 파일전송)** / **FTPS (SSL/TLS 기반 FTP)**                            | 응용/전송계층 | 파일 전송 시 기밀성·무결성       | SFTP≠FTPS 구분        |
| 전자결제 (웹 기반)           | 평문(HTTP) | ** SET,** **TLS 기반 HTTPS**                                                 | 전송계층    | 카드정보 등 민감정보 보호        | 결제는 별도 프로토콜 아님, TLS |

# 📘 VPN (Virtual Private Network, 가상 사설망) 정리

## 1️⃣ VPN 개념

|항목|내용|
|---|---|
|한글|가상 사설망|
|영어|Virtual Private Network|
|목적|공용망(인터넷) 위에 사설망처럼 안전한 통신 구성|
|제공 요소|기밀성(Confidentiality), 무결성(Integrity), 인증(Authentication)|

---

## 2️⃣ VPN 동작 원리

```text
터널링(Tunneling) + 암호화(Encryption) + 인증(Authentication)
```

- 터널링: 패킷을 다른 패킷 안에 넣어 전달
    
- 암호화: 데이터 보호
    
- 인증: 사용자/장비 검증
    

---

## 3️⃣ VPN 종류

|구분|한글|영어|특징|
|---|---|---|---|
|사이트 간 VPN|Site-to-Site VPN|LAN ↔ LAN 연결|기업 지사 연결|
|원격 접속 VPN|Remote Access VPN|사용자 ↔ 회사망|재택근무|

## 4️⃣ VPN 주요 프로토콜

| OSI 계층    | 프로토콜 (한글 / 영어)                                       | 역할       | 암호화 여부     | 시험 포인트          |
| --------- | ---------------------------------------------------- | -------- | ---------- | --------------- |
| 7~5 응용~세션 | **SSL VPN (SSL Virtual Private Network)**            | 웹 기반 VPN | ⭕ (TLS 사용) | 브라우저 기반         |
| 4 전송      | **TLS (Transport Layer Security, 전송계층보안)**           | 암호화 채널   | ⭕          | HTTPS 기반        |
| 3 네트워크    | **IPSec (Internet Protocol Security, 인터넷 프로토콜 보안)**  | IP 패킷 보호 | ⭕          | Site-to-Site 대표 |
| 3 네트워크    | **IKE (Internet Key Exchange, 인터넷 키 교환)**            | 키 협상     | ⭕          | IPSec 구성요소      |
| 2 데이터링크   | **L2TP (Layer 2 Tunneling Protocol, 제2계층 터널링 프로토콜)** | 터널링      | ❌          | 암호화 없음          |
| 2 데이터링크   | **PPTP (Point-to-Point Tunneling Protocol)**         | 터널링      | 약함         | 구식·취약           |
| 2 데이터링크   | **PPP (Point-to-Point Protocol)**                    | 인증·링크 설정 | ❌          | L2TP 기반         |

# 🔐 5️⃣ IPSec (Internet Protocol Security) 구조

IPSec은 시험 단골이다.

### 구성 요소

| 구성  | 한글                                           | 역할            |
| --- | -------------------------------------------- | ------------- |
| AH  | 인증 헤더 (Authentication Header)                | 인증(무결성)       |
| ESP | 캡슐화 보안 페이로드 (Encapsulating Security Payload) | 인증(무결성)+ 암호화  |
| IKE | 키 교환                                         | 세션키 생성        |

---

### 동작 모드

|모드|설명|
|---|---|
|전송 모드 (Transport Mode)|원래 IP 헤더 유지, 데이터 부분만 보호|
|터널 모드 (Tunnel Mode)|원래 IP 패킷 전체를 새 패킷 안에 캡슐화|

📌 Site-to-Site는 보통 **터널 모드**

# 🔒 6️⃣ SSL VPN (TLS VPN)

|항목|설명|
|---|---|
|기반|TLS (Transport Layer Security)|
|접근 방식|웹 브라우저 기반|
|장점|방화벽 통과 쉬움|
|단점|네트워크 전체 연결에는 제한적|

|구분|IPSec VPN|SSL VPN|
|---|---|---|

|설치|장비 중심|클라이언트/웹|
|기업 간 연결|적합|제한적|
|원격 사용자|가능|매우 적합|

| 정의    | 추가 정보 없이는 특정 개인을 알아볼 수 없도록 처리한 개인정보 |
| 법적 근거 | 「개인정보 보호법」 제2조                      |

👉 완전 익명정보(Anonymized Data)와 다름  
👉 여전히 **개인정보**에 해당

---

## 2️⃣ 가명처리(Pseudonymization) 정의

|구분|내용|
|---|---|
|한글|가명처리|
|영어|Pseudonymization|
|의미|식별자를 대체·삭제하여 식별 가능성을 낮추는 처리|

예:

- 이름 → 임의 코드
    
- 주민번호 → 일부 마스킹
    
- 전화번호 → 해시값

|허용 목적|
|---|
|통계 (Statistics)|

# 📘 보안 도구 통합 정리표 (목적 기반 분류)

### 🔥 **각 도구 설명**

#### 1️⃣ **리버스 엔지니어링** (Reverse Engineering)

- **목적**: **실행 파일**을 분석하여 **악성 코드**나 **취약점**을 찾아내는 데 사용
    
- **대표 도구**: **OllyDBG, IDA, Ghidra**
    
- **시험 키워드**: **디버깅**, **디스어셈블**  
    -> 악성 코드를 분석하거나, **바이너리 분석**을 통해 취약점을 파악할 때 쓰인다.
    

#### 2️⃣ **패킷 분석기** (Packet Analyzer)

- **목적**: **네트워크 트래픽**을 **모니터링**하고 **분석**하는 도구
    
- **대표 도구**: **Wireshark, tcpdump**
    
- **시험 키워드**: **패킷 캡처**, **트래픽**  
    -> **네트워크 패킷**을 **캡처**하고 분석하여 **보안 위협**을 파악한다.
    

#### 3️⃣ **취약점 스캐너** (Vulnerability Scanner)

- **목적**: **시스템**의 **취약점**을 **자동으로 검사**하여 **위험 요소**를 식별
    
- **대표 도구**: **OpenVAS, Nessus**
    
- **시험 키워드**: **취약점 자동 스캔**  
    -> **네트워크**나 **시스템**에서 **자동으로 취약점을** 찾아내는 도구.
    

#### 4️⃣ **침입 탐지 시스템** (Intrusion Detection System)

- **목적**: **침입**이나 **비정상적인 활동**을 **탐지**하는 시스템
    
- **대표 도구**: **Snort, Suricata**
    
- **시험 키워드**: **시그니처 탐지**  
    -> **네트워크**에서 발생하는 **침입**을 **탐지**하여 경고를 발생시킨다.
    

#### 5️⃣ **침입 방지 시스템** (Intrusion Prevention System)

- **목적**: **침입을 실시간으로 차단**하는 시스템
    
- **대표 도구**: **Snort IPS Mode**
    
- **시험 키워드**: **실시간 차단**  
    -> **침입 탐지**와 **차단**을 동시에 처리하여, **네트워크 보안**을 실시간으로 강화한다.
    

#### 6️⃣ **웹 취약점 점검** (Web Vulnerability Scanner)

- **목적**: **웹 애플리케이션**의 **취약점**을 **검사**하여 **보안 리스크**를 식별
    
- **대표 도구**: **Burp Suite, Paros, OWASP ZAP**
    
- **시험 키워드**: **XSS**, **SQLi 테스트**  
    -> **웹 취약점**(예: **SQL Injection**, **Cross-Site Scripting**)을 점검하는 도구.
    

#### 7️⃣ **무선 침투 테스트** (Wireless Pentest)

- **목적**: **Wi-Fi 네트워크**의 보안을 **테스트**하고 **취약점**을 찾는 데 사용
    
- **대표 도구**: **Aircrack-ng**
    
- **시험 키워드**: **WPA 크랙**  
    -> **Wi-Fi 패스워드**를 **크랙**하거나, 무선 네트워크의 보안을 테스트한다.
    

#### 8️⃣ **포트 스캐너** (Port Scanner)

- **목적**: **서버**나 **네트워크**에서 **열린 포트**를 **스캔**하여 **서비스**를 확인
    
- **대표 도구**: **Nmap**
    
- **시험 키워드**: **포트 스캔**  
    -> **네트워크 포트**의 **상태**를 **스캔**하여 **열린 포트**를 찾고 보안을 강화한다.
    

#### 9️⃣ **로그 분석** (Log Analysis)

- **목적**: **로그 데이터를 분석**하여 **보안 위협**을 탐지하고 **위협 패턴**을 파악
    
- **대표 도구**: **ELK Stack**
    
- **시험 키워드**: **SIEM 연계**  
    -> **로그 분석**을 통해 **보안 위협**을 탐지하고, **경고**를 발생시킨다.
    

#### 🔟 **보안 정보 이벤트 관리** (SIEM)

- **목적**: **보안 이벤트**를 **통합적으로 분석**하여 **보안 사고를 실시간으로 파악**
    
- **대표 도구**: **Splunk**
    
- **시험 키워드**: **이벤트 상관분석**  
    -> **보안 정보**를 **통합적으로 관리**하고, **이벤트 상관분석**을 통해 **보안 위협**을 탐지한다.

# 📘 리눅스 권한
# 🔥 1️⃣ 기본 구조

```
[u][g][o]
```

- u = 소유자
    
- g = 그룹
    
- o = 기타 사용자
    

각 자리 = 숫자 하나

---

# 🔥 2️⃣ 숫자 뜻

```
r = 4
w = 2
x = 1
```

더해서 계산.

예:

```
7 = rwx
6 = rw-
5 = r-x
4 = r--
0 = ---
```

---

# 🔥 3️⃣ 예시

```
chmod 640 file
```

→ 6 = rw-  
→ 4 = r--  
→ 0 = ---

👉 소유자: 읽기+쓰기  
👉 그룹: 읽기  
👉 기타: 없음

---

# 🔥 4️⃣ 특수권한 (앞자리 하나 더)

```
4xxx → SUID
2xxx → SGID
1xxx → Sticky
```

예:

```
chmod 4755 file
```

→ 4 = SUID  
→ 755 = 일반권한

# 🔥 한줄 요약

```
권한 = [특수][u][g][o]
r=4 w=2 x=1
```

# 📘  Zero Trust (제로 트러스트)

> **어떤 사용자·기기·네트워크도 기본적으로 신뢰하지 않고 모든 접근을 지속적으로 검증하는 보안 모델**

📚 출처

- Forrester Research – John Kindervag
    
- NIST SP 800-207 (Zero Trust Architecture)
    

---

## 🧠 핵심 철학

```text
Never Trust, Always Verify
절대 신뢰하지 말고 항상 검증하라
```

---

## ⚔️ 기존 보안 vs Zero Trust

|구분|기존 보안 (Perimeter Security)|Zero Trust|
|---|---|---|
|신뢰 기준|내부망 신뢰|아무도 신뢰하지 않음|
|인증 방식|최초 인증|지속적 검증|
|접근 통제|네트워크 중심|사용자·기기 중심|
|보안 철학|외부 위험|내부도 위험|

---

## 📜 Zero Trust 핵심 원칙

|원칙|설명|
|---|---|
|🔎 Verify Explicitly|사용자·기기·위치 등을 계속 검증|
|🔒 Least Privilege|최소 권한만 부여|
|🚨 Assume Breach|이미 침해되었다고 가정|

(NIST SP 800-207)

---

## 🧾 검증 요소

접속 요청 시 다음 요소들을 평가한다.

|요소|설명|
|---|---|
|👤 사용자|누구인가|
|💻 기기|신뢰 가능한 장치인가|
|🌍 위치|접속 위치|
|⏰ 시간|접속 시간|
|📊 행위|이상 행동 여부|

---

## ⚙️ 구현 기술

|기술|역할|
|---|---|
|🔑 MFA|다중 인증|
|🪪 IAM|계정 및 접근 관리|
|🌐 ZTNA|Zero Trust Network Access|
|🧩 Micro-segmentation|네트워크 세분화|
|🛡 Endpoint Security|단말 보안|

---

## 🎯 한 줄 정리

> **Zero Trust = 내부망도 절대 신뢰하지 않는다**

보안 솔루션은 보통 **네트워크 / 데이터 / 인증 / 단말 / 관제**로 묶어서 출제된다. (정보보호 개론 및 NIST 보안 아키텍처 분류에서도 유사하게 계층적으로 구분됨)

#  📘 🛡 보안 솔루션 핵심 정리 

## 🌐 네트워크 보안

|솔루션|설명|
|---|---|
|🔥 **Firewall**|IP, 포트, 프로토콜 기준으로 **네트워크 트래픽을 허용/차단하는 기본 보안 장비**|
|🚨 **IDS (Intrusion Detection System)**|공격 패턴을 **탐지만 하고 차단하지는 않는 침입 탐지 시스템**|
|⚔ **IPS (Intrusion Prevention System)**|공격을 **탐지하고 실시간으로 차단까지 수행하는 침입 방지 시스템**|
|🌐 **WAF (Web Application Firewall)**|**웹 애플리케이션 공격(SQL Injection, XSS 등)**을 탐지·차단|
|🔐 **VPN (Virtual Private Network)**|**암호화 터널을 통해 인터넷 구간에서 안전한 통신을 제공**|
|🧩 **NAC (Network Access Control)**|**단말 인증과 보안 상태 검사 후 내부 네트워크 접속을 통제**|
|🌊 **Anti-DDoS**|대량 트래픽 공격을 **분산 처리하거나 필터링하여 서비스 가용성을 보호**|

---

## 📂 데이터 보안

|솔루션|설명|
|---|---|
|📤 **DLP (Data Loss Prevention)**|**중요 데이터가 이메일, USB, 클라우드 등을 통해 외부로 유출되는 것을 탐지·차단**|
|🔒 **DRM (Digital Rights Management)**|**문서의 열람, 복사, 인쇄 등의 사용 권한을 제어하여 콘텐츠 사용을 관리**|
|🗄 **DB 접근제어**|**데이터베이스 접근을 사용자·권한 기준으로 통제하고 로그를 관리**|
|🔑 **DB 암호화**|**데이터베이스 내 중요 데이터를 암호화하여 저장 데이터 보호**|

---

## 👤 인증 / 접근 관리

|솔루션|설명|
|---|---|
|🔐 **SSO (Single Sign-On)**|**한 번의 인증으로 여러 시스템에 접근할 수 있는 통합 인증 체계**|
|🪪 **IAM (Identity & Access Management)**|**사용자 계정, 권한, 인증 정책을 통합적으로 관리하는 시스템**|
|🔑 **MFA (Multi-Factor Authentication)**|**지식·소유·생체 등의 여러 인증 요소를 결합한 다중 인증 방식**|

# 1️⃣ Buffer Overflow (버퍼 오버플로우)

### 취약 코드

```c
char buf[10];
gets(buf);
```

문제

- 입력 길이 제한 없음 → 메모리 덮어쓰기
    

### 대응

```c
fgets(buf, sizeof(buf), stdin);
```

문제

- 사용자 입력이 포맷 문자열이 됨
    

### 대응

```c
printf("%s", user_input);
```

# 📘  블록 암호 운용 모드 (Block Cipher Mode)

블록 암호는 보통 **AES 같은 암호 알고리즘을 어떻게 운용할지**를 결정하는 방식이다.

예

```text
AES + 운용모드
```

---

# 주요 운용 모드

## 1️⃣ ECB (Electronic Codebook, 전자 코드북)

**특징**

- 같은 평문 → 같은 암호문
    
- 구조 노출
    

**특징 요약**

```text
보안 취약
거의 사용 안함
```

---

## 2️⃣ CBC (Cipher Block Chaining, 암호 블록 연쇄)

**특징**

- 이전 암호문과 XOR 후 암호화
    
- 패턴 숨김 가능
    

**단점**

```text
병렬 처리 불가
```

---

## 3️⃣ CFB (Cipher Feedback, 암호 피드백)

**특징**

- 블록 암호를 **스트림 암호처럼 사용**
    

---

## 4️⃣ OFB (Output Feedback, 출력 피드백)

**특징**

- 키 스트림 생성 후 XOR
    
- 오류 전파 없음
    

---

## 5️⃣ CTR (Counter Mode, 카운터 모드)

**특징**

- 카운터 값 암호화 후 XOR
    
- 병렬 처리 가능
    
- 고속
    

---

## 6️⃣ GCM (Galois Counter Mode)

**특징**

- CTR 기반
    
- **기밀성 + 무결성 제공**
    

---

## 7️⃣ CCM (Counter with CBC-MAC)

**특징**

- CTR + CBC-MAC 결합
    
- **기밀성 + 인증 제공**
    

---

# 🎯 시험 핵심 구분
|모드|한글명|동작 방식|특징|시험 포인트|
|---|---|---|---|---|
|**ECB**|전자 코드북|각 블록을 독립적으로 암호화|동일 평문 → 동일 암호문|패턴 노출 / 보안 취약|
|**CBC**|암호 블록 연쇄|이전 암호문과 XOR 후 암호화|패턴 숨김|병렬 처리 불가|
|**CFB**|암호 피드백|이전 암호문을 입력으로 사용|블록 암호를 스트림처럼 사용|오류 전파 발생|
|**OFB**|출력 피드백|키 스트림 생성 후 XOR|스트림 암호처럼 동작|오류 전파 없음|
|**CTR**|카운터 모드|카운터 값을 암호화 후 XOR|병렬 처리 가능|고속 암호화|
|**GCM**|갈루아 카운터 모드|CTR + GHASH 인증|기밀성 + 무결성|AEAD 모드|
|**CCM**|카운터 + CBC-MAC|CTR + CBC-MAC|기밀성 + 인증|무선 네트워크에서 사용|

# 🔎 주요 특징 비교

|특징|ECB|CBC|CFB|OFB|CTR|
|---|---|---|---|---|---|
|패턴 노출|있음|없음|없음|없음|없음|
|병렬 처리|가능|불가|불가|가능|가능|
|오류 전파|없음|있음|있음|없음|없음|


# 🔐 블록 암호 운용 모드 시험 암기표

## ① 기본 블록 모드

|모드|한글|특징|시험 포인트|
|---|---|---|---|
|**ECB**|전자 코드북|블록 독립 암호화|**패턴 노출 (가장 취약)**|
|**CBC**|암호 블록 연쇄|이전 암호문과 XOR|**가장 대표적 모드**|

핵심 기억

```text
ECB → 단순 / 패턴 노출
CBC → 체인 구조
```

---

## ② 스트림 방식 모드

|모드|한글|특징|시험 포인트|
|---|---|---|---|
|**CFB**|암호 피드백|이전 암호문 사용|스트림처럼 동작|
|**OFB**|출력 피드백|키 스트림 생성|오류 전파 없음|
|**CTR**|카운터 모드|카운터 암호화|**병렬 처리 가능 / 빠름**|

핵심 기억

```text
CFB / OFB / CTR
→ 스트림처럼 동작
```

특히

```text
CTR → 병렬 처리 가능
```

---

## ③ 인증 포함 모드 (AEAD)

|모드|특징|시험 포인트|
|---|---|---|
|**GCM**|CTR + 인증|**기밀성 + 무결성**|
|**CCM**|CTR + CBC-MAC|**기밀성 + 인증**|

핵심 기억

```text
GCM / CCM
→ 인증까지 제공
```

|질문|정답|
|---|---|
|패턴 노출 모드|ECB|
|병렬 처리 가능|CTR|
|기밀성 + 무결성|GCM|
|기밀성 + 인증|CCM|


# 🔐 주요 암호 알고리즘 정리

## 1️⃣ AES (Advanced Encryption Standard, 고급 암호화 표준)

**유형**

```text
대칭키 암호
```

**특징**

- 블록 암호
    
- 키 길이: **128 / 192 / 256 bit**
    
- 현재 가장 널리 사용
    

예

- HTTPS
    
- VPN
    
- Wi-Fi
    

---

## 2️⃣ RSA (Rivest–Shamir–Adleman)

**유형**

```text
비대칭키 암호
```

**특징**

- 공개키 / 개인키 사용
    
- 소인수분해 기반
    

사용

- 키 교환
    
- 전자서명
    

---

## 3️⃣ ECC (Elliptic Curve Cryptography, 타원곡선 암호)

**유형**

```text
비대칭키 암호
```

**특징**

- 타원곡선 이산대수 문제 기반
    
- **짧은 키로 높은 보안**
    

예

```text
ECC 256bit ≈ RSA 3072bit
```

---

## 4️⃣ Diffie-Hellman (디피-헬만 키 교환)

**유형**

```text
키 교환 알고리즘
```

**특징**

- 공개 채널에서 **공유 키 생성**
    

사용

- TLS
    
- VPN
    

---

## 5️⃣ Hash Function (해시 함수)

대표 알고리즘

- SHA-256
    
- SHA-3
    

**특징**

```text
단방향 함수
```

사용

- 패스워드 저장
    
- 데이터 무결성
    

---

## 6️⃣ MAC (Message Authentication Code)

**목적**

```text
메시지 무결성 + 인증
```

특징

- 대칭키 기반
    

---

## 7️⃣ HMAC (Hash-based MAC)

**특징**

```text
Hash + Secret Key
```

사용

- API 인증
    
- TLS

```text
RSA / ECC → 공개키
DH → 키 교환
Hash → 무결성
MAC → 인증
```


# 📘  🌐 VLAN (Virtual LAN, 가상 LAN)

**정의**

```text
물리적으로 하나의 네트워크를
논리적으로 여러 개의 네트워크로 분리하는 기술
```

즉

```text
스위치 하나 → 여러 개의 LAN처럼 사용
```

---

# 🎯 VLAN이 필요한 이유

기존 네트워크

```text
스위치 1개
→ 전체가 하나의 브로드캐스트 도메인
```

문제

- 브로드캐스트 트래픽 증가
    
- 보안 문제
    
- 네트워크 관리 어려움
    

---

# VLAN 적용

```
스위치
 ├ VLAN10 (개발팀)
 ├ VLAN20 (회계팀)
 └ VLAN30 (서버)
```

👉 **같은 스위치라도 서로 통신 불가**

(라우터 또는 L3 스위치 필요)

```text
VLAN 하나 = 하나의 Broadcast Domain
```

# 🛡 IDS / IPS / Snort / iptables 정리

## 1️⃣ IDS (Intrusion Detection System, 침입 탐지 시스템)

|항목|내용|
|---|---|
|역할|공격 탐지|
|특징|탐지만 하고 차단은 하지 않음|

```text
IDS = 탐지만
```

---

## 2️⃣ IPS (Intrusion Prevention System, 침입 방지 시스템)

|항목|내용|
|---|---|
|역할|공격 탐지 + 차단|
|특징|실시간 트래픽 차단|
|위치|네트워크 경로 중간|
|예|Snort (IPS 모드)|

핵심

```text
IPS = 탐지 + 차단
```

---

## 3️⃣ Snort

**정의**

```text
오픈소스 네트워크 침입 탐지 시스템
```

기능

|기능|설명|
|---|---|
|패킷 분석|네트워크 트래픽 분석|
|공격 탐지|시그니처 기반 탐지|
|룰 기반|탐지 규칙 사용|

---

### Snort 룰 구조

예

```text
alert tcp any any -> 192.168.0.1 21
```

구성

| 요소                  | 의미   |
| ------------------- | ---- |
| alert               | 경고   |
| tcp                 | 프로토콜 |
| !192.168.0.1 21 any | 출발지  |
| →                   | 방향   |
| 192.168.0.1 21      | 목적지  |
any = 모든 것(anything)

---

### Snort 주요 액션

|명령|의미|
|---|---|
|alert|경고|
|log|로그 기록|
|pass|통과|
|drop|차단|

---

## 4️⃣ iptables

**정의**

```text
리눅스 방화벽 설정 도구
```

기능

|기능|설명|
|---|---|
|패킷 필터링|트래픽 허용 / 차단|
|NAT 설정|주소 변환|
|포트 제어|포트 접근 제한|

# iptables 한 줄 구조

```bash
iptables -A INPUT -p tcp --dport 22 -j ACCEPT
```

이걸 **4개만 보면 된다.**

|부분|의미|
|---|---|
|-A|규칙 추가|
|INPUT|들어오는 패킷|
|--dport 22|목적지 포트 22|
|-j ACCEPT|허용|
# 📦 주요 옵션

|옵션|의미|
|---|---|
|-A|규칙 추가|
|-D|규칙 삭제|
|-L|규칙 목록 조회|
|-F|규칙 초기화|

---

# 🔎 주요 조건

|옵션|의미|
|---|---|
|-p|프로토콜|
|-s|출발지 IP|
|-d|목적지 IP|
|--sport|출발지 포트|
|--dport|목적지 포트|

---

# ⚡ 주요 동작 (Target)

|옵션|의미|
|---|---|
|ACCEPT|허용|
|DROP|차단|
|REJECT|차단 + 응답|

명령 예

iptables -A INPUT -p tcp -s 203.0.113.10 --dport 22 -j ACCEPT

읽는 순서

체인 → 출발지 → 목적지 포트 → 동작

---

# 한 줄 해석

```text
22번 포트로 들어오는 TCP 접속을 허용
```

즉

```text
SSH 접속 허용
```

# 시험용 최소 암기

iptables는 **3개 체인만 기억**

|체인|의미|
|---|---|
|INPUT|들어오는 패킷|
|OUTPUT|나가는 패킷|
|FORWARD|다른 네트워크로 전달|

---

# 진짜 시험용 요약

```text
iptables
-A → 규칙 추가
INPUT → 들어오는 패킷
--dport → 목적지 포트
-j → 동작
```

---

# 딱 이 정도만 알면 된다

```text
iptables = 리눅스 방화벽
INPUT = 들어오는 패킷
-j ACCEPT = 허용
-j DROP = 차단
```

|기술|역할|
|---|---|
|IDS|공격 탐지|
|IPS|공격 탐지 + 차단|
|Snort|IDS / IPS 구현|
|iptables|방화벽|

# 🧠 시험 기억 구조

```text

```

# 🔐 SSO (Single Sign-On)

## 📌 정의

```text
한 번의 인증(Login)으로 여러 시스템에 접근할 수 있는 인증 방식
```

예

```text
회사 포털 로그인
→ 메일
→ 그룹웨어
→ ERP
→ 파일 서버

다시 로그인 필요 없음
```

---

# 📊 핵심 구조

|구성요소|역할|
|---|---|
|User|사용자|
|SSO Server|인증 담당 서버|
|Application|서비스 시스템|

---

### ① 로그인

```text
사용자 → SSO 서버
ID / Password 인증
```

### ② 토큰 발급

```text
SSO 서버
→ 인증 토큰 생성
```

---

### ③ 서비스 접근

```text
사용자 → 다른 시스템 접근

```

# 🔐 메시지 인증 방식 비교

|구분|한글명|키 종류|제공 기능|
|---|---|---|---|
|MAC|메시지 인증 코드|대칭키|무결성 + 송신자 인증|
|HMAC|해시 기반 메시지 인증 코드|대칭키|무결성 + 송신자 인증|
|Digital Signature|전자서명|비대칭키|무결성 + 인증 + 부인방지|

---

# 1️⃣ MAC (Message Authentication Code)

### 구조

```text
MAC = Message + Secret Key
```

송신자

```
MAC = f(message, secret key)
```

수신자

```
MAC 다시 계산 → 비교
```

---

### 특징

|항목|설명|
|---|---|
|키|대칭키|
|무결성|가능|
|송신자 인증|가능|
|부인방지|불가능|

---

### 왜 부인방지가 안됨?

```text
송신자와 수신자가
같은 키를 공유
```

그래서

```text
누가 메시지를 만들었는지
증명 불가능
```

---

# 2️⃣ HMAC

### 정의

```text
해시 함수를 사용하는 MAC
```

공식

```text
HMAC = H((K ⊕ opad) || H((K ⊕ ipad) || message))
```

(RFC 2104)

---

### 특징

|항목|설명|
|---|---|
|기반|해시 함수|
|키|대칭키|
|무결성|제공|
|인증|제공|
|부인방지|없음|

---

# 3️⃣ Digital Signature

### 구조

송신자

```
signature = Sign(private key, message)
```

수신자

```
Verify(public key)
```

---

### 특징

|항목|설명|
|---|---|
|키|비대칭키|
|무결성|가능|
|인증|가능|
|부인방지|가능|

---

# 🔑 핵심 차이

|기능|MAC/HMAC|전자서명|
|---|---|---|
|무결성|O|O|
|인증|O|O|
|부인방지|X|O|

---

# 📌 시험 핵심 암기

```text
대칭키 인증
= MAC / HMAC
= 부인방지 없음

비대칭키 인증
= Digital Signature
= 부인방지 가능
```

# 🔐 인증 방식(Authentication Factors)

|인증 요소|영어|설명|예|
|---|---|---|---|
|**지식 기반 인증**|Knowledge Factor|사용자가 **알고 있는 정보**|비밀번호, PIN|
|**소유 기반 인증**|Possession Factor|사용자가 **가지고 있는 것**|OTP 토큰, 스마트카드, 보안카드|
|**생체 기반 인증**|Inherence Factor|사용자의 **신체 특징**|지문, 홍채, 얼굴|
|**행위 기반 인증**|Behavioral Factor|사용자의 **행동 패턴**|타이핑 패턴, 마우스 패턴|

#  🔐  IDS

먼저 이것을 고정해야 한다.

Positive = 공격으로 판단  
Negative = 정상으로 판단

그리고

True = 판단이 맞음  
False = 판단이 틀림
# 📊 IDS 판정표

|실제 상황|IDS 판단|결과|
|---|---|---|
|공격|공격|**True Positive (TP)**|
|정상|정상|**True Negative (TN)**|
|정상|공격|**False Positive (FP)**|
|공격|정상|**False Negative (FN)**|

# 🎯 시험에서 자주 나오는 두 개

|유형|의미|
|---|---|
|**False Positive**|정상인데 공격으로 오탐|
|**False Negative**|공격인데 정상으로 판단|

# 🎯 시험용 한 줄 암기

False Positive = 정상인데 공격이라고 오탐  
False Negative = 공격인데 정상이라고 놓침

# 🔥 보안 시험에서 중요 포인트

|유형|위험|
|---|---|
|False Positive|가용성 문제|
|False Negative|보안 문제|

# 🔐 블록체인 주요 공격 유형

| 공격          | 영어                | 공격 대상    | 원리                 | 특징            |
| ----------- | ----------------- | -------- | ------------------ | ------------- |
| **이중지불 공격** | Double Spending   | 블록체인 거래  | 같은 코인을 두 번 사용      | 블록 확정 전 거래 취소 |

| **허위정보전파**  | Eclipse Attack    | 노드 네트워크  | 특정 노드의 연결을 공격자가 장악 | 잘못된 블록 정보 전달  |
| **재진입 공격**  | Reentrancy Attack | 스마트 컨트랙트 | 함수 실행 중 재호출        | 반복 출금 가능      |
