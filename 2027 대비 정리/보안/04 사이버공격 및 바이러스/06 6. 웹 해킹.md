## 📘 6. 웹 해킹

### 📌 개념 및 대표 기법

|기법|설명|대응|
|---|---|---|
|**XSS (Cross Site Scripting)**|•게시판·웹 메일 등에 삽입된 악의적 스크립트에 의해 페이지가 깨지거나 쿠키·개인정보를 특정 사이트로 전송 (입력값 검증 부재의 일종)|•웹서버에서 사용자 입력 철저히 필터링<br>•특수 문자를 HTML 문자로 변경<br>•게시판에서 HTML형태의 입력 차단<br>•수상한 메일 URL 직접 클릭 금지, 브라우저 패치 실시|
|**SQL Injection**|•잘못 만들어진 웹어플리케이션에서 입력받은 데이터를 SQL문에 붙여서 처리할 때 악의의 SQL문을 변경하여 사용자 인증 우회 또는 DB 정보 취득|•사용자 입력 필터링<br>•SQL 에러메시지 사용자에게 미노출<br>•웹어플리케이션이 사용하는 DB유저의 권한 제한|
|**Buffer Overflow**|•버퍼의 한계를 검검하지 않는 오류를 이용하여 악의적인 코드를 입력하여 프로그램 동작 변경 또는 다운<br>•해킹된 프로그램이 관리자 권한으로 실행 시 관리자 권한 획득 가능|•웹어플리케이션의 한계점검 코드 추가<br>•실행권한을 제한된 권한 유저로 제한<br>•버퍼오버플로우 취약점 있는 함수·코드 사용 금지|
|**파일 업로드**|•웹서버에서 사용하는 동일한 프로그램언어로 된 파일을 업로드하고 실행할 수 있도록 하는 취약점 이용 (일명 Web Shell)<br>•악성코드를 업로드하거나 서버 파일에 악성코드 추가<br>•웹서버 내 모든 컨텐츠 삭제 및 관리자 권한 취득 가능|•게시판 업로드 문서의 확장자 제한<br>•파일업로드 디렉토리에 있는 문서는 실행불가 설정|
|**불필요하게 노출된 파일 이용**|•관리자가 부주의로 프로그램파일이나 정보파일을 백업이나 임시작업 용도로 웹서버에 존재하게 하였을 때 해커가 이를 이용|•웹서버의 불필요한 파일 제거<br>•중요파일의 위치와 Permission 점검|

### 📌 XSS 종류

|종류|설명|
|---|---|
|**Reflective XSS**|•공격자가 악성스크립트를 포함한 URL만을 사이트에 노출<br>•악성스크립트가 서버에 직접 저장되지 않음|
|**Stored XSS**|•공격자가 취약한 게시판에 악성 스크립트를 직접 입력|
|**DOM Based XSS**|•DOM(Document Object Model)을 통해 악성 스크립트가 DOM 구문 수행 시마다 삽입|

### 📌 변형 SQL Injection

| 종류                                   | 설명                                                                                                                                                                                 |
| ------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Blind SQL Injection**<br>(무차별로 대입) | •SQL문 조작으로 인증 우회 외에 DB에 저장된 값(주로 패스워드) 추정 가능<br>•결과가 True or False인지만 확인이 목적<br>•예: `SELECT * FROM USER WHERE substring(password,1,1)='a'` → a~z, 0~9 **반복 입력하면 패스워드 첫 번째 문자 알아냄** |
| **Mass SQL Injection**               | •DB 스키마 정보를 읽어와서 모든 컬럼 값을 대량 변조<br>•주로 SQL 인젝션을 통해 게시판 혹은 사용자에게 노출되는 페이지 정보를 저장하고 있는 컬럼에 악성 스크립트를 삽입                                                                               |

### 📌 CSRF (Cross-site request forgery)

|구분|내용|
|---|---|
|**개념**|•변조된 교차 사이트 요청 공격<br>•공격 기법 자체는 XSS와 동일, 공격의 목적이 다름|
|**XSS와 차이**|•XSS: 사용자 Cookie값·Session ID 탈취 목적<br>•CSRF: 서버에 특정 악의적인 요청(POST, GET)을 수행하도록 유도하는 것이 목적|
