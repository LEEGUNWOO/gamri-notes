## PERT 핵심 계산식

| 공식 종류           | 계산식              | 특징                                                     |
| --------------- | ---------------- | ------------------------------------------------------ |
| ==**삼각분포**==        | ==((O + M + P)/3)==  | ==단순평균, 모든 값에 동일 가중치, 빠르고 간단함 >> μ==                       |
| ==**PERT (베타분포)**== | ==((O + 4M + P)/6)== | ==M에 높은 가중치, 현실적 예측에 유리  >> μ==                            |
| ==**표준편차** (PERT)== | ==((P - O)/6)==      | ==예측의 불확실성 정도 측정 >> σ==                                    |
| 분산 (Variance)   | ((P - O)/6)²     | 분산은 최소값(O)과 최대값(P)을 사용하여 작업 시간의 **변동성**을 측정하는 데 사용됩니다. |
| Z스코어            | Z = (X-μ)/σ      | 3점기법의 계산 평균 =  μ   ,표준편차 = σ   X = 관측값                 |
|                 |                  |                                                        |

✔ O 최적시간 M 가능성있는시간 P 비관적시간
✔ 베타분포 가정  
✔ 3점 추정
✔ Z=(값−평균)/표준편차

| ------------- | ------ | --------------- |
| ==**1시그마 (1σ)**== | ==μ ± 1σ== | ==**68.23%**==      |
| ==**2시그마 (2σ)**== | ==μ ± 2σ== | ==**95.5%**==       |
| ==**3시그마 (3σ)**== | ==μ ± 3σ== | ==**99.7%**==       |
| **4시그마 (4σ)** | μ ± 4σ | **99.9937%**    |
| **5시그마 (5σ)** | μ ± 5σ | **99.99994%**   |
| **6시그마 (6σ)** | μ ± 6σ | **99.9999998%** |


# EVM 
### 획득일정 (Earned Schedule, ES) 최소 정리

|항목|내용|
|---|---|
|정의|**EV가 계획상 달성되어야 하는 시점(시간)**|
|목적|EVM의 **일정 성과를 시간 단위로 분석**|
|필요 데이터|**PV(계획가치) 누적 일정**|
|핵심 특징|**비용이 아니라 시간 기반 일정관리**|

### 관련 지표

| 항목  | 의미                      |
| --- | ----------------------- |
| EV  | 실제 완료된 작업의 예산 가치        |
| PV  | 계획된 작업의 예산 가치           |
| ES  | **EV가 계획 일정에서 달성되는 시점** |
| AT  | 현재 실제 시간                |

| 지표     | 공식              | 의미                        |
| ------ | --------------- | ------------------------- |
| ES     | Earned Schedule | EV에 해당하는 계획 일정 시점         |
| SV(t)  | ES − AT         | 시간 기준 일정 편차               |
| SPI(t) | ES / AT         | 시간 기준 일정 성과지수 (비용기준과 동일함) |
pv rate = bac / 총기간

ES = EV/ (총기간/ pv rate)

## 투자평가 기법 정리

|기법|영어|계산 방법|의미|판단 기준|
|---|---|---|---|---|
|PV|Present Value|미래 현금흐름 ÷ (1+r)^t|미래 돈의 현재 가치|값이 클수록 좋음|
|NPV|Net Present Value|PV 합 − 초기투자비|순현재가치|**NPV > 0 투자 가능**|
|EPV|Equivalent Present Value|반복사업을 동일기간 기준 PV 환산|반복 투자 비교|값이 큰 사업 선택|
|회수기간법|Payback Period|투자금 회수까지 걸리는 기간|투자금 회수 속도|짧을수록 좋음|
|IRR|Internal Rate of Return|NPV=0 되는 할인율|투자 수익률|**IRR > 요구수익률**|
|BCR|Benefit Cost Ratio|편익 PV ÷ 비용 PV|비용 대비 편익 비율|**BCR > 1**|
# 상황 (이야기)

건우가 **100만원으로 커피 자판기 사업**을 고민합니다.

- 투자: **100만원**
    
- 매년 수익: **50만원**
    
- 기간: **3년**
    
- 은행 이자율: **10%**
    

건우는 이렇게 생각합니다.

100만원 투자해서 3년동안 150을 벌겠구나!!

하지만 비교

---

# 1️⃣ 미래 돈을 지금 가치로 바꾼다

### PV (현재가치, Present Value)

건우 생각

> "3년 뒤 50만원은 지금 50만원이랑 같지 않지?"

은행 금리 **10%** 기준으로 계산합니다.

|시점|실제 받는 돈|현재 가치|
|---|---|---|
|1년 후|50|45|
|2년 후|50|41|
|3년 후|50|37|

현재 가치 합

[  
45 + 41 + 37 = 123  
]

즉

> 이 사업이 만들어내는 **현재 가치 = 123만원**

---

# 2️⃣ 투자보다 이익인지 확인

### NPV (순현재가치, Net Present Value)

건우 생각

> "투자 100인데 현재 가치가 얼마지?"

[  
NPV = 123 - 100 = 23  
]

의미

> 이 사업 하면 **23만원 가치가 생긴다**

판단

**NPV > 0 → 투자**

---

# 3️⃣ 언제 투자금 회수되는지 확인

### 투자회수기간 (Payback Period)

건우 생각

> "돈 언제 회수하지?"

|연도|누적수익|
|---|---|
|1년|50|
|2년|100|
|3년|150|

투자 회수

**2년**

---

# 4️⃣ 이 사업 수익률이 몇 %인지

### IRR (내부수익률, Internal Rate of Return)

건우 생각

> "이 사업은 연 몇 %짜리 사업이지?"

계산하면

IRR ≈ **23%**

판단

은행 금리

**10%**

즉

> 은행보다 훨씬 높은 사업

---

# 5️⃣ 편익이 비용보다 큰지

### BCR (편익비용비, Benefit Cost Ratio)

건우 생각

> "1만원 투자하면 얼마 가치 생기지?"

[  
BCR = 123 / 100 = 1.23  
]

의미

> **1만원 투자 → 1.23만원 가치**

판단

**BCR > 1 → 투자**

---

# 6️⃣ 다른 기간 사업과 비교

### EPV (연간균등현재가치, Equivalent Present Value)

건우가 고민합니다.

- 커피머신 A → 3년
    
- 커피머신 B → 5년
    

기간이 다르므로

> **연간 가치 기준으로 환산해서 비교**

이때 사용하는 것이 **EPV**

---

# 전체 흐름 (직관적 순서)

건우가 투자 판단하는 순서

1️⃣ 미래 돈을 현재 가치로 환산 → **PV**  
2️⃣ 투자하고 남는 가치 확인 → **NPV**  
3️⃣ 언제 투자금 회수 → **투자회수기간**  
4️⃣ 사업 수익률 확인 → **IRR**  
5️⃣ 편익/비용 비율 확인 → **BCR**  
6️⃣ 기간 다른 사업 비교 → **EPV**

## 조직 구조 비교

| 구분          | 기능 조직 (Functional Organization) | 프로젝트 조직 (Projectized Organization) | 매트릭스 조직 (Matrix Organization) |
| ----------- | ------------------------------- | ---------------------------------- | ----------------------------- |
| 조직 구조       | 기존 부서 중심 (개발팀, 운영팀 등)           | 프로젝트 중심으로 독립 조직                    | 기능 + 프로젝트 혼합                  |
| 권한 중심       | 기능 관리자                          | 프로젝트 관리자                           | 기능 관리자 + 프로젝트 관리자             |
| 프로젝트 관리자 권한 | 매우 약함                           | 매우 강함                              | 중간 (유형에 따라 다름)                |
| 인력 배치       | 부서 내 인력                         | 프로젝트 전담 인력                         | 부서 소속 + 프로젝트 참여               |
| 의사결정        | 기능 관리자                          | 프로젝트 관리자                           | 공동 결정                         |
| 자원 관리       | 기능 부서                           | 프로젝트 관리자                           | 기능 관리자 중심                     |
| 커뮤니케이션      | 수직 구조                           | 프로젝트 중심                            | 복합 구조                         |
| 장점          | 전문성 유지                          | 의사결정 빠름                            | 자원 활용 효율                      |
| 단점          | 프로젝트 우선순위 낮음                    | 자원 중복 가능                           | 권한 충돌                         |

## 매트릭스 조직 세부 유형

| 유형                        | 특징                     |
| ------------------------- | ---------------------- |
| 약한 매트릭스 (Weak Matrix)     | 기능 관리자 권한이 강함          |
| 균형 매트릭스 (Balanced Matrix) | 기능 관리자와 프로젝트 관리자 권한 비슷 |
| 강한 매트릭스 (Strong Matrix)   | 프로젝트 관리자 권한이 강함        |

(출처: PMI, PMBOK Guide)

## 핵심 암기 포인트 (시험)

|조직|핵심 특징|
|---|---|
|기능 조직|**부서 중심, PM 권한 약함**|
|프로젝트 조직|**프로젝트 중심, PM 권한 강함**|
|매트릭스 조직|**이중 보고 구조**|

---

## 한 줄 정리

- **기능 조직** → 부서 중심
    
- **프로젝트 조직** → 프로젝트 중심
    
- **매트릭스 조직** → 부서 + 프로젝트 혼합

# 린 방식 (Lean)

## 정의

> **가치를 창출하지 않는 모든 낭비를 제거하여 효율을 극대화하는 관리 방식**

---

# 린 방식의 핵심 개념

|개념|설명|
|---|---|
|고객가치 중심 (Value)|고객이 원하는 가치에 집중|
|낭비 제거 (Waste Elimination)|불필요한 활동 제거|
|지속적 개선 (Continuous Improvement)|끊임없는 프로세스 개선|
|흐름 최적화 (Flow)|작업 흐름을 끊김 없이 유지|

---

# 린에서 말하는 7가지 낭비 (시험 자주 나옴)

|낭비 유형|설명|
|---|---|
|과잉생산|필요 이상 생산|
|대기|작업 지연|
|운송|불필요한 이동|
|과잉가공|불필요한 처리|
|재고|과도한 재고|
|동작|불필요한 작업 동작|
|불량|품질 문제로 재작업|

---

# 시험용 핵심 한 줄

린 방식 = 낭비 제거 + 고객 가치 극대화
