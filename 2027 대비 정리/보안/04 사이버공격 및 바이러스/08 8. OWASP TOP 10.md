|구분|내용|
|---|---|
|**개념**|•국제 웹 보안 표준 기구 OWASP에서 공개하는 취약점 목록<br>•웹 애플리케이션 취약점 중 빈도가 많이 발생하고 보안상 영향을 크게 줄 수 있는 것들 10가지를 선정하여 3~4년 주기로 발표|

### 📌 OWASP TOP 10 (2025 기준)

| TOP 10                                | 취약점                 |
| ------------------------------------- | ------------------- |
| Broken Access Control                 | •취약한 접근 통제          |
| Security Misconfiguration             | •보안 설정 미흡           |
| Software Supply Chain Failures        | •소프트웨어 공급망 실패       |
| Cryptographic Failures                | •암호화 실패             |
| Injection                             | •인젝션                |
| Insecure Design                       | •취약한 설계             |
| Authentication Failures               | •인증 실패              |
| Software or Data Integrity Failures   | •소프트웨어 및 데이터 무결성 실패 |
| Logging & Alerting Failures           | •로깅 및 경고 실패         |
| Mishandling of Exceptional Conditions | •예외적 조건의 미흡         |

### 📌 OWASP 세부 항목

| 항목                                                                   | 개념                                                                                       | 영향도                                                                                                                                      | 대응방안                                                                                                                                      |
| -------------------------------------------------------------------- | ---------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------- |
| **A1. Broken Access Control**<br>(취약한 접근 통제)                         | •접근 통제 실패. **공격자가 권한을 우회**하거나 무력화하여 허용된 권한을 넘어서는 정보 조회·수정·삭제가 가능한 취약점                    | •시스템 파일 노출 가능: passwd, shadow 등 운영체제 계정 관리 정보 노출 가능                                                                                      | •권한 검증 미들웨어를 통한 역할 기반(RBAC)과 같은 견고한 권한 검증<br>•순차적으로 증가하지 않는 유니크 키 사용                                                                      |
| **A2. Security Misconfiguration**<br>(보안 설정 미흡)                      | •시스템·애플리케이션·클라우드 서비스·컨테이너 등의 환경 설정이 보안상 적절하게 구성되지 않은 취약점 이용                              | •클라우드 설정 오류(S3 버킷 노출, IAM 과도한 권한 부여 등), CORS 오설정 등                                                                                       | •서비스 환경 설정 시 보안 사항 준수<br>•기본이 잘 설정된 OS 이미지·컨테이너 이미지·IaC 템플릿을 만들어 사용 및 템플릿 보안 스캔                                                           |
| **A3. Software Supply Chain Failures**<br>(소프트웨어 공급망 실패)             | •소프트웨어 공급망 보안 실패. 오래된 라이브러리 관리를 넘어서 공급망에서의 문제 및 공급망 관리의 보안까지 포함                          | •외부 종속성에서 발생하는 취약점(예: React2Shell)<br>•빌드 시스템·서명 인증서 해킹<br>•npm이나 PyPI와 같은 패키지 저장소 해킹                                                    | •SCA(Software Composition Analysis) 도구 사용 및 SBOM(Software Bill of Materials) 구축<br>•빌드 파이프라인의 접근 통제 및 외부 공급망(주로 레파지토리)의 접근 권한 관리 및 MFA 적용 |
| **A4. Cryptographic Failures**<br>(암호화 실패)                           | •암호학적 실패. 민감한 데이터를 노출시킬 수 있는 암호화 부재·암호화 강도가 낮은 암호화·암호화 키 유출 및 하드코딩된 암호화 키 등 암호화 관리 미흡    | •부적절한 암호화 키 관리(하드코딩·키 교체 주기·키 접근 제어)                                                                                                     | •AES-256, SHA-256, TLS 1.3과 같이 강력하며 잘 검증된 암호화 표준을 사용                                                                                      |
| **A5. Injection**<br>(인젝션)                                           | •SQL·LDAP 등의 전송데이터에 매개변수 값 등을 조작하여 악의적인 질의(Query)를 삽입, 공격자가 의도하는 결과 출력                   | •SQL 인젝션에 취약한 쿼리수행 구문: `Select * from member where id='$id' and pass='$pass'` → id 필드에 `test'--` 삽입 시 --를 주석으로 인식하여 id만 가지고 인증 수행 가능성 존재 | •주석(--, #등) 문자열을 필터링<br>•prepared statement 구문으로 질의 수행<br>•저장 프로시저 사용                                                                     |
| **A6. Insecure Design**<br>(취약한 설계)                                  | •안전하지 않은 설계. 코딩 단계가 아닌 설계 단계에서 보안 제어가 누락되거나 비효율적일 때 발생하는 위험                              | •비즈니스 로직 취약점                                                                                                                             | •위협 모델링 수행 및 개발 이전 디자인 리뷰<br>•Secure by Design (보안이 설계 기본 요구사항)                                                                           |
| **A7. Authentication Failures**<br>(인증 실패)                           | •인증 실패. 유효하지 않거나 부정확한 사용자를 정상적인 요청으로 인식하도록 **시스템을 속일 수 있게 만드는 위험**                       | •약한 비밀번호 정책·크리덴셜 스터핑 및 무차별 대입 공격 가능<br>**•인증시스템의 문제 취약한 접근통제는 공격자가 무력화**                                                                 | •강력한 비밀번호 정책 및 보호(유출되었거나 약한 비밀번호를 사용하지 못하게 제한)<br>•표준 인증 라이브러리 사용                                                                         |
| **A8. Software and Data Integrity Failures**<br>(소프트웨어 및 데이터 무결성 실패) | •소프트웨어 또는 데이터 무결성 실패. 소프트웨어나 데이터의 무결성을 검증하지 않고 사용하는 경우 발생하는 취약점                          | •호스트 서버의 변조·DNS 스푸핑 또는 전송 시의 코드 변조 등의 방법을 이용하여 공격자가 악의적인 코드를 실행할 수 있다                                                                    | •신뢰할 수 있는 종속성 저장소만 사용하고 다운로드 받은 소스코드나 실행파일은 무결성 검사를 실행해야 한다                                                                               |
| **A9. Logging & Alerting Failures**<br>(로깅 및 경고 실패)                  | •로깅 및 알람 실패. 악의적인 공격 징후에 대한 로그를 남기지 않거나 로그를 기록하더라도 모니터링을 수행하지 않으면 지속적인 공격의 대상이 될 가능성이 있음 | •침해가 발생해도 침해 사실을 인지하지 못 하거나 지속적인 공격에 무방비로 노출                                                                                             | •공격의 징후(대량의 포트 스캐닝·Brute force 공격 등)에 대한 로그 기록과 모니터링을 통해 공격 징후를 파악하고 사전에 공격을 차단할 수 있도록 방비                                                 |
| **A10. Mishandling of Exceptional Conditions**<br>(예외적 조건의 미흡)       | •잘못된 예외 처리. 예상치 못하거나 예측 불가능한 상황에 대해 예방·감지 및 대응하는 데 실패하여 장애·취약점으로 이어질 때 발생하는 위험           | •에러 정보 노출 - 중요정보·민감 정보 노출<br>•Failing Open                                                                                               | •엄격한 입력값 검증<br>•Fail closed                                                                                                               |
