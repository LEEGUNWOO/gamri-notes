# ⭐ 네트워크보안

알겠어, 먼저 포트번호 검색하고 TCP/IP 계층도 추가해서 완성본 뽑을게.IANA 기준 포트번호 검증 완료. 이제 완성본 뽑을게.

검증 결과:

- HTTP: 80, POP3: 110, TFTP: 69, FTP: 21, Telnet: 23, SMTP: 25, DNS: 53, SNMP: 161 ✓
- SIP: 5060 (IANA 등록 표준)


## 📘 OSI 7 Layer & TCP/IP

### 📌 OSI 7계층 개념 및 목적

|구분|내용|
|---|---|
|개념|•모든 유형의 컴퓨터 시스템간 통신을 허용하는 네트워크 설계를 위한 계층구조|
|목적|•하드웨어/소프트웨어 변경 없이 서로 다른 시스템(통신망)간 개방 통신|
|특징|•물리적 접속 규격부터 업무처리까지 7계층으로 나눠 통신 기능 체계화|

### 📌 OSI 계층 그룹 분류

|그룹|계층|역할|
|---|---|---|
|네트워크 지원 계층|•1·2·3계층|•하나의 장치에서 다른 장치로 전송되는 데이터의 물리적인 면 처리|
|트랜스포트 계층|•4계층|•두 개의 서브 그룹을 링크, 하위층 전송 내용을 상위층이 사용할 수 있는 형태로 변환|
|사용자 지원 계층|•5·6·7계층|•관련없는 소프트웨어 시스템간의 상호 운용성 제공|

### 📌 OSI 7계층 & TCP/IP 전체 비교

|OSI 계층|TCP/IP|데이터 단위|주요 프로토콜|N/W 장비|핵심 기능|
|---|---|---|---|---|---|
|7. Application|Application|•Message|•FTP(21), Telnet(23), SMTP(25)<br>•DNS(53), TFTP(69), SNMP(161)<br>•HTTP(80), POP3(110), SIP(5060)|•Gateway|•응용 프로그램간 네트워크 정보 교환<br>•최종 서비스 제공|
|6. Presentation|Application|•Message|•JPEG, MPEG, ASCII, EBCDIC, TIFF|•Gateway|•코드변환, 데이터 압축, 암복호화<br>•응용 프로그램간 Data 표현차이 해소|
|5. Session|Application|•Message|•-|•Gateway|•응용 프로그램간 논리적 연결 확립/관리<br>•전송모드 결정(직렬, 병렬, 동기, 비동기)|
|4. Transport|Transport|•Segment|•TCP, UDP, SCTP|•-|•신뢰성 있는 연결 보장<br>•Message 분할과 재조립|
|3. Network|Internet (IP)|•Packet|•IP, ICMP, ARP, RARP, IGMP|•Router|•송수신 Node간 패킷 전송<br>•Routing(논리적 통신선로)<br>•교환방식(회선, 패킷)|
|2. Data Link|Network Access|•Frame|•Ethernet, HDLC, PPP, ATM, X.25|•Bridge, 스위치|•인접 Node간 Frame 전송<br>•오류제어, Frame화<br>•BEC, FEC, 해밍코드, MAC|
|1. Physical|Network Access|•Bit|•-|•Repeater, 허브|•물리적 링크 성립·관리<br>•전기·기계·기능·절차적 사양 정의|

### 📌 TCP/IP 개념 및 특징

| 구분  | 내용                                                                                 |
| --- | ---------------------------------------------------------------------------------- |
| 개념  | •네트워크를 상호 연결시켜 컴퓨터간 정보 전송하는 프로토콜 집합<br>•TCP와 IP 2가지만이 아닌 관련 모든 프로토콜 총칭             |
| 구성  | •IP(3계층, 패킷 통신) + TCP(4계층, 전송 조절)                                                  |
| 특징  | •특정 HW/OS에 의존하지 않는 오픈 프로토콜<br>•전세계 유일한 주소체계 사용<br>•ISO 공식 표준은 아니나 사실상 표준(De Facto) |

### 📌 TCP/IP 4계층 구조도

```
┌───────────────────────────────────────────────┐
│                Application 계층                │
│   FTP(21) · Telnet(23) · SMTP(25) · DNS(53)  │
│   TFTP(69) · SNMP(161) · HTTP(80)            │
│   POP3(110) · SIP(5060)                       │
│              (OSI 5·6·7계층 대응)              │
├───────────────────────────────────────────────┤
│                Transport 계층                  │
│            TCP              UDP               │
│   ┌─────────────────┬─────────────────┐       │
│   │ 순서유지        │ 순서유지 안함   │       │
│   │ 중복·손실 없음  │ 중복·손실 가능  │       │
│   │ 에러시 재전송   │ 재전송 없음     │       │
│   │ 슬라이딩윈도우  │ 흐름제어 없음   │       │
│   └─────────────────┴─────────────────┘       │
│              (OSI 4계층 대응)                  │
├───────────────────────────────────────────────┤
│              Internet (IP) 계층                │
│        IP · ICMP · ARP · RARP · IGMP         │
│              (OSI 3계층 대응)                  │
├───────────────────────────────────────────────┤
│             Network Access 계층                │
│      Ethernet · HDLC · PPP · ATM · X.25      │
│              (OSI 1·2계층 대응)                │
└───────────────────────────────────────────────┘
```

### 📌 계층별 상세 기능

#### 🔥 Physical (물리 계층)

|항목|내용|
|---|---|
|핵심 기능|•물리적 링크 성립·관리·활성화|
|규격 정의|•전기적·기계적·기능적·절차적 사양|
|세부 내용|•핀 배치, 전압, 전선 명세<br>•물리적 정보 전달 매개체 연결 성립 및 종료<br>•다수 사용자간 통신 자원 효율적 분배|

#### 🔥 Data Link (데이터 링크 계층)

|항목|내용|
|---|---|
|핵심 기능|•인접한 두 Node간 Frame 전송|
|세부 내용|•DATA Link 설정 및 Frame화<br>•물리 계층 데이터 전송 제어<br>•물리 계층 오류 탐지·수정 기능 및 절차적 수단 제공|
|오류제어 방식|•BEC, FEC, 해밍코드, MAC|

#### 🔥 Network (네트워크 계층)

|항목|내용|
|---|---|
|핵심 기능|•송신 Node로부터 수신 Node까지 패킷 전송<br>•네트워크 전체에 걸친 논리적 통신선로 제공(Routing)|

| 프로토콜 | 기능                                      |
| ---- | --------------------------------------- |
| IP   | •패킷 스위칭 네트워크에서 데이터 송수신                  |
| ICMP | •호스트-게이트웨이간 에러 및 제어정보 전달 (ping)         |
| ARP  | •IP주소 → MAC주소 변환                        |
| RARP | •MAC주소 → IP주소 변환                        |
| IGMP | •**호스트가 멀티캐스트 그룹 가입/탈퇴를 라우터에 알리는** 프로토콜 |

#### 🔥 Transport (전송 계층)

|항목|내용|
|---|---|
|핵심 기능|•통신 Host사이의 신뢰성 있는 연결 보장|
|세부 내용|•네트워크 계층의 패킷에 대한 전송 제어<br>•Message 분할과 재조립|

|구분|TCP|UDP|
|---|---|---|
|데이터 순서|•순서 유지|•순서 유지 안함|
|데이터 중복/손실|•없음|•가능|
|에러 제어|•에러 시 재전송|•재전송 없음|
|흐름 제어|•슬라이딩 윈도우|•없음|
|주요 기능|•Full-duplex, Byte stream service 지원|•빠른 응답 요구 응용서비스, 비연결형|

#### 🔥 Session (세션 계층)

|항목|내용|
|---|---|
|핵심 기능|•응용 프로그램간 논리적 연결 확립·관리|
|세부 내용|•세션 연결 생성, 데이터 전송 중 연결 유지 및 해제 통제<br>•데이터 송수신 설정 및 동기 제어|
|Transport와 차이|•세션계층 = 응용 프로그램 사이 연결<br>•전송계층 = 컴퓨터 시스템 사이 연결|
|전송모드|•Simplex: 한 방향으로만 통신<br>•Half-duplex: 양방향이나 한 번에 한 시스템만 전송<br>•Full-duplex: 양방향 동시 전송 가능|

#### 🔥 Presentation (표현 계층)

|항목|내용|
|---|---|
|핵심 기능|•응용 프로그램간 Data 표현차이 해소|
|세부 기능|•코드변환, 데이터 압축, 암복호화|
|관련 표준|•ASCII, EBCDIC, TIFF, JPEG, MPEG|
#### 🔥 Application (응용 계층)

|항목|내용|
|---|---|
|핵심 기능|•응용 프로그램간 Network를 통한 정보 교환<br>•응용 프로세스와 직접 관계하여 일반적인 응용 서비스 수행<br>•가장 상위 계층으로 일반 사용자에게 최종 서비스 제공|
|관련 프로토콜|•SMTP, HTTP, FTP, WWW, TFTP|


|프로토콜|Port|주요 기능|
|---|---|---|
|FTP|•21|•서버-클라이언트 파일 전송|
|Telnet|•23|•원격 접속|
|SMTP|•25|•전자우편 송신|
|DNS|•53|•도메인 이름 → IP주소 변환|
|TFTP|•69|•간이 파일 전송|
|SNMP|•161|•네트워크 장치 정보 수집·관리|
|HTTP|•80|•WWW상에서 HTML 문서 송수신|
