## 📘 1. 공격(Attacks) / 해킹(Hacking)

### 📌 개념

| 구분     | 내용                                                                                                           |
| ------ | ------------------------------------------------------------------------------------------------------------ |
| **개념** | •정보 시스템을 파괴·마비시키거나 정상적인 운용을 방해하는 행위<br>•정보 시스템에 침투하여 정보를 위조·변조·파괴·유출하는 행위<br>•정보 시스템의 기밀성·무결성·가용성을 해치는 모든 행위 |

### 📌 공격의 분류

| 구분                      | 설명                                          | 예시                                                                   |
| ----------------------- | ------------------------------------------- | -------------------------------------------------------------------- |
| 수동적 공격 (Passive Attack) | •실제 악의적 행위 없음 (몰래 도청)<br>•기밀성 해침<br>•탐지 어려움 | •Sniffing<br>•Traffic Analysis<br>•Port Scan                         |
| 능동적 공격 (Active Attack)  | •실제 악의적 행위 수행<br>•무결성·가용성 해침<br>•탐지 가능      | •Spoofing<br>•Replay Attack<br>•메시지 변조<br>•DOS<br>•Session Hijacking |

### 📌 공격 유형 전체 개요 (목적·계층 기준)

| 계층       | 공격 유형 (대분류)                  | 대표 기법                                                   | 핵심 포인트                         |
| -------- | ---------------------------- | ------------------------------------------------------- | ------------------------------ |
| **네트워크** | 스니핑 (Sniffing)               | Packet Sniffing, Switch Jamming                         | 네트워크 패킷을 몰래 도청하여 평문 데이터 탈취     |
| **네트워크** | 스푸핑 (Spoofing)               | IP Spoofing, ARP Spoofing, DNS Spoofing, Email Spoofing | IP·MAC·DNS 등 식별정보를 위조하여 신원 속임  |
| **네트워크** | 중간자 공격 (MitM)                | SSL Stripping                                           | 두 통신 주체 사이에 끼어들어 내용 도청·변조      |
| **네트워크** | 세션 하이재킹 (Session Hijacking)  | IP Spoofing 활용                                          | 이미 맺어진 세션을 탈취하여 정상 사용자 행세      |
| **네트워크** | 재전송 공격 (Replay Attack)       | -                                                       | 캡처한 인증 패킷을 그대로 재전송하여 인증 우회     |
| **네트워크** | 서비스 거부 (DoS/DDoS)            | SYN Flood, Smurf, Slowloris, DNS Amplification          | 대량 트래픽·비정상 패킷으로 서비스 마비         |
| **응용/웹** | 웹 공격 (Web Attack)            | XSS, CSRF, SQL Injection, Buffer Overflow               | 입력값 검증 부재를 이용해 스크립트·쿼리·메모리 조작  |
| **응용/웹** | 무차별 대입 (Brute Force)         | Dictionary Attack, Rainbow Table                        | 가능한 모든 경우의 수를 대입하여 암호·계정 탈취    |
| **시스템**  | 권한 상승 (Privilege Escalation) | Local Exploit                                           | 일반 사용자 권한으로 관리자 권한까지 획득        |
| **시스템**  | 악성코드 (Malware)               | Trojan, Ransomware, Rootkit, Dropper                    | 악성 프로그램으로 시스템 감염·데이터 암호화·은닉    |
| **무선**   | 블루투스 공격                      | 블루스나핑, 블루버깅, 블루본, KNOB                                  | 블루투스 취약점으로 데이터 탈취·기기 원격 제어     |
| **무선**   | 무선 AP 공격                     | 이블 트윈                                                   | 가짜 AP로 사용자를 속여 Wi-Fi 트래픽 탈취    |
| **기타**   | APT                          | Zero-day, 워터링홀, 스피어피싱                                   | 특정 목표를 장기간 지속적·조직적으로 공격        |
| **기타**   | 암호 공격                        | CCA, CPA, Rainbow Table                                 | 수학적 기법·사전·무차별 대입으로 암호문 해독 시도   |
| **기타**   | AI 공격                        | Poisoning, Evasion, Inversion, Membership Inference     | 학습 데이터 조작·적대적 예제로 AI 모델 오작동 유발 |
| **기타**   | 사회공학                         | 피싱, 스미싱, 스피어피싱                                          | 사람의 심리를 이용해 자격증명·정보 탈취         |
| **기타**   | 정보 수집 (Footprinting)         | Whois, Port Scan, 블루프린팅                                 | 공격 전 대상 시스템 정보를 수집하는 사전 단계     |

### 📌 공격기법 총괄표
| 대분류                              | 공격 기법                           | 공격 원리                                                                                                                  | 시험 키워드                                                          |
| -------------------------------- | ------------------------------- | ---------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------- |
| **스니핑 (Sniffing)**               | Packet Sniffing                 | •NIC를 Promiscuous 모드로 설정<br>•자신에게 오지 않는 패킷도 전부 수신<br>•평문 전송 데이터 그대로 탈취                                                 | •Promiscuous 모드<br>•Traffic Analysis                            |
|                                  | Switch Jamming                  | •가짜 MAC 주소를 대량 전송 → 스위치 MAC 테이블 오버플로우<br>•테이블 가득 차면 스위치가 허브처럼 모든 포트에 브로드캐스트<br>•공격자도 남의 패킷 수신 가능                       | **•MAC Flooding**<br>•MAC 테이블 오버플로우<br>•Fail Open               |
| **스푸핑 (Spoofing)**               | IP Spoofing                     | •자신의 IP를 신뢰된 호스트 IP로 위조<br>•Sequence Number 추측 → ACK 전송 (Blind IP Spoofing)<br>•사전에 대상 클라이언트 DoS로 무력화 (RST 방지)         | •Blind IP Spoofing<br>•Kevin Mitnick 1995                       |
|                                  | ARP Spoofing                    | •MAC주소 위장<br>•위조된 ARP Reply로 상대방 ARP Cache에 공격자 MAC 주입<br>•이후 트래픽이 공격자를 경유하게 됨                                         | •ARP Cache 위조<br>•2계층 공격<br>**•ARP Cache Poisoning**            |
|                                  | DNS Spoofing                    | •DNS 응답 메시지를 위조하여 정상 도메인 IP를 공격자 서버 IP로 변조<br>•사용자가 의도치 않은 서버로 접속                                                      | •DNS Cache Poisoning<br>•파밍(Pharming)                           |
|                                  | Email Spoofing                  | •이메일 From 필드(발신자 주소)를 위조<br>•신뢰할 수 있는 발신자인 척 악성 메일 발송                                                                  | •From 필드 위조                                                     |
| **중간자 공격 (MitM)**                | SSL Stripping                   | •HTTPS 요청을 가로채 공격자↔서버는 HTTPS, 공격자↔사용자는 HTTP로 통신<br>•사용자는 암호화된 줄 알지만 실제로는 평문 전송                                         | •HTTPS → HTTP 다운그레이드<br>•암호화 제거                                 |
| **세션 하이재킹**                      | IP Spoofing 활용                  | •Sniffing으로 TCP Sequence Number 패턴 파악<br>•클라이언트 A를 DoS로 무력화<br>•A의 IP로 위조하여 서버 B와 통신                                   | •Sequence Number 조작<br>•DoS 선행                                  |
| **재전송 공격 (Replay Attack)**       | -                               | •Sniffing으로 정상 인증 패킷 캡처<br>•데이터 해독 없이 캡처한 패킷 그대로 재전송<br>•서버는 정상 인증으로 인식                                                | •타임스탬프로 대응<br>•IPSec 사용                                         |
| **서비스 거부 (DoS/DDoS)**            | SYN Flood                       | •대량의 SYN 패킷 전송 → 서버가 SYN-ACK 응답 후 ACK 대기<br>•ACK 없이 Half-open 연결만 쌓여 백로그 큐 고갈                                          | •Half-open<br>•백로그 큐 고갈                                         |
|                                  | Smurf                           | •Victim IP로 위조된 ICMP Echo Request를 브로드캐스트 주소로 전송<br>•서브넷 내 모든 호스트가 Victim에게 응답 → 트래픽 폭주                                | •ICMP Broadcast<br>•Amplification                               |
|                                  | Fraggle                         | •Smurf와 동일한 방식으로 UDP 패킷 사용<br>•UDP Broadcast 증폭으로 Victim 마비                                                            | •Smurf의 UDP 버전                                                  |
|                                  | Land Attack                     | •출발지 IP와 목적지 IP를 동일하게 설정하여 전송<br>•패킷 받은 호스트가 자기 자신에게 응답 → 무한 루프 발생                                                     | •출발지=목적지 IP<br>•Loop 유발                                         |
|                                  | Teardrop                        | •IP Fragment Offset을 조작하여 패킷 조각 전송<br>•수신측 재조합 시 데이터가 겹치거나 누락 → OS 오류·다운                                               | •Fragment Offset 조작<br>•재조합 오류                                  |

|                                  | Bonk                            | •IP 패킷의 시퀀스 번호를 처음부터 전부 동일하게 조작하여 전송<br>•수신측 재조합 시 오류 발생                                                               | •시퀀스 번호 전부 동일                                                   |
|                                  | Boink                           | •정상적으로 시퀀스 번호 증가하다가 중간에 특정 패킷만 동일하게 중첩<br>•Bonk의 변형                                                                    | •Bonk 변형<br>•중간 번호 중첩                                           |

### 📌 스니핑 (Sniffing)

|구분|내용|
|---|---|
|**개념**|•한 서브 네트워크에서 전송되는 패킷을 몰래 엿듣는 행위  <br>•Eavesdropping(도청) 공격의 대표적 예|
|**목적**|•평문 데이터 탈취 (계정정보·패스워드·개인정보 등)|
|**공격방법**|•NIC를 Promiscuous 모드로 설정 → 자신에게 오지 않는 패킷도 전부 수신  <br>•Switch Jamming, ARP Spoofing, ICMP Redirect 등 사전 공격 병행|
|**대응**|•통신 내용 암호화|
### 📌 스푸핑 (Spoofing)

|구분|내용|
|---|---|
|**개념**|•해커가 자신의 식별정보를 다른 것으로 위장하는 공격<br>•IP·DNS·MAC Address 등 정보를 위조<br>•TCP/IP 프로토콜의 취약성(IP·MAC 기반 인증) 악용|
|**목적**|•신원 위조를 통한 인증 우회 및 정보 탈취|
|**공격방법**|•IP·MAC·DNS 등 식별정보를 위조하여 신뢰된 호스트인 척 통신<br>•인증 체계가 IP·MAC 기반일 때 특히 효과적|
|**대응**|•IDS 설치<br>•TCP 프로토콜 패치<br>•외부에서 내부망 IP를 Source로 가진 패킷 필터링|

|**ARP Spoofing**|•ARP Cache 테이블에 위조된 MAC 주소 주입<br>•트래픽을 공격자 경유하게 만들어 도청·변조 가능<br>•스니핑·MitM 사전 공격으로 활용|•ARP Cache 위조<br>•2계층 공격|

### 📌 세션 하이재킹 (Session Hijacking)

|구분|내용|
|---|---|
|**개념**|•이미 맺어진 TCP 세션을 탈취하여 정상 사용자인 척 통신하는 공격<br>•MitM의 한 종류 (MitM ⊃ Session Hijacking)|
|**목적**|•인증 없이 정상 사용자 권한으로 서버에 접근|
|**공격방법**|•A-B 세션 맺음 (TCP 3-way Handshaking 완료)<br>•Sniffing으로 TCP Sequence Number 패턴 파악<br>•A를 DoS로 무력화 (A가 RST 못 보내게)<br>•IP Spoofing으로 A인 척 B와 통신|
|**대응**|•Sequence Number 복잡성 높임<br>•암호화 통신 (SSL/TLS)|
