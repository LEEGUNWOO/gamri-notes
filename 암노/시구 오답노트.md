
# 🧠 컴퓨터구조 - 명령어(Instruction) 정리

## 📌 1. 명령어란?

|항목|내용|
|---|---|
|명령어 (Instruction)|CPU가 수행할 작업을 나타내는 정보|
|역할|무슨 연산을 할지, 어떤 데이터를 대상으로 할지 지정|
|핵심 구성|연산 종류 + 피연산자 정보|

---

## 📌 2. 명령어 형식 (Instruction Format)

명령어는 보통 다음과 같은 요소로 구성된다.

```text
[ Mode | Opcode | Address ]
```

|구성 요소|의미|
|---|---|
|Mode Field|주소지정방식(Addressing Mode) 결정|
|Opcode|수행할 연산의 종류|
|Address / Operand Field|피연산자의 주소 또는 값|

### ✅ 해석

- **Mode** : 주소를 어떻게 해석할지
    
- **Opcode** : 무엇을 할지
    
- **Address** : 누구를 대상으로 할지
    

---

## 📌 3. 핵심 용어 정리

|용어|의미|쉽게 말하면|
|---|---|---|
|Opcode|연산코드|무슨 일을 할지|
|Operand|피연산자|누구를 대상으로 할지|
|Mode Field|주소지정 모드|주소를 어떤 방식으로 해석할지|
|EA (Effective Address)|유효주소|실제로 접근할 메모리 주소|

---

## 📌 4. 명령어 실행 과정

CPU는 명령어를 보통 다음 순서로 처리한다.

```text
Fetch → Decode → Execute
```

|단계|설명|
|---|---|
|Fetch|메모리에서 명령어를 읽어옴|
|Decode|명령어를 해석|
|Execute|연산 수행|

---

## 📌 5. Fetch Cycle 핵심

|항목|의미|
|---|---|
|PC (Program Counter)|다음 명령어 주소 저장|
|IR (Instruction Register)|현재 명령어 저장|

```text
IR ← M[PC]
PC ← PC + 명령어 길이
```

### ✅ 의미

- 메모리에서 **PC가 가리키는 명령어**를 읽어와 IR에 저장
    
- 다음 명령어를 위해 PC 증가
    

---

## 📌 6. 명령어 길이 (Instruction Length)

|항목|내용|
|---|---|
|명령어 길이|한 명령어를 구성하는 전체 비트 수|
|예시|16bit, 32bit, 64bit|
|특징|길수록 표현력은 커지지만 구조는 복잡해짐|

### ✅ 바이트 단위 주소지정(Byte Addressing)일 때 PC 증가량

|명령어 길이|바이트 수|PC 증가량|
|---|--:|--:|
|16 bit|2 byte|2|
|32 bit|4 byte|4|
|64 bit|8 byte|8|

# 🧩 7. 주소 수에 따른 명령어 형식

## 7-1. 3주소 명령어

```text
OP A, B, C
C ← A OP B
```

예:

```text
ADD A, B, C
C ← A + B
```

|항목|내용|
|---|---|
|특징|결과 저장 위치가 따로 있음|
|장점|가장 직관적|
|단점|명령어 길이가 길어짐|

---

## 7-2. 2주소 명령어

```text
OP A, B
A ← A OP B
```

예:

```text
ADD A, B
A ← A + B
```

|항목|내용|
|---|---|
|특징|결과가 첫 번째 operand에 저장|
|장점|3주소보다 짧음|
|단점|원래 A 값이 덮어써짐|

---

## 7-3. 1주소 명령어

```text
OP A
AC ← AC OP A
```

예:

```text
ADD A
```

|항목|내용|
|---|---|
|특징|누산기(AC, Accumulator) 사용|
|장점|명령어 짧음|
|단점|AC에 의존|

---

## 7-4. 0주소 명령어

```text
PUSH A
PUSH B
ADD
POP C
```

|항목|내용|
|---|---|
|특징|Stack 기반|
|장점|명령어 길이 가장 짧음|
|단점|직관성이 떨어질 수 있음|

---

## 📌 8. 주소 수 비교표

|구분|형식|결과 저장 위치|특징|
|---|---|---|---|
|3주소|OP A, B, C|C|가장 직관적|
|2주소|OP A, B|A|많이 출제|
|1주소|OP A|AC|누산기 사용|
|0주소|OP|Stack Top|스택 구조|

# 🗺️ 9. 주소지정방식 (Addressing Mode)

## 📌 개념

|항목|내용|
|---|---|
|주소지정방식 (Addressing Mode)|Operand를 어디서 가져올지 정하는 방식|
|결정 주체|Mode Field|
|핵심|EA(유효주소, Effective Address) 계산|
|목적|명령어의 주소 필드를 어떻게 해석할지 결정|
  
---

## 📌 10. 주소지정방식 정리표

| 영문명                          | 한글명             | EA 계산식        | Operand   | 설명                                          |
| ---------------------------- | --------------- | ------------- | --------- | ------------------------------------------- |
| Implicit Addressing          | 묵시적 주소지정 방식     | 별도 주소 없음      | 암묵적으로 정해짐 | 명령어 자체에 피연산자가 포함되거나 AC, Stack Top 등으로 자동 결정 |
| Immediate Addressing         | 즉시적 주소지정 방식     | 없음            | A         | 값 자체가 명령어 안에 포함됨                            |
| Direct Addressing            | 직접 주소지정 방식      | EA = A        | M[A]      | 주소 필드의 값이 실제 주소                             |
| Indirect Addressing          | 간접 주소지정 방식      | EA = M[A]     | M[EA]     | 주소 필드가 가리키는 곳에 실제 주소가 있음                    |
| Register Addressing          | 레지스터 주소지정 방식    | 없음            | R         | 피연산자가 레지스터에 있음                              |
| Register Indirect Addressing | 레지스터 간접 주소지정 방식 | EA = R        | M[R]      | 레지스터가 실제 주소를 가지고 있음                         |
| Indexed Addressing           | 인덱스 주소지정 방식     | EA = A + IX   | M[EA]     | 주소 필드 값에 Index Register 값을 더함 (배열)          |
| Relative Addressing          | 상대 주소지정 방식      | EA = PC + A   | M[EA]     | 현재 PC 값을 기준으로 주소 계산                         |
| Base Addressing              | 베이스 주소지정 방식     | EA = Base + A | M[EA]     | Base Register 값을 기준으로 주소 계산 (프로그램 위치변경)     |

---

## 📌 11. 주소지정방식 한 줄 암기

| 영문명                          | 한글명             | 핵심 기억                   |
| ---------------------------- | --------------- | ----------------------- |
| Implicit Addressing          | 묵시적 주소지정 방식     | 피연산자가 명시되지 않고 암묵적으로 정해짐 |
| Immediate Addressing         | 즉시적 주소지정 방식     | 값 자체                    |
| Direct Addressing            | 직접 주소지정 방식      | 주소                      |
| Indirect Addressing          | 간접 주소지정 방식      | 주소의 주소                  |
| Register Addressing          | 레지스터 주소지정 방식    | 레지스터 값                  |
| Register Indirect Addressing | 레지스터 간접 주소지정 방식 | 레지스터가 가리키는 메모리          |
| Indexed Addressing           | 인덱스 주소지정 방식     | A + IX                  |
| Relative Addressing          | 상대 주소지정 방식      | PC + A                  |
| Base Addressing              | 베이스 주소지정 방식     | Base + A                |

---

## 📌 12. 메모리 접근 횟수

|영문명|한글명|메모리 접근 횟수|이유|
|---|---|--:|---|
|Implicit Addressing|묵시적 주소지정 방식|0|피연산자가 명령어 또는 내부 레지스터/스택으로 암묵 결정|
|Immediate Addressing|즉시적 주소지정 방식|0|명령어 안에 값 포함|
|Register Addressing|레지스터 주소지정 방식|0|레지스터 값 사용|
|Direct Addressing|직접 주소지정 방식|1|한 번 바로 접근|
|Indirect Addressing|간접 주소지정 방식|2|주소를 찾고 다시 접근|
|Register Indirect Addressing|레지스터 간접 주소지정 방식|1|레지스터에 주소가 있음|
|Indexed Addressing|인덱스 주소지정 방식|1|EA 계산 후 접근|
|Relative Addressing|상대 주소지정 방식|1|EA 계산 후 접근|
|Base Addressing|베이스 주소지정 방식|1|EA 계산 후 접근|

## 📌 13. 주소지정방식 예시 느낌

| 영문명                          | 한글명             | 예시 느낌             | 설명                            |
| ---------------------------- | --------------- | ----------------- | ----------------------------- |
| Implicit Addressing          | 묵시적 주소지정 방식     | `CMA`, `ADD`(0주소) | 피연산자가 AC나 Stack Top 등으로 자동 결정 |
| Immediate Addressing         | 즉시적 주소지정 방식     | `LOAD #5`         | 5라는 값 자체를 사용                  |
| Direct Addressing            | 직접 주소지정 방식      | `LOAD 500`        | 주소 500번지의 값을 가져옴              |
| Indirect Addressing          | 간접 주소지정 방식      | `LOAD @500`       | 500번지에 들어있는 주소를 한 번 더 따라감     |
| Register Addressing          | 레지스터 주소지정 방식    | `ADD R1`          | R1의 값을 사용                     |
| Register Indirect Addressing | 레지스터 간접 주소지정 방식 | `LOAD (R1)`       | R1이 가리키는 메모리의 값을 사용           |
| Indexed Addressing           | 인덱스 주소지정 방식     | `LOAD 100(XR)`    | 100 + IX 위치를 접근               |
| Relative Addressing          | 상대 주소지정 방식      | `BRANCH +20`      | 현재 PC 기준으로 이동                 |
| Base Addressing              | 베이스 주소지정 방식     | `LOAD 20(Base)`   | Base + 20 위치 접근               |

## 📌 14. 시험용 핵심 구분

|구분 포인트|답|
|---|---|
|값 자체를 사용|즉시적 주소지정 방식 (Immediate Addressing)|
|주소 필드가 실제 주소|직접 주소지정 방식 (Direct Addressing)|
|주소 필드가 가리키는 곳에 실제 주소가 있음|간접 주소지정 방식 (Indirect Addressing)|
|레지스터 자체의 값을 사용|레지스터 주소지정 방식 (Register Addressing)|
|레지스터가 메모리 주소를 가짐|레지스터 간접 주소지정 방식 (Register Indirect Addressing)|
|Index Register 더함|인덱스 주소지정 방식 (Indexed Addressing)|
|PC 기준 계산|상대 주소지정 방식 (Relative Addressing)|
|Base Register 기준 계산|베이스 주소지정 방식 (Base Addressing)|
|피연산자가 명시되지 않음|묵시적 주소지정 방식 (Implicit Addressing)|

# 🔥 24. 암기 블록

```text
명령어 = Opcode + Operand
Mode Field는 주소지정방식 결정
EA는 실제 접근 주소

3주소 : C ← A OP B
2주소 : A ← A OP B
1주소 : AC ← AC OP A
0주소 : Stack 기반

Implicit Addressing          : 묵시적 주소지정 방식       → 피연산자 암묵 결정
Immediate Addressing         : 즉시적 주소지정 방식       → 값 자체
Direct Addressing            : 직접 주소지정 방식         → EA = A
Indirect Addressing          : 간접 주소지정 방식         → EA = M[A]
Register Addressing          : 레지스터 주소지정 방식     → 레지스터 값
Register Indirect Addressing : 레지스터 간접 주소지정 방식 → EA = R
Indexed Addressing           : 인덱스 주소지정 방식       → EA = A + XR
Relative Addressing          : 상대 주소지정 방식         → EA = PC + A
Base Addressing              : 베이스 주소지정 방식       → EA = Base + A

LOAD  : 메모리 → 레지스터
STORE : 레지스터 → 메모리
PUSH  : 넣기
POP   : 꺼내기
ADD/SUB/MUL/DIV : 산술
```

|구분|명령어 흐름|데이터 흐름|구조 특징|대표 시스템|

# 병렬/분산 시스템 구조 정리

| 구분                                     | 구조 유형   | 메모리 구조  | 시스템 형태  | 특징              |
| -------------------------------------- | ------- | ------- | ------- | --------------- |
| ==**밀결합 시스템** (Tightly Coupled System)==   | ==병렬 컴퓨터==  | ==공유 기억장치== | ==하나의 컴퓨터== | ==CPU들이 동일 메모리 공유== |
| ==**소결합 시스템** (Loosely Coupled System)==   | ==분산 시스템==  | ==분산 기억장치== | ==여러 컴퓨터==  | ==네트워크로 연결==        |
| **SMP** (Symmetric Multiprocessor)     | 밀결합 시스템 | 공유 기억장치 | 단일 시스템  | 모든 CPU 동일 권한    |
| **MPP** (Massively Parallel Processor) | 병렬 컴퓨터  | 분산 기억장치 | 병렬 시스템  | CPU마다 메모리       |
| **분산 시스템** (Distributed System)        | 소결합 시스템 | 분산 기억장치 | 여러 컴퓨터  | 네트워크 기반 협력 처리   |
| **클러스터 컴퓨터**                           | 분산 시스템  | 분산 기억장치 | 여러 서버   | 하나의 시스템처럼 사용    |

# 프로세스
# 1. 프로세스 (Process)

**실행 중인 프로그램의 인스턴스**다.  
단순히 파일로 저장된 프로그램이 아니라, **메모리에 올라와 OS가 관리하는 실행 단위**다.

핵심:

- **자원(Resource) 소유의 단위**
    
- **독립된 주소 공간(Address Space)** 을 가짐
    
- 하나의 프로세스 안에 **여러 쓰레드(Thread)** 가 있을 수 있음
    

즉, 프로세스는 **“실행 환경 전체”** 에 가깝다.

---

# 2. PCB (Process Control Block)

PCB는 **프로세스 제어 블록**이다.  
운영체제가 **각 프로세스를 관리하기 위해 커널 안에 두는 관리 정보 묶음**이다.

쉽게 말하면:

- 프로세스 자체가 아니라
    
- **프로세스의 상태표 / 관리카드**
    

주요 내용:

- 프로세스 상태 (`new`, `ready`, `running`, `waiting`, `terminated`)
    
- 프로세스 ID(PID)
    
- 프로그램 카운터(다음 실행 명령 위치)
    
- CPU 레지스터 정보
    
- 스케줄링 정보
    
- 메모리 관리 정보
    
- 열린 파일 / 입출력 정보
    
- 부모/자식 프로세스 관계 등
    

핵심:

- **프로세스를 문맥교환(Context Switching)할 때 기준이 되는 정보**
    
- 운영체제가 프로세스를 추적·복원하는 데 필요
    

---

# 3. 프로세스 메모리 영역

프로세스는 보통 자기만의 주소 공간 안에 다음 영역을 가진다.

## (1) 코드 영역 (Code / Text)

- ==실행할 기계어 명령 ( 상수 함수)==
    
- ==보통 **읽기 전용**==
    
- 프로그램 본체
    

## (2) 데이터 영역 (Data)

- ==전역 변수(Global), 정적 변수(Static)==
    
- 프로그램 시작부터 종료까지 존재
    

## (3) 힙 영역 (Heap)

- ==동적 할당 메모리==
    
- ==`malloc`, `new` 등으로 확보==
    
- ==프로그래머/런타임이 관리==
    

## (4) 스택 영역 (Stack)

- 함수 호출 정보
    
- ==지역 변수==
    
- ==매개변수==
    
- 복귀 주소
    

중요:

- **프로세스의 코드/데이터/힙은 같은 프로세스의 쓰레드들이 공유**
    
- **스택은 보통 쓰레드별로 따로 가짐**
    

---

# 4. 쓰레드 (Thread)

쓰레드는 **CPU 실행 흐름의 단위**다.  
프로세스가 자원 소유 단위라면, 쓰레드는 **실제로 명령을 수행하는 단위**다.

핵심:

- 프로세스 안에서 동작
    
- 같은 프로세스의 다른 쓰레드와 **주소 공간을 공유**
    
- 그래서 생성/전환이 프로세스보다 가벼운 편
    

즉:

- 프로세스 = 작업장
    
- 쓰레드 = 그 작업장에서 실제 일하는 작업 흐름
    

---

# 5. TCB (Thread Control Block)

TCB는 **쓰레드 제어 블록**이다.  
운영체제가 **각 쓰레드를 관리하기 위해 저장하는 정보 묶음**이다.

PCB가 프로세스 관리표라면,  
TCB는 **쓰레드 관리표**다.

주요 내용:

- 쓰레드 ID(TID)
    
- 쓰레드 상태
    
- 프로그램 카운터
    
- 레지스터 값
    
- 스택 포인터
    
- 우선순위
    
- 스케줄링 정보
    
- 소속 프로세스 정보
    

핵심:

- **문맥교환 시 쓰레드 실행 상태 저장/복원**
    
- 특히 쓰레드별 **레지스터, PC, 스택 관련 정보**가 중요
    

---

# 6. 쓰레드끼리 공유하는 것

같은 프로세스 내부의 쓰레드들은 보통 아래를 **공유**한다.

- 코드 영역
    
- 데이터 영역
    
- 힙 영역
    
- 열린 파일
    
- 프로세스의 주소 공간
    
- 프로세스 자원
    

즉, 같은 프로세스의 쓰레드는 **같은 메모리 공간에서 함께 일한다**.

그래서 장점:

- 데이터 공유가 빠름
    

하지만 단점:

- 동기화 문제 발생 가능
    
- 한 쓰레드가 공유 데이터 망치면 다른 쓰레드에도 영향
    

---

# 7. 쓰레드가 독립적으로 가지는 것

이게 핵심이다.  
쓰레드는 공유만 하는 게 아니라, **자기 실행 흐름을 유지하려면 따로 가져야 하는 것**이 있다.

## 쓰레드별 독립 소유

- **프로그램 카운터(PC)**  
    → 각자 어디 명령을 실행 중인지 달라야 함
    
- **레지스터 집합(Register Set)**  
    → 계산 중간값, CPU 작업 상태가 다름
    
- **스택(Stack)**  
    → 함수 호출, 지역변수, 복귀주소가 각 쓰레드마다 다름
    
- **쓰레드 상태 정보**  
    → ready/running/waiting 등
    
- **스케줄링 정보**
    

핵심 한 줄:  
**쓰레드는 실행 흐름 유지를 위한 정보는 독립, 자원과 메모리 대부분은 공유**한다.

---

# 8. 프로세스 vs 쓰레드 한방 비교

|구분|프로세스|쓰레드|
|---|---|---|
|의미|실행 중인 프로그램|실행 흐름|
|단위 성격|자원 소유 단위|CPU 실행 단위|
|주소 공간|독립적|같은 프로세스 내에서 공유|
|생성/전환 비용|큼|상대적으로 작음|
|메모리|프로세스별 독립|코드/데이터/힙 공유|
|독립 소유|PCB, 주소 공간, 자원|PC, 레지스터, 스택, TCB|
