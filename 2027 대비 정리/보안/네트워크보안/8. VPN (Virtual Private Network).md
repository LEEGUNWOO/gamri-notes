## 📘 8. VPN (Virtual Private Network)

### 📌 개념 및 등장배경

|배경|내용|
|---|---|
|근무 환경 변화|•정보 공유 확산, 신속한 의사결정 요구증대<br>•네트워크 급속한 보급, 이동/재택 근무자의 증가|
|네트워크 비용증가|•전용선의 고비용, 원격 접속을 위한 다이얼업 비용 증가<br>•네트워크 장비 구축 및 유지보수 비용 증가|
|인터넷 기술 발전|•저가의 솔루션 등장, 신뢰성을 제공하는 검증된 암호 기술 개발|

### 📌 VPN 도입 효과

|구분|도입효과|내용|
|---|---|---|
|비용절감|회선 비용의 단일화|•기존의 인터넷 회선, 내부 기업전용회선(폐쇄망)을 인터넷 전용 회선으로 단일화|
|비용절감|네트워크 관리 비용 최소화|•다수 전용회선 관리, 유지 및 RAS 시스템의 모뎀 관리 등의 관리 비용이 줄어듦|
|확장성|클라이언트 추가 확장|•기존의 클라이언트 확장은 모뎀, 전화라인, 전용라인의 추가를 의미|
|확장성|지사, 타기업간 확장|•별도의 전용선 추가 없이 VPN 장비 추가 도입 확장|
|안정성|인증 제공|•데이터 인증(MD5, SHA-1), 사용자인증(인증서, 패스워드)|
|안정성|암호화 제공|•비밀키 암호화 등 세션 내용의 암호화|
|안정성|가용성 제공|•이중화, Fail-Safe 기능 등 제공|

### 📌 IPSec VPN

|구분|내용|
|---|---|
|개념|•IP 프로토콜의 일부인 IPSec 프로토콜을 이용하여 VPN을 구현 (전용 VPN 장비 기반 기술)|
|동작계층|•네트워크 계층 (3계층)|
|구성방법|•랜투랜 VPN(Site-to-Site, 혹은 Gateway-to-Gateway)<br>•원격접속 VPN(Site-to-Remote, 혹은 Gateway-to-Remote)|
|적합한 환경|•일반적인 본사-지사 간 VPN 환경<br>•C/S 기반 어플리케이션 운영|
|표준|•RFC 2401|
|장점|•보안수준과 암호화 기능 뛰어남 (높은 보안 수준 유지 가능)<br>•다양한 환경에 적용, 고객이 어플리케이션과 독립적으로 운영(투명성 제공)<br>•다양한 인터넷 접속기술 활용 가능 및 고객사 고유 정책 반영|
|단점|•높은 초기 도입 비용, 각 지사 VPN 장비 필요<br>•트래픽 제어 및 QoS 기능 미약<br>•지속적인 관리비용 발생, 대규모 원격 접속 환경에는 다소 부족함|

### 📌 MPLS VPN

| 구분     | 내용                                                                                    |
| ------ | ------------------------------------------------------------------------------------- |
| 개념     | •패킷 스위칭 기술인 MPLS 환경을 통해 VPN 구현 (네트워크 기반 기술)                                           |
| 동작계층   | •데이터 링크 계층 (2, 3계층)                                                                   |
| 구성방법   | •랜투랜, 원격접속                                                                            |
| 적합한 환경 | •시간에 민감한 어플리케이션 운영 환경 (음성, 동영상)                                                       |
| 표준     | •RFC 2547                                                                             |
| **장점** | **•트래픽 제어 및 QoS 기능 제공, 네트워크 레벨에서 VPN 기능 제공<br>•뛰어난 관리 편의성, 낮은 도입/관리 비용 (장비도입 필요 없음)** |
| **단점** | **•동일 ISP 내부에서만 운영 가능, 고객사 고유정책 반영 미약<br>•공중망 전송 시 암호화 기능 미약함, 대역폭에 비해 고비용 구조**       |
※ **ISP(Internet Service Provider)**  : 인터넷 통신사(KT, SK브로드밴드, LG유플러스)
### 📌 SSL VPN

|구분|내용|
|---|---|
|개념|•보안 통신프로토콜인 SSL 통해 VPN 구현 (네트워크 기반 기술)|
|동작계층|•전송 계층 ~ 응용 계층 (4~7계층)|
|구성방법|•원격접속|
|적합한 환경|•다수의 원격 사용자를 가진 환경<br>•웹 기반 어플리케이션 운영 환경|
|장점|•별도 장비 없이 웹브라우저만으로 VPN 구현 가능(Clientless VPN)<br>•뛰어난 사용성, 관리 편의성 (OS에 독립적)|
|단점|•적용 가능한 어플리케이션의 제한 (UDP 사용 제한)<br>•SSL 자체의 부하 (핸드 쉐이킹 지연, 암호화/복호화 지연)|

### 📌 계층별 VPN 기술 요약

| 계층  | 프로토콜  | 핵심 내용                                                                                                                                  | 쉬운 설명                                                     |
| --- | ----- | -------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------- |
| 2   | L2F   | •Layer 2 Forwarding<br>•원격지 ISP장비에서 터널 서버로 L2F 터널 생성                                                                                   | •최초의 터널링 프로토콜<br>•Cisco가 만든 원조격                           |
| 2   | PPTP  | •Point-to-Point Tunneling Protocol<br>•PPP의 확장, Client/Server 방식                                                                       | •MS가 만든 가장 단순한 VPN<br>•보안 취약해서 요즘 안 씀                     |
| 2   | L2TP  | •Layer 2 Tunneling Protocol<br>•PPTP + L2F 합친 것<br>•non-IP 환경도 지원, 멀티 프로토콜<br>•암호화 없음 → IPSec과 조합 필수<br>•X.25/ATM/Frame Relay/SONET 지원 | •PPTP와 L2F 장점만 합침<br>•근데 암호화가 없어서<br>•항상 IPSec이랑 같이 씀     |
| 2   | SSTP  | •MS가 개발, Windows Vista 배포<br>•SSL/TLS로 데이터 암호화                                                                                         | •HTTPS 포트(443) 써서 방화벽 잘 안 막힘                              |
| 2   | MPLS  | •Multi Protocol Label Switching<br>•IP 패킷에 Label 붙여 Label 스위칭으로 전송                                                                     | •패킷에 택배 운송장 붙이는 것<br>•라우팅 대신 라벨로 빠르게 전달                   |
| 3   | IPSec | •VPN 터널링 사실상 표준<br>•IPv6 기본 보안 프로토콜<br>•무결성/인증: AH<br>•기밀성: ESP                                                                        | •인터넷 구간 암호화의 표준<br>•AH는 서명만, ESP는 암호화까지<br>•L2TP랑 항상 짝꿍   |
| 4   | SSL   | •Client-Server 핸드셰이크 암호화<br>•인증: Handshake Protocol<br>•무결성: Record Protocol (압축 포함)                                                   | •우리가 매일 쓰는 HTTPS 기반<br>•브라우저만 있으면 VPN 가능<br>•별도 클라이언트 불필요 |
