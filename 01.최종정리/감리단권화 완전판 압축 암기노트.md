## 1️⃣ 정보시스템 감리 대상 (전자정부법 제57조, 시행령 제71조)

### ▶ 의무 감리 대상 [조건 1] - 하나라도 해당하면 의무

|구분|기준|
|---|---|
|정보시스템 구축사업|사업비 **5억 이상**|
|목적 기준|대국민 서비스·민원업무 처리용|
|공동 구축·사용|여러 중앙행정기관 등이 공동 구축·사용|
|EA·ISP 등|기관장이 감리 시행 필요성 인정 시|
|소액 (1억 미만)|감리효과가 높다고 기관장이 인정 시|

> ⭐ **사업비 = 총사업비 − HW·SW 단순 구입비용**

### ▶ 감리 생략 가능 [조건 2 - 공공PMO 수행 시]

사업비 **1억 이상~5억 미만** 전자정부사업으로 다음 중 하나:

- ① 대국민 서비스·민원업무 처리용
- ② 여러 기관 공동 구축·사용

또는 **사업기간 5개월 미만** 정보시스템 구축사업

→ PMO를 수행하면서 [조건 2]에 해당하는 경우 감리 생략 가능

### ▶ 예외 사항 [조건 3 - 기밀정보 취급기관]

아래 정보 취급 기관 → 기관장이 정보시스템 감리를 하게 할 수 있음  
(지정 시 감리법인 등록기준에 따른 기술능력·제정능력 강춘 기관으로 지정)

- 국방·외교·안보 등 국가안전보장에 관한 정보
- 사생활 보호 관련 개인정보
- 기타 기밀 유지·공신력 확보 필요 정보

### ▶ 감리 대상 여부 판단 절차

① 정보시스템 목적 기준 판단 (대국민서비스·공동구축사용)

- 해당 + 소액 아닌 경우 → 의무감리
- 해당 + 소액 → 효과 높다고 인정 시 의무감리 ② 정보시스템 구축사업 여부
- 사업비 5억 이상 → 의무감리 ③ EA·ISP·개발운영 서비스 → 기관장 필요성 판단

## 2️⃣ 감리법인의 업무범위·절차 (시행령 제72조)

### ▶ 업무 범위 (5가지)

1. 사업수행계획의 계약내용 반영 여부, 일정·산출물 계획의 적정성 검토·확인
2. 과업범위·요구사항의 설계 반영·구체화 여부 검토·확인
3. 과업 이행 여부 점검
4. 관련 법령·규정·지침 준수 여부 검토·확인
5. 감리기준에서 정하는 사항

### ▶ 업무 절차 (7단계)

1. 감리계약 체결
2. 예비조사 실시 및 감리계획 수립
3. 감리 착수회의 실시
4. 감리 시행 및 감리보고서 작성
5. 감리 종료회의 실시
6. 감리보고서 통보
7. 시정조치 결과 확인 및 통보

> 수행 형태에 따라 일부 절차 변경·생략 가능

### ▶ 기타 (전자정부법 제57조)

- 행정기관장은 사업자로 하여금 감리원 업무수행에 필요한 사항 지원, 정당한 사유 없이 개입·간섭 금지
- 행정기관장은 사업자로 하여금 시정조치 결과 보완·확보토록 의무화
- 감리 법인은 감리기준에 따라 적정하게 개발·구축되는지 점검

## 3️⃣ 감리법인 등록·변경·행정처분

### ▶ 등록 조건 (제58조, 시행령 제73조)

- **재정능력**: 자본금 **1억 이상**
- **기술능력**: 감리원 **5명 이상** (수석감리원 **1명 이상** 포함)
- 감리원 = 해당 법인에 **상시 근무** + 다른 감리법인 소속 **불가**

**결격 사유** (임원에 해당 시):

- 피성년후견인 또는 피한정후견인
- 등록취소된 감리법인 임원으로서 취소일부터 **2년** 미경과자 (취소 원인이 된 행위를 한 자·대표자)

### ▶ 변경 신고 (제58조, 시행령 제73조)

- **신고 시기**: 변경한 날부터 **3개월 이내** → 행정안전부장관
- **변경 신고 대상**:
    - 법인 명칭·상호, 대표자·임원
    - 감리원 변경 (등록기준 범위 **외** 변경)
    - 자본금 변경 (등록기준 범위 **외** 변경)
    - 사무실 소재지
    - 감리원 계속교육 이수사항 변경

### ▶ 결격 사유 확인 시점 (제61조, 시행령 제76조)

- 감리법인 등록신청을 받은 경우
- 감리법인 등록사항의 변경신고를 받은 경우
- 감리원증의 발급신청을 받은 경우

### ▶ 행정처분 기준 (제62조, 시행령 제77조)

**[일반기준]**

- 위반행위 2 이상 → 가장 무거운 처분 기준 적용 (업무정지는 합산 가능)
- 위반 횟수 기준 = 최근 **1년간** 같은 위반행위로 처분받은 경우
- 업무정지 시 고의·중과실·피해 규모 등 고려하여 **1/2 범위** 내 가중·감경

**[개별기준]**

| 위반행위                     | 1차     | 2차     | 3차 이상  |
| ------------------------ | ------ | ------ | ------ |
| 거짓·부정한 방법으로 등록           | 등록취소   |        |        |
| 최근 3년간 3회 이상 업무정지        | 등록취소   |        |        |
| 업무정지 기간 중 감리 수행          | 등록취소   |        |        |

## 4️⃣ 감리 인력 배치 (감리기준 제5조)

- 전체 투입공수의 **50% 이상** → 해당 감리법인 **소속 상주감리원**으로 배치

|유형|역할|자격 요건|
|---|---|---|
|**총괄감리원**|감리업무 총괄 능력·경험 보유|실제 감리 투입기간 **1년 이상**인 수석감리원 (필수) + 상주감리원 (필수)|
|**전문가**|총괄감리원 지원·자문|전체의 **30% 범위** 내에서 AI·클라우드·빅데이터·모바일·정보보호·법률·회계 전문가 배치 가능|
|**상주감리원**|상주감리 전담|수석감리원으로서 다음 중 하나: ①사업비 **20억 이상** 감리 **3회** 이상 참여, ②PM 또는 QA 경력 **3년 이상**, ③발주자가 적합하다고 인정|

> ⭐ 투입기간 1년 = 단계감리 **52주** = 상주감리 **1년**

## 5️⃣ 감리 대가 산정 (감리기준 제4조 - 별표 1)

### ▶ 기본 감리비 산정식

| 구분         | 산정식                                                                      |
| ---------- | ------------------------------------------------------------------------ |
| 보정 전 (3단계) | **0.9307** × IT감리 평균임금 × (사업비보정금액/FP단가)^**0.6385** × (1+제경비율) × (1+기술료율) |
| 보정 전 (2단계) | **0.8516** × IT감리 평균임금 × (사업비보정금액/FP단가)^**0.6385** × (1+제경비율) × (1+기술료율) |
| 보정 후       | 보정 전 기본감리비 × (1 + **난이도계수 합**)                                           |
| 상주·추가감리비   | (투입공수 × IT직무 평균임금) × (1+제경비율) × (1+기술료율)                                 |

- IT감리 평균임금: 소프트웨어 기술자 평균임금 중 **IT감리 전문임 평균** 적용
- FP단가: 한국소프트웨어산업협회 SW사업 대가기준율
- 직접경비: 여비·시험·진단도구 사용료·특수분야 전문가 자문비 등

### ▶ 감리대상사업비 보정비율

|구성항목|보정비율|
|---|---|
|SW 개발비·유지보수비·운영용역비|**1.000**|
|HW·SW 구입비·유지보수비|**0.456**|

|기타 전산설비·공사·이전·임차 관련|**0.000**|
|센서·단말장치·통신회선·전기 사용료|**0.000**|

## 6️⃣ 감리 절차 및 수행 가이드

### ▶ 3단계 감리 절차 흐름

```
[A00] 예비조사          [B00] 현장감리          [C00] 시정조치확인
감리계획서 작성·제출  →  점검 수행·보고서 작성  →  적합·조치 여부 확인
      ↓                        ↓                        ↓
  감리계획서              감리수행결과보고서          시정조치확인보고서
```

**전자정부법 시행령 제72조 ↔ 감리기준 ↔ 감리수행가이드 연계:**

- 단계1 (요구정의) → 제8조 → [1] 요구정의단계
- 단계2 (설계) → 제9조 → [2] 설계단계
- 단계3 (종료) → 제10조 → [3] 종료단계

### ▶ 감리 실시 시기

|유형|실시 시기|적용 기준|
|---|---|---|
|**3단계 감리**|요구정의·설계·종료|기본 원칙|
|**2단계 감리**|설계·종료|다음 중 **하나만** 만족해도 가능: ①사업비 **20억 미만**, ②사업기간 **6개월 미만**|
|**상주감리**|사업 전단계 또는 일부|대기업SW사업자 참여 제한 사업, 위험·난이도 높다고 판단되는 경우|

**추가 규정:**

- 발주자는 단계별 감리 외 **추가 감리** 가능
- 발주자는 요구사항정의서 과업내용 반영 여부 **직전 점검** 의무
- 상주감리 추가 시 → 요구정의 단계 감리 생략 가능 (상주감리원이 요구사항정의서 과업내용 반영 여부 직접 점검)

### ▶ 각 단계별 감리 핵심 내용

#### [요구정의단계 감리]

- **목적**: 계약문서 과업내용 → 요구사항정의서에 적정 반영 여부 점검
- **시기**: 분석단계 완료 시점 (일반적 개발방법론에서 별도 단계 아님)
- **사업자 역할**: 요구사항정의서 제출 + 세부 항목별 과업내용 반영 대비표 작성·발주자 확인 후 감리법인 제출 (감리 실시 전까지)
- **감리법인 역할**: 과업내용이 요구사항정의서에 적정 반영 여부 점검 → 결과를 감리수행결과보고서에 명시 → 발주자 제출

#### [설계단계 감리]

- **목적**: 설계 산출물의 과업내용 반영 여부 점검
- **시기**: 설계단계 완료 시점 (반복적 개발 시 중요도 높은 영역 포함, 반복 차수 정하여 수행)
- **사업자 역할**: 설계 산출물 제출 + 대비표 작성·제출
- **감리법인 역할**:
    1. 설계 산출물이 과업내용을 적정하게 반영하는지 여부
    2. 대비표가 세부 검사항목별 **적합·부적합 판정** 가능하도록 구체화되었는지 여부
- **보고**: 적합·부적합으로 명시하여 발주자 제출

> ⭐ **대비표**: 요구사항추적표 기반으로 요구사항ID 단위 → 설계내용·검사대상·검사기준 추적 문서  
> 기능요구사항 → 테스트 시나리오로 구체화 가능

#### [종료단계 감리]

- **목적**: 과업이행 여부 최종 점검
- **시기**: 사업자가 **통합시험 완료 후** + 시정조치 확인 기간이 검사 이전 종료 가능한 시점
- **감리법인 역할**: 검사 전까지 세부 검사항목별 과업내용 이행의 적합·부적합 점검 → 보고서 제출
- 과업이행 확인 = **통합시험·연계시험 관점**에서 수행

- 인증 대상·방식, 인증 수행 제한, 비밀번호관리, 중요자원 접근 통제, 암호키 관리
- 암호연산, 중요정보 저장·전송, 예외처리, 세션 통제

**주요 구현단계 보안약점:**

- SQL·코드·경로·OS명령어·LDAP·XML 삽입, XSS
- CSRF, 서버사이드요청위조, HTTP응답분할
- 취약한 암호화, 하드코딩된 중요정보
- Null Pointer 역참조, 버퍼오버플로우, 해제된 자원 사용

### ▶ 보안약점 진단 도구 인정 기준 (3가지 중 하나)

1. 국가보안기술연구소장이 인증한 도구 (「정보보호시스템 평가·인증 지침」)
2. 한국인터넷진흥원장이 확인한 도구 (「정보보호제품 성능평가 운영 지침」)
3. 국가정보원장이 고시한 「국가정보보안기본지침」에 따라 '보안기능확인서' 발급된 도구

### ▶ 상용소프트웨어 직접구매

- **대상사업**: 총사업규모 **3억 이상**
- **대상 SW**: 가격 **5천만원 이상** (부가세 포함) + 다음 중 하나:
    1. 국가종합전자조달 종합쇼핑몰·디지털서비스몰 등록 SW
    2. **GS(품질)인증** SW
    3. **CC(정보보호시스템) 인증** SW 또는 국가정보원 검증·지정 SW
    4. **NET(신기술)** 인증 SW
    5. **NEP(신제품)** 인증 SW
- 동일 SW 다량 구매로 5천만원 초과 → 5천만원 이상 SW로 **간주**
- 국가종합전자조달 종합쇼핑몰 등록 제품 → 가격과 **무관하게** 포함
- 직접구매 제외 가능:
    1. 기존 정보시스템과 통합 불가 또는 현저한 비용 상승 우려
    2. 사업기간 내 완성 불가할 정도로 지연 우려
    3. 분리발주로 현저히 비효율적인 경우

### ▶ 웹사이트 품질관리 7대 원칙 (2024.4 개정) ★

1. **호환성**: 웹표준, 다양한 브라우저·모바일 동등 서비스
2. **접근성**: 신체·인지 제약 없이 정보·기능 접근 보장
3. **개방성**: 특정 검색엔진 없이 공개 정보 이용 가능
4. **접속성**: 응답시간·속도·용량·링크 연결 최적 상태
5. **편의성**: 이용자 중심 디자인·기능 구현
6. **효율성**: 유사·중복 웹사이트 통합 또는 폐기

- 지식재산권이 사업자에게 귀속 시 → 발주기관의 계약목적 관련 **사용에 제한 불가**

    1. 무상 하자보수기간 경과 후 발생 하자
    2. 이미 구매한 물품·연동 제품 기반 추가 개발·구축 (사용방법·환경 개선 요구사항 추가 포함)

    4. 계약서 미명시 사용자 교육·기술지원

## 1️⃣5️⃣ 협상에 의한 계약체결 기준 (기획재정부예규 제721호, 2024.9.13.)

### ▶ 주요 용어

- **제안요청서**: 계약담당공무원이 입찰참가자에게 제안서 제출을 요청하기 위해 교부·열람하는 서류
- **제안서**: 입찰참가자가 제안요청서·입찰공고에 따라 작성·제출하는 서류
- **협상적격자**: 제안서 평가결과에 따라 협상 대상자 자격을 갖춘 자

### ▶ 입찰공고

- 원칙: 제안서 제출 마감일 전일부터 기산하여 **40일 전**
- 긴급·고시금액 미만·재공고입찰: **10일 전**까지

### ▶ 제안요청서 명시 사항 (6가지)

1. 과업내용, 요구사항
2. 계약조건
3. 평가요소·평가방법
4. 제안서 규격·제출방법·제본형태
5. 제안서 보상에 관한 사항
6. 사업자 준수사항 (기타 필요사항)

### ▶ 제안서 평가 배점 (2024.9.13. 개정)

|구분|배점한도|
|---|---|
|**기술능력평가**|**70점**|
|**입찰가격평가**|**30점**|

**기술능력평가 항목 (각 항목 최대 30점):** 기술지식능력, 인력조직관리기술, 사업수행계획, 지원기술사후관리, 재무구조경영상태, **상호협력**, **외주근로자 근로조건**, **원가절감의 적정성**

**유형별 조정 (20점 범위 내):**

|유형|기술 수준|기술:가격|
|---|---|---|
|**기술강조형**|고도의 전문성·기술성|**90:10**|
|**기술·가격균형형**|일정 수준 이상|**80:20~60:40**|
|**가격중시형**|상대적으로 덜 중요|**50:50**|

### ▶ 협상적격자 선정·순위

- 선정 기준: 기술능력평가 점수 ≥ 기술능력평가 배점한도의 **85%**
- 협상 순서: 합산점수 고득점 순 → 기술능력평가 점수 → 배점 큰 항목 순
- 협상적격자 통지 항목: 협상순위, 전원의 기술·가격 점수·합산점수, 협상일정

### ▶ 협상 방법·범위·기간

- 우선순위자와 협상 성립 시 → 다른 적격자와 협상 **실시하지 않음**
- 미성립 시 → 차순위로 협상 진행, 모두 결렬 시 **재공고 입찰** 가능
- 범위: 사업내용·이행방법·일정 등 (계약담당공무원은 사업과 무관한 요구 불가)
- 가격 기준: 해당 사업 예산 이하, 제안 가격으로 협상
- 협상 기간: 통보일로부터 **15일 이내** (5일 조정, 미합의 시 **10일 연장** 가능)

### ▶ 계약 체결

- 협상 성립 후 **10일 이내** 계약 체결

### ▶ 가격의 적정성 심사 (제7조의2)

- 기술평가 변별력 확보 필요 높은 계약 → 입찰공고 시 명시한 **기준금액 미만** 입찰자에 대해 원가절감 적정성 심사 가능

## 1️⃣6️⃣ SW 사업 대가산정 가이드

### ▶ 대가산정 시점 3단계

- **예산확보 단계**: 개괄적 사업비 산정
- **사업발주 단계**: 제안요청서·발주금액 산정
- **사후정산 단계**: 사전 산정 vs 집행 사업비 차이 파악·정산

### ▶ 기능점수(FP) 기반 적정 개발기간 산정

|사업규모(FP)|1인 생산성(FP/MM)|
|---|---|
|1,000 미만|**19**|
|1,000~2,000 미만|**22**|
|2,000~3,000 미만|**24**|
|3,000 이상|**22**|

> **1인 총 투입기간 = 사업규모 / 1인 생산성**  
> **전체 개발기간 = 1인 총 투입기간 / 투입인력 수**

### ▶ 기능점수 방식 개발비 계산 절차

**1. 사전준비**

- 기능 요구사항 (기능점수 산정 대상)
- 비기능 요구사항 (보정계수 평가 시 사용)
- 정통법 또는 간이법 결정

**2. 개발대상 SW 기능점수 산정**

- ①기능점수 측정범위 설정
- ②애플리케이션 경계 설정
- ③데이터 기능점수 측정 (ILF, ELF)
- ④트랜젝션 기능점수 측정 (EI, EO, EQ)
- ⑤개발대상 기능점수 = **데이터 기능점수 + 트랜젝션 기능점수**

**3~6. 보정 전 → 보정 후 → 이윤 → 개발비 산정**

- 보정 전 개발원가 = 기능점수 × 기능점수당 단가
- 보정 후 개발원가 = 보정 전 × 규모보정계수 × 연계복잡성 × 성능요구 × 운영환경호환성 × 보안성
- 이윤 = 개발원가의 **25% 이내**
- **SW 개발비 = 보정 후 개발원가 + 이윤 + 직접경비**

> ⭐ FP 방식 → **제경비·기술료 별도 산정 불필요** (단가에 포함)

### ▶ DET(데이터요소유형) 식별 기준 ★

- 경계를 넘나드는 **사용자가 인식 가능한 유일하고 반복되지 않는 속성** → 1개 DET
- **DET에서 제외**하는 것:
    - 보고서 제목·화면 ID·열 제목·속성 제목 같은 **리터럴**
    - 날짜·시간 속성 같은 **애플리케이션이 생성하는 스탬프**
    - 페이지 설정 변수·페이지 번호·위치정보
    - 이전/다음/처음/마지막 같은 **내비게이션 보조수단**
    - 트랜젝션에 의해 경계 내부에서 생성되어 내부에만 저장되는 속성
    - 내부논리파일·외부연계파일에서 검색되지만 경계를 넘지 않고 처리되는 속성

### ▶ FTR(참조파일유형) 식별 기준

- 트랜젝션 기능에 의해 **읽히거나 유지되는 ILF** 또는 **읽히는 ELF**
- 수정과 조회가 동시에 적용되는 ILF → **한번만** FTR로 식별

**Log in 식별 기준:**

- 단순 Valid Check만 → **EQ (외부조회)**
