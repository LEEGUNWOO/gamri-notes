## 📘 5. 컴퓨터 포렌식 (Computer Forensic)

### 📌 개념

|구분|내용|
|---|---|
|**개념**|•컴퓨터를 매개로 한 범죄행위에 대한 법적 증거자료 확보를 위해 디지털 자료를 수집·분석·보존하여 법적 증거력을 갖도록 하는 절차 및 방법<br>•추적과 복구를 용이하게 하기 위한 기법|

### 📌 포렌식의 특징

|특징|설명|
|---|---|
|**정당성의 원칙**|•적법한 절차를 거쳐 증거 획득<br>•위법 수집 증거는 법적 효력 상실|
|**재현의 원칙**|•피해 당시와 동일 조건에서 현장 검증 시 동일한 결과 도출|
|**신속성의 원칙**|•휘발성 정보수집 가능성은 신속한 조치에 의해 결정|
|**연계 보관성의 원칙 (Chain of Custody)**|•디지털 증거물 획득 → 이송 → 분석 → 보관 → 법정 제출의 각 단계 책임자 명시|
|**무결성의 원칙**|•획득한 디지털 증거가 위조·변조되지 않았음을 증명|

### 📌 포렌식 주요 기능

|구분|기능|
|---|---|
|**디지털증거 획득 및 무결성 검증**|•무결성 검증: MD5 Hash, CRC Check<br>•디스크 이미지 복제 및 압축|
|**파일 복구**|•삭제 또는 손상된 파일 복구|
|**고급 검색**|•파일명·String 검색, 이메일 데이터 검색<br>•웹 히스토리, 파일 슬랙 검색|
|**기타**|•변조 확장자 검색 (내용은 Doc인데 확장자 다른 경우)<br>•암호파일 검색|

### 📌 포렌식 요구 기능

|요구 기능|설명|
|---|---|
|**증거 보존 기능**|•시스템 기록 원본 손상 없이 유지<br>•원본 디스크 접속 시 쓰기 방지 조치<br>•원본 디스크 이미지 생성|
|**증거 수집/분석 기능**|•삭제 내용 복구<br>•디스크 저장 내용 검색·비교|
|**문자열 검색 기능**|•정보유출·불순한 내용 파일 기록 검색<br>•전산 감사 시 부정사용 내역 검색|

### 📌 디지털 포렌식 관련 용어

|용어|설명|비고|
|---|---|---|
|**HPA (Host Protected Area)**|•ATA4 표준 추가 디스크 예약영역, 포맷·삭제 시에도 데이터 보존 가능|•DCO (Device Configuration Overlays)|
|**MBR (Master Boot Record)**|•하드디스크 첫 섹터의 512바이트 영역, 부팅 정보 저장||
|**Slack space**|•파일 관리 시스템의 고정 분할로 발생하는 유효공간<br>•블록(클러스터) 중 일부 사용되지 않는 공간 ★2023기출|•Ram Slack, Drive Slack|
|**디가우징 (Degaussing)**|•자성 저장 매체의 완전 데이터 소거를 위해 강력한 자기장으로 자기력 History 제거||
|**웹 아티팩트**|•웹 서버와 통신하며 클라이언트에 생성된 흔적<br>•대부분 웹브라우저 로그|•임시 인터넷 파일<br>•Index.dat|
|**이미징**|•원본 저장매체 변조 방지를 위해 별도 매체에 블록 단위(File 단위 아님)로 복제|•메모리의 경우 DUMP 작업 수행|
|**라이브 리스판스 (Live Response)**|•휘발성 정보를 신속히 수집하기 위해 대상 시스템의 전원이 켜진 상태로 정보 수집||
