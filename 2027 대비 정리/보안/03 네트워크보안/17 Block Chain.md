## 📘 Block Chain

### 📌 개념

|구분|내용|
|---|---|
|개념|•P2P 기반의 공개 거래 장부. 원장거래(ex: 비트코인) 검증에 사용되며, 인증 및 분산화된 트랜잭션 검증에 활용|
|구조|•특정 개인에 의한 임의 조작이 불가능하도록 고안된 분산 DB의 한 형태<br>•지속적으로 증가하는 이전 Hash Tree 기반의 연결 리스트(Linked List) 자료구조|
|대표 암호화폐|•BitCoin, Ethereum|

### 📌 주요 개념

| 개념       | 설명                                                                                                                     |
| -------- | ---------------------------------------------------------------------------------------------------------------------- |
| **작업증명** | •Brute Force를 통해 조건에 맞는 nonce 값을 찾으면 신규 블록 생성 → nonce 찾은 사용자에게 보상<br>•블록체인 성격에 따라 **지분증명(Proof of Stake**) 적용하는 경우도 있음 |
| 거래합의     | •FT(Fault Tolerance) 기반 트랜잭션 승인은 노드 간 합의가 있을 경우 승인<br>•전체 노드의 특정수를 점유하지 않는 이상 트랜잭션 변조 불가능                              |
| 암호화 기술   | •거래 무결성 검증을 위해 Hash 암호화 기법(Hash tree) 사용<br>•사용자 인증 및 트랜잭션 승인을 위해 공개키 기반 암호화 기법 사용                                     |
| 스마트 컨트랙트 | •단순 트랜잭션 데이터뿐 아니라 거래 명세(Logic)를 프로그램화 하여 블록으로 관리                                                                       |
※ **nonce**: "Number used ONCE"의 약자. 블록 해시값이 특정 조건(ex: 0000으로 시작)을 만족할 때까지 무작위 대입하는 숫자. 조건에 맞는 nonce를 찾으면 블록 생성 성공 → 채굴 보상 획득. 이 과정이 **작업증명(PoW)** 이자 **채굴(Mining)**.

### 📌 Fork

|구분|내용|
|---|---|
|개념|•블록체인 네트워크에서 규격을 변경하여 새로운 규격을 사용하는 것을 분기(Fork)라 함|
|하드 포크|•규격을 변경하여 기존 블록체인과 영구적으로 분기되어 새로운 체인을 형성|
|소프트 포크|•규격은 변경하였지만 기존 블록체인과 호환성을 유지하여 기존 체인과 동시 사용 가능|

### 📌 블록체인 보안

| 구분               | 세부 내용                                                                                                                                                                             |
| ---------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 블록체인 보안 요건       | •공개 원장 정보이므로 **불특정 다수 접근 가능(Public blockchain 기준)**<br>•블록체인 무결성 보장 위해 **기록된 트랜잭션은 임의 삭제 불가**<br>•개인정보 등 **민감 정보 기록 시 가명처리·익명처리 등 보호조치 필요**<br>•**스마트 컨트랙트에 악의적 기능이 포함되지 않음을 보장** |
| 블록체인을 활용한 보안 서비스 | **•트랜잭션 데이터의 무결성을 검증하기 위한 보안 서비스<br>•분산 신원 증명(Decentralized Identifier, DID) 등**                                                                                                  |
### 📌 블록체인 공격기법

| 공격 기법           | 영어                 | 공격 대상            | 내용                                                                                                   |
| --------------- | ------------------ | ---------------- | ---------------------------------------------------------------------------------------------------- |
| 이중 지불 공격        | Double Spending    | **블록체인 거래**      | **•한 번 사용한 자산을 다시 한 번 더 사용하는 공격**<br>•합의 알고리즘이 각 트랜잭션을 처리하고 검증하는데 걸리는 시간 간격을 악용<br>•블록 확정 전 거래 취소 가능 |
| 재진입 공격          | Reentrancy Attack  | **==스마트 컨트랙트==** | •**스마트 계약 코드의 특정 거래가 합의·검증되는 시간 간격을 이용한 공격**<br>•잔고 차감 전에 재호출로 반복 출금 가능                              |
| **허위 정보 전파 공격** | **Eclipse Attack** | 노드 네트워크          | •공격자가 다수의 노드를 장악하여 **==특정 노드를 블록체인 네트워크로부터 고립==**시키는 공격<br>•잘못된 블록 정보 전달                             |
| 51% 공격          | 51% Attack         | **블록체인 합의**      | **•과반 해시파워 점유로 블록 조작<br>•거래 되돌리기 가능**                                                                |
| 시빌 공격           | Sybil Attack       | 노드 네트워크          | •공격자가 다수의 **가짜 신원/노드를 생성**하여 독립 참여자인 것처럼 행동<br>•네트워크 전체 신뢰 붕괴 목적                                     |

**혼동 포인트 정리:**

|구분|시빌 공격|이클립스 공격|
|---|---|---|
|수단|가짜 **신원/노드** 대량 생성|실제 노드로 피해 노드 **포위**|
|목적|네트워크 **전체** 신뢰 붕괴|**특정 노드 1개** 고립|
|피해 범위|네트워크 전체|타겟 노드만|
|키워드|가짜, 다수, 압도|고립, 피어 연결 편향, 포위|

**이중지불** — 블록체인 **거래 확정 타이밍**을 노리는 거야. 블록에 기록되기 전 그 짧은 순간에 같은 돈을 두 곳에 동시에 보내는 것.

**재진입** — 스마트 컨트랙트 **코드 실행 순서**를 노리는 거야. 출금 함수 실행 중에 잔고 차감이 되기 전에 다시 출금 함수를 호출해서 반복 탈취하는 것.

|구분|이중지불|재진입|
|---|---|---|
|대상|블록체인 거래 자체|스마트 컨트랙트 코드|
|원리|같은 코인을 두 군데 동시 전송|잔고 차감 전에 함수 재호출|
|비유|수표 두 장에 같은 계좌번호|ATM이 잔고 확인 전에 계속 돈 뱉어내는 것|
