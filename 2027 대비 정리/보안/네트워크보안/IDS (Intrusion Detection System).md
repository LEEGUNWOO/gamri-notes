## 📘 IDS (Intrusion Detection System)

### 📌 개념 및 주요 기능

|구분|내용|
|---|---|
|개념|•조직 IT시스템의 기밀성, 무결성, 가용성을 침해하고, 보안정책을 위반하는 침입사건을 사전 또는 사후에 감시, 탐지, 대응하는 보안 시스템<br>•컴퓨터 시스템의 비정상적인 사용, 오용, 남용 등을 실시간으로 탐지하는 시스템|
|보안 서비스|•기밀성·무결성: 비인가자 침입 탐지, 백도어 탐지, Sniffing, 바이러스, 내부자 불법 행위<br>•가용성: DOS공격, 시스템변경, 인터넷 웜 탐지|
|주요 기능|•저수준 필터링, 고수준 필터링을 이용한 감사자료 수집 가능<br>•오용탐지 기능, 비정상 침입탐지 기능<br>•서비스 거부 탐지기능, 자동침입 분류기능, 침입탐지 경보 및 대응, 시스템 취약성 점검 및 복구기능, 침입자 역추적<br>•새로운 침입 패턴 분석|

### 📌 IDS 분류 (1) - Information Source에 의한 분류

| 구분       | NIDS (Network based IDS)                                                                                    | HIDS (Host based IDS)                                                                                                            |
| -------- | ----------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| 동작       | •네트워크에 흐르는 패킷들을 검사하여 침입을 판단함                                                                                | •시스템상에 설치되어 사용자가 시스템에서 행하는 행위나 파일의 체크를 통해 침입을 판단함                                                                                |
| 자료원      | •promiscuous 모드로 동작하는 네트워크 카드나 스위치                                                                          | •시스템 로그, 시스템 콜, 이벤트 로그 ※ **파일 무결성(MD5 해시 체크섬)** 추가 가능                                                                            |
| 탐지 가능 공격 | •스캐닝, 서비스 거부 공격(DOS), 해킹                                                                                    | •부적절한 로그인 시도, 비인가자의 시스템 사용, 바이러스, 웜                                                                                              |
| 장점       | •네트워크 자원의 손실 및 패킷의 변조가 없음(캡처만 하기 때문)<br>•거의 실시간으로 탐지가 가능함<br>•감시 영역이 하나의 네트워크 서브넷으로써, HIDS에 비해 큼            | •침입의 성공여부 식별이 가능함<br>•실제 해킹 및 해킹 시도 판단이 용이<br>•주로 SW적으로 서버같은 시스템에 인스톨되며, 설치 및 관리가 간단함                                            |
| 단점       | •부가 장비가 필요함(스위치 등)<br>•Gbit 네트워크 환경에서는 정상적으로 작동하기 힘들 수 있음<br>•암호화된 패킷은 분석 불가<br>•False Positive가 높음(양치기 소년) | •감시 영역이 하나의 시스템으로 한정됨<br>•탐지 가능한 공격에 한계가 있음(주로 이벤트 로그로만 탐지)                                                                      |
| 대응 기술    | •방화벽/라우터/스위치와 연계하여 세션 종료<br>•방화벽/라우터/스위치등 네트워크 장비의 설정 변경<br>•관리자 통보(이메일, SMS, SNMP)                         | •Log Off, Lock Out User, 사용자 계정 Disable<br>•특정 응용프로그램이나 시스템의 셧다운, 프로세스 종료<br>•탐지된 시스템의 취약성 Correction<br>•관리자 통보(이메일, SMS, SNMP) |
| 한계       | •오탐으로 인해 정상적인 세션이 종료<br>•서비스 거부 공격(DOS)의 경우 대응이 불가능(탐지만 가능)<br>•능동적인 대응 기능 미비                               | •오탐으로 인해 정상적인 사용자가 자신의 계정을 사용할 수 없는 문제<br>•해킹 성공이후 프로세스 종료 등의 대응은 무의미                                                            |


| 구분    | 네트워크 기반 IDS (NIDS)                  | 호스트 기반 IDS (HIDS)                 |
| ----- | ----------------------------------- | --------------------------------- |
| 설치 위치 | 네트워크 구간 (스위치, 라우터 등)                | 개별 서버/PC 내부                       |
| 감시 대상 | 네트워크 트래픽 전체                         | 시스템 내부 활동 (로그, 파일, 프로세스)          |
| 탐지 방식 | 패킷 분석                               | 로그/파일 변경/시스템 호출 분석                |
| 장점    | 전체 네트워크 감시 가능  <br>실시간 트래픽 분석       | 내부 침입 탐지 가능  <br>암호화 트래픽 이후 분석 가능 |
| 단점    | 암호화 트래픽 분석 어려움  <br>내부 행위 상세 분석 어려움 | 각 호스트마다 설치 필요  <br>관리 부담 증가       |
| 탐지 범위 | 외부 공격, 네트워크 기반 공격                   | 내부 침입, 권한 상승, 파일 변조               |
| 성능 영향 | 네트워크 장비 성능 영향                       | 호스트 자원 사용                         |

### 📌 IDS 분류 (2) - 자료원 분석 방법에 의한 분류

| 구분       | 오용탐지 (Misuse)                                                  | 비정상탐지 (Anomaly)                                                             |
| -------- | -------------------------------------------------------------- | --------------------------------------------------------------------------- |
| 동작방식     | •시그니처(signature)기반 = Knowledge 기반                              | •프로파일(Profile)기반 = Behavior(행위) 기반 = Statistical(통계)기반                      |
| 침입 판단 방법 | •미리정의된 Rule에 매칭<br>•이미 정립된 공격 패턴을 미리 입력하고, 이것에 매칭됨             | •미리학습된 사용자패턴에 어긋남<br>•정상적, 평균적 상태를 기준으로 하여 이에 급격한 변화가 있을때 침입 판단             |
| 사용기술     | •패턴비교, 전문가시스템                                                  | •신경망, 통계적방법, 특징추출                                                           |
| 장점       | •빠른 속도, 구현이 쉬움<br>•사용자의 이해가 쉬움<br>•False Positive 가 낮음         | •알려지지않은 공격(Zero Day Attack)대응 가능<br>•사용자가 미리 공격 패턴을 정의할 필요 없음               |
| 단점       | •False Negative 큼<br>•알려지지 않은 공격 탐지불가<br>•대량의 자료를 분석 하는 데는 부적합 | •어떤 행위가 정상인지, 비정상인지 결정할 수 있는 임계치를 정하기 힘듦<br>•False Positive 가 큼<br>•구현이 어려움 |

> **False Positive**: 공격 아닌데도 공격이라 오판 하는 것 — **2024년 기출** **False Negative**: 공격인데도 공격이 아니라 오판 하는 것

### 📌 IDS 분류 (3) - 대응방법에 따른 분류

|구분|설명|
|---|---|
|수동적 대응 (Passive Response)|•IDS는 관리자에게 침입 정보만 제공<br>•실제 대응행동은 제공된 정보를 기초로 관리자가 수행<br>•종류: 알림 및 경보 등 (예: E-Mail, SMS)|
|능동적 대응 (Active Response)|•실제 대응행동을 IDS가 자동적으로 수행<br>•종류: 환경변경, 침입자에 대한 역공격 등 (예: Firewall과 연동, 세션차단)|

### 📌 방화벽 vs IDS 비교

|구분|방화벽 (FIREWALL)|침입탐지 (IDS)|
|---|---|---|
|목 적|•접근통제 및 인가, 방어|•실시간 탐지, 분석, 대응|
|위치|•외부와 내부 경계에서 외부 출입자 방어(수위실)|•내부에서 불법사용자 감시(CCTV)|
|운영 정책|•명백하게 허용하는 것만 허용|•명백하게 금지하는 것만 금지|
|장 점|•엄격한 Access Control<br>•인가된 트래픽만 허용<br>•보안정책 적용가능|•실시간 침입탐지<br>•통계적 분석 및 리포팅<br>•사후분석 대응기술|
|단 점|•Dial-Up등은 방화벽 통과 많음<br>•다양한 접근통제는 트래픽 부하 유발<br>•바이러스 침투나 암호화된 해킹 프로그램 차단불가|•변형된 침입탐지 패턴 취약<br>•트래픽 양이 떨어지는 경우 탐지율 낮음<br>•오판율이 높게 나타남|

### 📌 Snort (NIDS 대표 도구)

|구분|내용|
|---|---|
|개념|•패킷 수집 라이브러리인 libpcap에 기반한 실시간 트래픽 분석과 IP 네트워크 상에서 패킷 로깅이 가능한 네트워크 침입탐지시스템(N-IDS)|

#### 🔥 룰 구조 - HEADER

| action | protocol | 출발지 IP        | 출발지 PORT | 패킷 흐름 | 목적지 IP    | 목적지 PORT | OPTION |
| ------ | -------- | ------------- | -------- | ----- | --------- | -------- | ------ |
| Alert  | tcp      | $EXTERNAL_NET | any      | ->    | $HOME_NET | 21       |        |

#### 🔥 HEADER 옵션

|값|설명|구분|
|---|---|---|
|Alert|•Alert(경고+로그), log(로그), pass(무시), dynamic(로그), activate(경고+dynamic규칙 활성화)|Action|
|tcp|•tcp, udp, ip, icmp|프로토콜|
|$EXTERNAL_NET|•(default: any)|출발지 IP|
|any|•any, 포트번호|출발지 PORT|
|->|•-> 단방향, <> 양방향|패킷 흐름|
|$HOME_NET|•환경 변수에 정의된 로컬IP, 직접 IP주소 입력해도 무관|목적지 IP|
|21|•any, 포트번호|목적지 PORT|

#### 🔥 룰 구조 - OPTION

**OPTION 예시:** `(msg:"tcp syn flooding"; content:"GET / HTTP1.1"; flags:S; threshold:type threshold, track by_dst, count 10000, seconds 2; sid:1000000;)`

|옵션|설명|예시 해석|
|---|---|---|
|msg|•경고에서 출력할 메시지|•"tcp syn flooding" → 경고 시 출력할 메시지|
|content|•패킷 페이로드 내 문자열/바이너리 패턴 비교|•"GET / HTTP1.1" → 해당 문자열 포함 패킷 탐지|
|nocase|•content 검색 시 대소문자 구분 안 함|•content:"GET"; nocase; → 대소문자 무관하게 GET 탐지|
|offset|•패킷에서 문자열 검색 시작 위치 지정|•offset:0; → 페이로드 첫 번째 바이트부터 검색 시작|
|depth|•패킷 페이로드에서 특정 깊이 안에서 패턴 검색|•depth:4; → 페이로드 앞 4바이트 안에서만 패턴 검색|
|dsize|•패킷 페이로드 크기 확인, 버퍼 오버플로우 탐지에 활용|•dsize:300<>400; → 페이로드 크기가 300~400바이트인 패킷만 탐지|
|flags|•TCP 플래그 ex> S=SYN, A=ACK, F=FIN, R=RST|•flags:S; → SYN 플래그 패킷만 탐지|
|threshold|•track by_src(출발지 기준), track by_dst(목적지 기준)<br>•type: threshold(패킷발생량), limit(임계시간), both(최초 패킷만 기록)|•type threshold, track by_dst, count 10000, seconds 2; → 2초간 동일 목적지로 10,000개 이상 패킷 발생 시 탐지|
|flow|•TCP 세션 상태 및 방향 지정|•flow:to_server,established; → 서버 방향으로 이미 연결된 세션만 탐지|
|sid|•규칙 고유 ID|•sid:1000000; → 이 룰의 고유 ID|
|rev|•룰 수정 횟수, 수정 시 숫자 증가|•rev:1; → 최초 작성, rev:2; → 1회 수정된 룰|
