## 📘 9. 무선 네트워크 보안

### 📌 무선 LAN 서비스 흐름

|단계|내용|
|---|---|
|①|•PDA/노트북 → AP(Access Point): 서비스 요청|
|②|•AP → 인증서버: 인증 요청|
|③|•인증서버 → AP: 사용자 인증|
|④|•AP → PDA/노트북: 서비스 제공|
|구간|•무선(PDA↔AP): WEP/WPA/WPA2/WPA3 등<br>•유선(AP↔인증서버): RADIUS 등|

### 📌 무선 LAN 보안기술 진화
| 구분   | 내용                                                                                                                                            |
| ---- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| SSID | Service Set Identifier  <br>무선랜에서 네트워크 이름을 의미하며 기본적인 접속 제어 기능 제공  <br>AP가 비콘 신호에 SSID를 포함하여 브로드캐스트  <br>SSID만으로는 보안 기능이 약하며 접속 허용/차단 기준으로는 취약 |
| WEP  | Wired Equivalent Privacy  <br>무선 데이터 보안을 위한 초기 암호화 방식  <br>대칭키 구조 사용  <br>RC4 (40bit key) 암호 알고리즘 사용  <br>취약점 존재로 현재 사용 비권장                   |
| WPA  | Wi-Fi Protected Access  <br>WEP 보완 기술  <br>RC4 (128bit key) 사용  <br>TKIP 적용하여 동적 키 사용(무결성: Michael 알고리즘  *※MIC*)                              |
| WPA2 | AES-CCMP 암호화 사용  (TKIP도 지원)<br>인증 방식: IEEE 802.1X (기업) / PSK (개인)  <br>                                                                       |
| WPA3 | SAE 인증 방식 적용 (PSK 개선)  <br>GCMP-256(128) 암호화 사용  <br>PMF (Protected Management Frames) 적용                                                     |
※ PSK (Pre-Shared Key): 사전에 공유된 비밀번호(Wi-Fi 비밀번호)로 인증. 비밀번호 알면 누구나 접속 가능하고 오프라인 사전공격에 취약.
※ SAE (Simultaneous Authentication of Equals): WPA3에서 PSK 대체. 비밀번호가 같아도 매번 다른 키 생성(Dragonfly 핸드셰이크). 오프라인 사전공격 차단, PFS 지원. 
**비밀번호로 접속은 동일하나, 비밀번호 탈취 시에도 과거 트래픽 복호화 불가.**

##### 인증구분

| 구분      | 의미                   |
| ------- | -------------------- |
| WPA-PSK | 모두가 같은 와이파이 비밀번호를 공유 |
| WPA-EAP | 사람마다 자기 계정으로 인증      |

### 📌 WIPS (Wireless Intrusion Prevention System)

|항목|내용|
|---|---|
|정의|•인가되지 않은 무선단말기의 접속을 차단하고<br>•Rogue AP 및 취약한 AP를 탐지하는 솔루션|
|구성|•1개의 관리서버와 다수의 센서로 구성|
|센서|•AP, 모바일 단말기 탐지 및 모니터링|
|관리서버|•미리 등록된 Whitelist 및 RSSI 신호세기 등을 통해<br>•정상/미인가 무선장치 분류|

### 📌 참고용어

|용어|내용|
|---|---|
|Rogue AP|•공개된 무선 AP와 동일한 SSID를 사용하여<br>•모바일 단말기의 접속을 유도 후 패킷 스니핑을 통해 사용자 정보를 탈취하는 Access Point<br>•(Evil twin 공격에 사용)|
|War Driving|•무선 AP 탐지 장비를 이용하여 주변 트래픽을 도청|

### 📌 블루투스 보안

|항목|내용|
|---|---|
|규격|•IEEE 802.15.1<br>•경량형, 저전력으로 휴대폰, 랩탑 PC 등과 같은 휴대장치, 네트워크 접속점, 여타의 기기들과 10~100m 정도의 짧은 영역에서 연결을 제공하는 무선 인터페이스 규격|
|블루스나핑(BlueSnarfing)|•블루투스 모바일 기기의 개인 정보를 열람 및 탈취|
|블루프린팅(Blueprinting)|•서비스 발견 프로토콜(SDP)을 이용하여<br>•블루투스 디바이스를 탐색|
|블루버깅(BlueBugging)|•블루투스 통신을 통해 모바일 기기를 제어(Control)|
|블루본(BlueBorne)|•운영체제의 높은 권한을 가진 블루투스 프로세스를 공격하여<br>•권한을 획득하는 공격|
|블루재킹(Bluejacking)|•블루투스 통신 기능을 악용하여<br>•스팸 등 대량의 메시지를 발송하는 공격|
