## **📘** **공개키 암호화 (비대칭키 암호화)**

**개념**
- 암호화 키와 복호화 키가 다른 암호화 방식

**등장 배경**

| 문제       | 설명                                    |
| -------- | ------------------------------------- |
| 키관리 문제   | 비밀키의 배분, 공유문제, 수많은 키의 저장, 관리 문제       |
| ==**인증**==   | ==**메시지의 주인(작성자 또는 발신자)을 인증할 필요**==       |
| ==**부인방지**== | ==**메시지의 주인(작성자 또는 발신자)이 아니라고 부인함을 방지**== |

**특징**
- 모든 사용자는 항상 (개인키=사설키, 공개키) 쌍을 가짐
- 개인키는 외부 유출없이 안전하게 개인이 저장, 공개키는 외부에 공개해도 됨
- 암호 효율과 강도는 대칭키 암호화에 비해 현격하게 떨어짐

**동작 방식**
- Alice가 Bob의 공개키로 평문을 암호화 → 전송 → Bob이 자신의 개인키로 복호화


### 📌**공개키 암호화의 두 가지 모드**

| 구분      | 암호 모드                                                   | 인증 모드                        |
| ------- | ------------------------------------------------------- | ---------------------------- |
| **송신자** | **수신자의 공개키로 암호화**                                       | **송신자의 개인키로 암호화**            |
| **수신자** | **수신자의 개인키로 복호화**                                       | **송신자의 공개키로 복호화**            |
| **목적**  | **기밀성 (주로 키 교환 용도)**                                    | **인증, 부인방지**                 |
| 특징      | **메시지는 키 같은 작은 메시지, 대용량 메시지 암호화 아님<br>(속도 대칭키에 비해 느림)** | **누구나 공개키로 복호화 가능 → 송신자 인증** |


### 📌**공개키 암호화 기본 알고리즘**

| 알고리즘     | 특징                                                     | 수학적 배경        | 장점                               | 단점                              |
| -------- | ------------------------------------------------------ | ------------- | -------------------------------- | ------------------------------- |
| DH       | 최초의 공개키 알고리즘, 키 분배 전용                                  | 이산대수문제        | 키분배에 최적화, 키는 필요시에만 생성·저장 불필요     | 암호모드 사용 불가(인증 불가), <br>위조에 취약   |
| **RSA**  | **1978 개발, ITU·ANSI 등 대부분의 공개키 암호화 표준, 상용으로 가장 많이 사용** | **소인수분해 어려움** | **특허 2000년 만료, 여러 라이브러리 존재**     | **컴퓨터 속도 발전으로 키의 길이 점점 증가**     |
| DSA(DSS) | NIST 개발, 전자서명 알고리즘 표준                                  | 이산대수문제        | 간단한 구조(Yes or No 결과만 가짐)         | 전자서명 전용, <br>암호화·키교환 불가         |
| **ECC**  | **짧은 키로 높은 암호 강도, <br>PDA·스마트카드·핸드폰**                  | **타원곡선**      | **오버헤드 적음, 163키 = RSA 1024키 강도** | **키테이블(20K byte)가 필요함 (구현 복잡)** |

### 📌**공개키 암호화 vs 대칭키 암호화 알고리즘 비교**

| 구분      | 대칭키 암호화                     | 공개키 암호화                |
| ------- | --------------------------- | ---------------------- |
| 키 개수    | 1개 [전체 : N(N-1)/2]          | 2개 [전체 : 2N]           |
| 키 보관    | 개인이 비밀리에 보관                 | 개인키는 비밀 보관, 공개키는 외부 공개 |
| 키 교환    | 어렵고 위험함                     | 매우 쉬움                  |
| 키 길이    | 128, 256bit (작음)            | 512, 1024, 2048bit (큼) |
| 암복호화 속도 | 매우 빠름                       | 매우 느림 (100~1000배 이상)   |
| 평문 길이   | 대용량 가능                      | 대용량 어려움 (시간, 연산 제약)    |
| 기밀성     | ✅                           | ✅                      |
| 인증      | ❌                           | ✅                      |
| 부인방지    | ❌                           | ✅                      |
| 주 사용처   | 일반 평문 암호화, 네트워크 통신 데이터 암복호화 | 키 교환, 메시지 인증(전자서명)     |
