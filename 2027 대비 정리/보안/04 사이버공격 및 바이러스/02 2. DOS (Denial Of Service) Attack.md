## 📘 2. DOS (Denial Of Service) Attack

### 📌 개념 및 특징

|구분|내용|
|---|---|
|**개념**|•공격자가 호스트의 HW·SW를 무력화하여 적법한 사용자의 서비스 요구를 거부하게 만드는 행위<br>•Victim의 가용성을 저해|
|**가용성 저해**|•루트 권한 획득이 목적이 아님<br>•데이터 파괴·변조·훔치기가 목적이 아님|
|**추적·대응 어려움**|•공격자 추적 어려움<br>•매우 다양한 공격 방법 존재 (Null Routing, 사이버대피소 등)|
|**기타**|•다른 공격의 사전 공격으로 활용 가능<br>•비악의적 사용자 실수로도 발생 가능|

### 📌 DOS Attack 유형

|유형|내용|
|---|---|
|**내부 공격**|•시스템 리소스를 점유·고갈시켜 마비<br>•내부 사용자에 의해 발생, 주로 실수|
|**외부 공격**|•악의적 목적으로 특정 시스템 공격<br>•특정 프로세스·네트워크 기능 마비<br>•대부분의 DOS 공격에 해당|

### 📌 IP Fragmentation Attack

|구분|내용|
|---|---|
|**개념**|•MTU보다 패킷이 크면 분할(fragmentation) 필요 → 이를 비정상적으로 수행하게끔 패킷 위조|
|**종류**|•Ping of Death: 규정 이상 크기의 IP 패킷 전송 → OS 마비<br>•Tiny Fragmentation: 최초 패킷을 헤더길이보다 작게 보내 방화벽 우회<br>•Tear Drop: 패킷 순서를 비정상적으로 만들어 재조합 시 에러 발생 → 서비스 중지|

### 📌 SYN Flooding

|구분|내용|
|---|---|
|**개념**|•Victim의 TCP 네트워크 서비스를 불능으로 만드는 DOS 공격의 대표적 예|
|**공격방식**|•TCP 3-way Handshaking 취약성 이용<br>•Victim을 계속 half-open 상태로 만들어 다른 TCP 요청 처리 불가|
|**대응**|•서버 백로그 큐 크기 증가 또는 Connection Time-Out 시간 줄임<br>•패킷 필터링 사용|

### 📌 ICMP Smurf Attack

|구분|내용|
|---|---|
|**개념**|•IP Broadcast 주소 방식과 ICMP 패킷을 이용한 서비스 거부 공격|
|**공격방식**|•다수 호스트 존재하는 서브넷에 ICMP Echo 패킷을 Broadcast 전송 (Source는 Victim IP로 위조)<br>•대량의 응답 패킷이 Victim으로 집중 → 마비|
|**대응**|•라우터에서 ICMP Broadcast 금지|

### 📌 기타 DOS 공격 유형

|유형|내용|
|---|---|
|**Ping of Death**|•ICMP 패킷을 정상보다 아주 크게 만들어 전송 → 라우팅 과정에서 작은 조각으로 분할 → 재조합 시 과부하|
|**Land Attack**|•출발지·목적지 IP를 동일하게 설정 → 패킷 받은 호스트가 루프 상태에 빠져 IP 프로토콜 스택 장애 유발|
|**Teardrop Attack**|•헤더가 조작된 IP 패킷 조각 전송 → 데이터 겹치거나 누락 → 재조합 시 부하 발생|
|**Bonk**|•최초·두번째·세번째 패킷의 시퀀스 번호를 모두 동일하게 조작하여 전송|
|**Boink**|•정상적으로 시퀀스 번호 증가하다가 중간에 특정 패킷의 시퀀스 번호를 동일하게 중첩하여 전송|

> ※ **Bonk vs Boink**: 둘 다 시퀀스 번호 조작 공격. Bonk는 처음부터 전부 같은 번호, Boink는 중간에 특정 패킷만 중첩.

### 📌 DDOS (Distributed DOS)

|구분|내용|
|---|---|
|**개념**|•분산 네트워크 환경에서 다수의 호스트를 점령하여 DOS 공격<br>•공격자 위치·경로 추적 어려움|
|**공격방식**|•공격자 → 마스터 공격명령 → 에이전트 공격명령 → 수많은 에이전트가 Victim에 DOS 수행|
|**구성요소**|•공격자(Attacker): 공격 주도하는 해커 PC<br>•마스터(Master): 공격자 명령을 받아 에이전트 관리<br>•핸들러(Handler): 마스터에 설치되는 프로그램 (클라이언트)<br>•에이전트(Agent): 실제 공격 수행 시스템<br>•데몬(Daemon): 에이전트에 설치되는 프로그램|

### 📌 Botnet

|구분|내용|
|---|---|
|**개념**|•트로이 목마·악성 소프트웨어로 빼앗은 다수의 좀비 PC로 구성되는 네트워크<br>•감염된 호스트 = Bot, Bot들의 네트워크 = BotNet<br>•IRC채널·HTTP 프로토콜로 명령 지시|
|**구성**|•Bot Master → C&C Server(Command & Control) → Zombie PC|
