## 📘 SET (Secure Electronic Transaction)

### 📌 개념 및 특징 
![[Pasted image 20260613111336.png]]

|구분|내용|
|---|---|
|개념|•1996년 Visa·Master 카드사 합의로 만든 전자결제 보안 프로토콜|
|특징|•전자서명·인증서 기반 안전한 거래<br>•공개키 기반구조(혼합 암호화) 적용<br>•신용카드 지급결제에만 특화<br>•SSL 대비 속도 느림, 인프라 구축 복잡|

### 📌 보안 기능

|구분|내용|
|---|---|
|암호화|•대칭키 암호화 사용<br>•대칭키 분배는 공개키(RSA-1024bit) 사용|
|전자증명|•X.509 형식 전자증명서로 구매자·상인 신원 증명|
|전자서명|•해시함수로 무결성 확인<br>•거래 당사자 모두 서명하는 이중서명 방식|

### 📌 이중서명

|항목|내용|
|---|---|
|흐름|•구매정보 Hash → 해시값 A<br>•지불정보 Hash → 해시값 B<br>•A+B 합산 후 재Hash → 최종 해시값<br>•최종 해시값을 사용자 개인키로 암호화|
|목적|•상점: 지불정보(카드번호 등) 못 봄<br>•은행: 구매 물건 내용 못 봄<br>•결제 금액 정확성은 양쪽 모두 확인 가능|
