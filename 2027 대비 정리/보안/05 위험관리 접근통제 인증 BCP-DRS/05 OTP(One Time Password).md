## 📘 OTP(One Time Password)

### 📌 OTP의 개념

일회용 비밀번호로 한 번 사용된 OTP는 다시 사용할 수 없으며 같은 비밀번호가 생성되지 않는 인증 수단이다.

### 📌 OTP 특징

#### 🔥 이중요소 인증수단

•OTP를 사용하기 위해서는 OTP 생성매체(첫번째 인증요소, OTP Token)가 필요 •전자금융서비스 이용자는 본인이 가진 OTP 생성매체에 의해 생성된 비밀번호(두번째 인증요소)로 사용자 인증을 받음 ⇒ 보안 수준 높음

#### 🔥 동적 비밀번호

•OTP 생성매체에 의해 필요한 시점에 발생되고 매번 다른 번호로 생성되는 동적 비밀번호 •한번 생성되어 사용되면 다시 사용할 수 없어 중간에 해킹을 당해도 사용 불가 •생성 방식에 따라 사용자나 은행이 OTP 생성 어플리케이션을 관리해야 하는 부담

#### 🔥 OTP 생성 매체 필요

•OTP를 생성할 수 있는 기능을 가진 장치(OTP 토큰) 필요

### 📌 동기식, 비동기식 OTP

#### 🔥 동기식 방식과 비동기식 방식의 비교

| 구분      | 동기식                                                            | 비동기식                                                                                       |
| ------- | -------------------------------------------------------------- | ------------------------------------------------------------------------------------------ |
| 동작      | •미리 정한 고정된 시간간격 주기로 새로운 난수값을 생성<br>•난수값을 이용 사용자와 서버는 패스워드를 공유함 | •서버는 난수를 발생하여 시도값으로 사용자에게 전달<br>•사용자는 시도값을 암호화하여 서버에 응답값으로 전달, 서버는 응답값을 복호화하여 사용자가 옳은지를 인증 |
| 주요 매커니즘 | **•시간 또는 이벤트 동기화**                                             | •시도-응답방식(challenge-response)                                                               |
| 장점      | •속도 빠름(간단한 절차)                                                 | •안전성이 매우 높다(양방향 인증)                                                                        |
| 단점      | •사용자→서버 단방향 인증                                                 | •복잡한 절차를 통한 인증으로 속도 느림                                                                     |

#### 🔥 동기식 OTP

•토큰은 Seed(시간), 사용자의 비밀키에 기초하여 패스워드 생성 •사용자는 자신의 PIN(ID)와 패스워드를 입력하여 서버에 로그인함 •서버는 사용자의 PIN을 참고하여 저장된 사용자의 비밀키를 검색하여 꺼냄 •서버는 사용자의 비밀키와 Seed(시간)으로 패스워드 생성 •생성된 패스워드가 사용자가 입력한 패스워드와 같다면 사용자를 인증함

#### 🔥 비동기식 OTP

•사용자는 서버에게 PIN을 전달함 •서버는 난수를 생성하여 이 값을 시도값으로 사용자에게 전달함 •사용자는 받은 시도값을 자신의 비밀키로 암호화 한후 이 값을 응답값으로 서버에 전달함 •서버는 사용자의 PIN을 참고하여 사용자의 비밀키를 검색하여 꺼냄 •서버는 초기의 시도값을 사용자의 비밀키로 암호화 함, 이 암호화된 값이 응답값과 같다면 사용자를 인증함

#### 🔥 [참고] S/KEY

벨 연구소(Bell Lab.)에서 개발한 방식으로 유닉스 계열 운영체제의 인증에 사용된다. 클라이언트가 생성한 난수값을 서버에 전송한 뒤 해당 난수값을 입력값으로 해쉬값을 생성한다. **해시체인(Hash Chain)방식으로** 계속해서 다수의 해시값을 만들고, 인증을 수행 시 최초 해시값부터 차례로 하나씩 사용하는 OTP 방식이다.
