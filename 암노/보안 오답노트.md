# 🔐 SSL / TLS 정리

|구분|내용|
|---|---|
|SSL|Netscape가 만든 초기 보안 프로토콜 (현재 폐기)|
|TLS|SSL의 후속 표준, 현재 인터넷 보안 통신 표준|
|목적|기밀성(Confidentiality), 무결성(Integrity), 인증(Authentication)|
|적용|HTTPS, SMTP, IMAP, FTPS 등|
|동작 구조|TCP 위에서 동작하는 보안 계층|

참고: TLS 정의 (IETF RFC 5246, RFC 8446)

---

# 📚 SSL → TLS 버전 흐름

|프로토콜|상태|
|---|---|
|SSL 2.0|폐기|
|SSL 3.0|폐기|
|TLS 1.0|구버전|
|TLS 1.1|구버전|
|TLS 1.2|널리 사용|
|**TLS 1.3**|최신 표준|

---

# 🧩 TLS 동작 구조

HTTPS 연결 순서

```text
1. TCP 3-Way Handshake
2. TLS Handshake
3. 암호화된 데이터 통신
```

---

# ⚙️ TLS Handshake 핵심 기능

|기능|설명|
|---|---|
|서버 인증|인증서를 통한 서버 신원 확인|
|알고리즘 협상|암호 알고리즘 선택|
|키 교환|세션키 생성|
|세션 생성|대칭키 기반 통신 시작|
# 📊 TLS Handshake 핵심 흐름

|단계|메시지|설명|
|---|---|---|
|1|ClientHello|클라이언트가 지원하는 암호 알고리즘, TLS 버전 전송|
|2|ServerHello|서버가 사용할 암호 알고리즘 선택|
|3|Certificate|서버 인증서 전송|
|4|Key Exchange|키 교환 정보 전달|
|5|Finished|세션키 생성 완료|

---

# 🔄 TLS Handshake 전체 흐름

Client                        Server  
   │                             │  
   │------- ClientHello -------->│  
   │                             │  
   │<------ ServerHello ---------│  
   │<------ Certificate ---------│  
   │<------ ServerHelloDone -----│  
   │                             │  
   │------ Key Exchange -------->│  
   │------ Finished ------------>│  
   │                             │  
   │<------ Finished ------------│  
   │                             │  
      🔒 이후부터 암호화 통신

|기술|역할|
|---|---|
|전자서명|서버 인증|
|공개키 암호|키 교환|
|대칭키 암호|데이터 암호화|
|해시 / MAC|무결성 검증|

# 🔑 TLS에서 사용되는 암호 기술

|기술|역할|
|---|---|
|대칭키 암호|데이터 암호화 (AES, ChaCha20)|
|공개키 암호|키 교환 / 인증|
|해시|무결성|
|MAC / AEAD|메시지 인증|

---

# 🚀 TLS 1.3 특징 (시험 핵심)

|특징|설명|
|---|---|
|핸드셰이크 단축|2 RTT → **1 RTT**|
|0-RTT 지원|이전 세션 재사용 시 즉시 데이터 전송|
|취약 알고리즘 제거|RSA Key Exchange, SHA-1 등 제거|
|Forward Secrecy 기본|Diffie-Hellman 기반 키 교환|
|암호 스위트 단순화|AEAD 기반 암호만 사용|

---

# 🔒 TLS 1.3 사용 암호 방식

대표 cipher suite

```text
TLS_AES_128_GCM_SHA256
TLS_AES_256_GCM_SHA384
TLS_CHACHA20_POLY1305_SHA256
```

구조

```text
대칭키 암호 + AEAD + 해시
```

# 🔄 TLS 1.2 vs TLS 1.3 차이

|구분|TLS 1.2|TLS 1.3|
|---|---|---|
|Handshake|2 RTT|1 RTT|
|0-RTT|없음|지원|
|Key Exchange|RSA / DH|**ECDHE만 사용**|
|암호 알고리즘|다양|단순화|
|보안성|높음|**더 높음**|

→ Diffie-Hellman 사용

암호화 + 무결성 동시 제공


# 🔥 VPN 시험 핵심 비교

|구분|IPSec VPN|SSL VPN|
|---|---|---|
|계층|네트워크 계층|전송 계층|
|설치|장비 중심|클라이언트/웹|
|기업 간 연결|적합|제한적|

# 🧠 시험에서 자주 묻는 것

- IPSec의 AH vs ESP 차이
    
- 터널 모드 vs 전송 모드
    
- SSL VPN은 TLS 기반
    
- L2TP 단독 사용 취약
    

---

# 📌 최종 암기 블록

```text
VPN = 터널링 + 암호화 + 인증
IPSec = AH + ESP + IKE
IPSec 모드 = 전송 / 터널
SSL VPN = TLS 기반
```

# 블록암호 운용모드 비교 (시험용)

| 모드                              | 구조 / 원리               | 특징                      | 오류 전파     | 병렬 처리   |
| ------------------------------- | --------------------- | ----------------------- | --------- | ------- |
| **ECB** (Electronic Codebook)   | 블록을 **독립적으로 암호화**     | 동일 평문 → 동일 암호문 (패턴 노출)  | 1블록만 영향   | 가능      |
| **CBC** (Cipher Block Chaining) | 이전 **암호문과 XOR 후 암호화** | 가장 오래된 방식, 초기벡터(IV) 필요  | 2블록 영향    | 복호화만 가능 |
| **CFB** (Cipher Feedback)       | 이전 **암호문을 암호화하여 XOR** | 스트림처럼 동작                | 일부 비트 영향  | 불가      |
| **OFB** (Output Feedback)       | 이전 **출력값을 암호화하여 XOR** | 오류 전파 거의 없음             | 동일 비트 위치  | 가능      |
| **CTR** (Counter Mode)          | **카운터 암호화 후 XOR**     | 매우 빠름, 병렬 처리 가능         | 해당 블록만 영향 | 가능      |
| **GCM** (Galois/Counter Mode)   | **CTR + GHASH 인증**    | ==기밀성 + 무결성 제공 (AEAD)== | 블록 영향     | 가능      |
| **CCM** (Counter with CBC-MAC)  | **CTR + CBC-MAC 인증**  | ==기밀성 + 무결성 제공 (AEAD)== | 블록 영향     | 제한적     |
 
---

## Bell-LaPadula & Biba 보안 모델 규칙 정리

(출처: Bell & LaPadula, _Secure Computer Systems_, MITRE, 1973 / Biba, _Integrity Considerations for Secure Computer Systems_, MITRE, 1977)

|모델|보안 목적|규칙 이름|규칙 내용|
|---|---|---|---|
|Bell-LaPadula|기밀성(Confidentiality)|Simple Security Property|**No Read Up** → 높은 등급 읽기 금지|
|Bell-LaPadula|기밀성(Confidentiality)|Star Property (*-Property)|**No Write Down** → 낮은 등급 쓰기 금지|
|Bell-LaPadula|기밀성(Confidentiality)|Discretionary Security Property|ACL 등 재량적 접근제어 적용|
|Biba|무결성(Integrity)|Simple Integrity Property|**No Read Down** → 낮은 무결성 읽기 금지|
|Biba|무결성(Integrity)|Star Integrity Property|**No Write Up** → 높은 무결성 쓰기 금지|
|Biba|무결성(Integrity)|Invocation Property|낮은 무결성 주체가 높은 무결성 객체 호출 금지|

---

### 시험 핵심 기억

```
Bell-LaPadula : No Read Up / No Write Down  → 기밀성
Biba          : No Read Down / No Write Up → 무결성
```

## SPN vs Feistel 간략 정리

| 구분      | **SPN (Substitution-Permutation Network)** | **Feistel Network**     |
| ------- | ------------------------------------------ | ----------------------- |
| 구조      | **S-box + P-box 반복**                       | **좌우(L/R) 분할 후 XOR 연산** |
| 데이터 처리  | 블록 전체 처리                                   | 블록을 **L / R 두 부분**으로 나눔 |
| 복호화     | **역연산 필요**                                 | **같은 구조 사용**            |
| 대표 알고리즘 | **AES**                                    | **DES**                 |
