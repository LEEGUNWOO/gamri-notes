# 시구1 네트워크 — OSI 7계층 데일리 암기노트 (Obsidian용)

> [!tip] 사용법(매일 3분 루틴)
> 1) **L1→L7** 한 번 훑기 (각 계층 “키워드/장비/프로토콜 세트”만)
> 2) **함정 박스**만 다시 보기
> 3) 마지막의 **필수 암기 세트**만 체크

---

## 한 줄 암기
- **물데네전세표응** = 물리 → 데이터링크 → 네트워크 → 전송 → 세션 → 표현 → 응용
- **주소 흐름**: **MAC(L2) → IP(L3) → Port(L4)**

---

## L1 물리 (Physical)
**대표 키워드**: 신호, 변조, 라인코딩, 전송매체, 대역폭, SNR, Shannon, (WDM/DWDM), (SONET/SDH), xDSL  
**한 줄 설명**: 비트를 **전기/광/무선 신호**로 바꿔 *실제로* 보내는 계층

- **PDU**: Bit
- **주소**: 없음(주소 개념은 보통 L2부터)
- **대표 장비**: **리피터**, **허브**, **모뎀**, 미디어컨버터(동선↔광), 케이블/커넥터
- **핵심 기술 세트(시험용)**
  - **변조(반송파)**: ASK / FSK / PSK / QAM  
    - ASK=진폭, FSK=주파수, PSK=위상, QAM=진폭+위상(혼합)
  - **다중화**: FDM / TDM / WDM(광) / CDM(CDMA) / OFDM  
    - FDM=주파수 분할, TDM=시간 분할, WDM=파장 분할, OFDM=직교 주파수 분할
  - **라인코딩(베이스밴드)**: NRZ / RZ / Manchester / Differential Manchester
  - **아날로그→디지털 변환**: PCM(표본화→양자화→부호화)
  - **전송기술**: xDSL / DWDM / (SONET/SDH)

> [!note] 통신이론 공식(빈출)
> - **SNR(dB) = 10 · log10(S/N)**
> - **Shannon 용량**: **C = B · log2(1 + S/N)**
>   - B(Hz), C(bps) 단위 실수 조심

> [!warning] L1 빈출 함정(꼭 외우기)
> - **라인코딩(베이스밴드)** ↔ **변조(반송파)** 혼동
> - FDM/TDM/WDM/OFDM을 서로 뒤섞음
> - dB 변환(10log)에서 실수(선형값: 10^(dB/10))

---

## L2 데이터링크 (Data Link)
**대표 키워드**: 프레임, MAC주소, 스위칭, 오류검출(CRC), 매체접근(MAC), VLAN(802.1Q), PPP  
**한 줄 설명**: 같은 링크/LAN 안에서 **프레임을 MAC 기반으로 전달**

- **PDU**: Frame
- **주소**: **MAC 주소**
- **대표 장비**: **브리지**, **L2 스위치**, 무선 AP(출제에선 보통 L2 브리징 성격)
- **핵심 프로토콜/기술 세트**
  - **Ethernet (IEEE 802.3)**: 대표 LAN
  - **MAC(매체 접근 제어)**: CSMA/CD(유선 공유매체) / CSMA/CA(무선)
  - **오류검출/정정**: 패리티(검출 약함) / **CRC(검출 강함)** / **해밍코드(정정)**
  - **VLAN(802.1Q)**, **PPP**, (프레임릴레이: 교재 범위면 L2 성격으로 출제)

> [!note] L2 핵심 요령(장비/개념 판별)
> - “**무슨 정보를 보고 전달하나?**” → MAC 주소를 보면 L2 가능성 큼
> - 스위치는 보통 **소스 MAC 학습 → 목적 MAC 포워딩** 흐름

> [!warning] L2 빈출 함정
> - 스위치 환경도 “항상 CSMA/CD”라고 단정(현실은 전이중/충돌 거의 없음)
> - **CRC는 정정이 아니라 검출 중심**(정정은 해밍 쪽)
> - VLAN을 IP 서브넷과 동일시(연계되지만 개념 다름)

---

## L3 네트워크 (Network)
**대표 키워드**: IP, 라우팅, 서브넷/CIDR, 단편화, ICMP/IGMP, ARP, NAT, Mobile IP  
**한 줄 설명**: **IP 주소 기반**으로 다른 네트워크까지 패킷을 전달(경로 선택)

- **PDU**: Packet
- **주소**: **IP 주소**
- **대표 장비**: **라우터**, **L3 스위치**
- **핵심 프로토콜/기술 세트**
  - **IP(IPv4/IPv6)**
  - **ICMP**(진단/오류: ping, TTL 초과 등)
  - **IGMP**(멀티캐스트 그룹 관리)
  - **ARP/RARP**(IP↔MAC 해석: 링크 내부)
  - **라우팅 프로토콜 분류(비교 빈출)**: RIP(거리벡터) / OSPF(링크상태) / BGP(경로벡터)
  - **주소/경계 기술**: 서브넷·CIDR / **단편화(Fragmentation)** / **NAT(SNAT·DNAT·PAT)** / Mobile IP(범위면)

> [!tip] NAT 방향(시험에 그대로 나옴)
> - **SNAT**: 내부 → 외부 (출발지 변환)
> - **DNAT**: 외부 → 내부 (목적지 변환)
> - **PAT(NAPT)**: 다수 내부를 1개 공인 IP로(포트로 구분, L3+L4 성격)

> [!warning] L3 빈출 함정(가장 많이 틀림)
> - **MTU는 링크 특성(L2)**, 하지만 **단편화는 IP(L3)**에서 처리
> - ARP를 “라우팅”으로 착각(같은 링크에서 해석)
> - SNAT/DNAT 방향을 자주 뒤집음

---

## L4 전송 (Transport)
**대표 키워드**: Port, TCP/UDP, 신뢰성, 흐름제어(Window), 혼잡제어, 재전송, RTT, ARQ  
**한 줄 설명**: 호스트 내부 **프로세스(포트)** 단위로 종단 간 전송을 제어

- **PDU**: TCP=Segment / UDP=Datagram
- **주소**: **Port 번호**
- **대표 장비**: L4 스위치(로드밸런서로 출제), 방화벽(주로 L3~L4 관점)
- **핵심 프로토콜/기술 세트**
  - **TCP / UDP / SCTP(범위면)**
  - **ARQ(재전송) 3종 세트**: Stop-and-Wait / Go-Back-N / Selective Repeat
  - 성능 키워드: RTT, Window, 재전송 타이머, 처리량(근사)

> [!note] TCP 처리량 근사(시험용 감각)
> - **처리량 ≈ Window / RTT**  (단위: Byte/s 또는 bit/s 변환 주의)

> [!warning] L4 빈출 함정
> - **흐름제어(Window)** vs **혼잡제어**를 섞어 말함
> - RTT(왕복) vs 편도 지연을 혼동
> - Byte↔bit(×8), ms↔s(÷1000) 단위 실수

---

## L5 세션 (Session)
**대표 키워드**: 세션 설정/유지/종료, 동기화(체크포인트)  
**한 줄 설명**: 통신의 “대화 단위(세션)”를 관리

- **PDU**: Data
- **주소**: 없음(주로 개념)
- **대표 장비**: 보통 장비로 안 묻고 개념 출제
- **출제 포인트**: “세션 관리 기능이 무엇인가” 수준(비중 낮은 편)

> [!warning] L5 함정
> - TCP 연결관리(핸드셰이크)와 **동일시**하기(시험은 역할 구분을 묻기도 함)

---

## L6 표현 (Presentation)
**대표 키워드**: 인코딩/디코딩, 압축, 암호화, 데이터 표현 변환  
**한 줄 설명**: 데이터 “형식”을 맞추고(표현), **압축/암호화**를 담당

- **PDU**: Data
- **대표 예시**: JPEG/MPEG(압축), TLS/SSL(암호화 성격)
- **출제 포인트**: “암호화/압축이 어느 계층 성격인가”

> [!warning] L6 함정
> - 암호화/압축을 무조건 응용(L7)로만 단정(교재/문제는 표현(L6) 성격으로도 자주 설명)

---

## L7 응용 (Application)
**대표 키워드**: 사용자 서비스, HTTP/DNS/DHCP/메일, 관리(SNMP), IoT(MQTT)  
**한 줄 설명**: 사용자가 직접 쓰는 네트워크 서비스 제공

- **PDU**: Data
- **대표 장비**: 프록시(L7), L7 로드밸런서, WAF(범위면), DPI(페이로드 분석 성격)
- **자주 나오는 서비스/포트(시험 단골)**
  - HTTP 80, HTTPS 443
  - DNS 53(주로 UDP)
  - DHCP 67/68(UDP)
  - FTP 21(제어) / 20(데이터)
  - 텔넷 23(원격) / SSH 22(원격)
  - SMTP(서버 TO 서버 , 클라 -> 서버 ) 25, POP3(서버 -> 클라) 110, IMAP (서버->클라) 143
  - SNMP 161(UDP)
  - MQTT 1883(TCP)

> [!warning] L7 빈출 함정
> - “DNS는 무조건 UDP” 같은 단정(상황 따라 TCP도 가능)
> - FTP 20/21 역할(제어/데이터) 섞기
> - SNMP를 라우팅 프로토콜로 착각

---

## TCP vs UDP (시험용 비교: 이 블록만 외워도 득점)

> [!summary] TCP 한 줄
> **연결지향 + 신뢰성 + 순서보장 + 흐름/혼잡제어(윈도우/재전송/ACK)**

> [!summary] UDP 한 줄
> **비연결 + 저오버헤드 + 순서/신뢰성 보장 없음(필요하면 응용이 보완)**

- **연결**: TCP(연결 설정) / UDP(없음)
- **신뢰성**: TCP(ACK·재전송·SEQ) / UDP(기본 없음)
- **성능 감각**: UDP가 “항상 빠르다”는 단정은 오답(손실/재전송 설계에 좌우)
- **대표 예**
  - TCP: HTTP/HTTPS, FTP, SMTP 등
  - UDP: DNS(일반 질의), DHCP, 스트리밍/VoIP(지연 민감)

## 절대 빼먹지 말아야 할 “세트 암기” 체크리스트

> [!check] L1 세트
> - 변조: **ASK/FSK/PSK/QAM**
> - 다중화: **FDM/TDM/WDM/CDM/OFDM**
> - 통신이론: **SNR(dB), Shannon**
> - 라인코딩: NRZ/Manchester
> - 전송기술: xDSL, DWDM, (SONET/SDH)

> [!check] L2 세트
> - 오류: **패리티/CRC/해밍**
> - MAC: **CSMA/CD vs CSMA/CA**
> - Ethernet(802.3), VLAN(802.1Q), PPP

> [!check] L3 세트
> - IP + **ICMP/IGMP/ARP**
> - 라우팅 비교: **RIP/OSPF/BGP**
> - 주소 계산: **서브넷/CIDR**
> - 경계: **MTU vs 단편화**, NAT(SNAT/DNAT/PAT)

> [!check] L4 세트
> - TCP/UDP(+SCTP)
> - ARQ: **SAW/GBN/SR**
> - 성능: **Window/RTT/재전송**

> [!check] L7 세트(포트)
> - HTTP 80 / HTTPS 443
> - DNS 53 / DHCP 67·68
> - FTP 20·21
> - SMTP 25 / POP3 110 / IMAP 143
> - SNMP 161 / MQTT 1883

## 빠른 판별법(문제에서 계층 찍는 법)
- **MAC 주소**가 보이면 → L2 가능성 큼
- **IP/라우팅/서브넷**이 보이면 → L3
- **Port/TCP·UDP/윈도우/RTT**가 보이면 → L4
- **HTTP/DNS/DHCP/메일**이 보이면 → L7
- **MTU**는 L2 느낌이지만 **단편화는 L3**(함정)
