<a id="evm-진척-측정-방법-정리"></a>
## 📘 EVM 진척 측정 방법 정리

### 1️⃣ 고정비율법 (Fixed Formula Method)

```text
작업 시작 시 일정 비율 인정
작업 완료 시 나머지 비율 인정
중간 진행률은 반영하지 않음
```

#### 📌 대표 유형

- 50-50 Rule

- 20-80 Rule

- 0-100 Rule (시작 0%, 완료 100%)

#### 📌 특징

- 진척 측정이 단순

- 단기 작업에 적합

- 과대 진척 인식 가능성 있음

---

### 2️⃣ 시작-종료법 (0-100 Rule)

```text
시작 시 0%
완료 시 100%
```

- 가장 보수적

- 완료 전까지 EV = 0

---

### 3️⃣ 가중 마일스톤 (Weighted Milestone)

```text
중간 마일스톤마다 가중치 부여
각 단계 완료 시 진척 인정
```

- 대규모·복잡 작업에 적합

- 통제력 우수

---

### 4️⃣ 달성률 방식 (Percent Complete)

```text
실제 작업 진행률을 그대로 반영
```

- 주관 개입 가능성

- 관리 성숙도 필요

---

### 5️⃣ 노력비례법 (Level of Effort, LOE)

```text
시간 경과에 비례해 진척 인정
산출물 없는 지원업무에 사용
```

---

## 🎯 시험용 구분 키워드

```text
정해진 % → 고정비율
중간 단계 여러 개 → 가중 마일스톤
완료 시 100 → 0-100
실제 진행률 → 달성률
지원업무 → LOE
```

## 📘 프로젝트 생애주기 4유형 비교표 (반복 vs 점증 명확화)

| 구분        | 예측형 (Predictive) | 반복적 (Iterative) | 점증적 (Incremental) | 애자일 (Agile)   |
| --------- | ---------------- | --------------- | ----------------- | ------------- |
| 기본 개념     | 처음에 전체 계획 수립     | 같은 기능을 반복 개선    | 기능을 하나씩 추가        | 반복 + 점증 결합    |
| 무엇이 변하는가  | 거의 변하지 않음        | **품질·해결책이 변함**  | **기능 범위가 늘어남**    | 품질 + 기능 모두 유연 |
| 결과물 제공    | 마지막에 전체 제공       | 개선 중인 결과        | 단계별 완성 기능         | 짧은 주기마다 완성 기능 |
| 피드백 역할    | 제한적              | 핵심 요소           | 선택적               | 필수            |
| 핵심 문장     | 계획 중심            | 같은 걸 고친다        | 다른 걸 더한다          | 적응 + 협업       |
| 시험 구분 키워드 | 순차적, 전통적         | 수정, 정제, 개선      | 점진적 제공, 단계적 기능    | 스프린트, 협업      |

## 📘 일정관리 기법 요약 암기본

### 1️⃣ 네트워크 기반 기법

|구분|CPM|PERT|GERT|
|---|---|---|---|
|기간 성격|결정적|확률적|확률적|
|3점 추정|❌|⭕|❌|
|기간 공식|ES/EF 계산|(O+4M+P)/6|고정 공식 없음|
|분산 계산|❌|⭕|❌|

|핵심 키워드|최장경로|베타분포|확률 분기|

---

### 2️⃣  CPM (Critical Path Method)

##### 1️⃣ 기본 계산식

```text
EF = ES + Duration
LS = LF - Duration
Float = LS - ES = LF - EF
```

✔ **Float = 0 → Critical Path**

(PMI, _PMBOK Guide_)

---

###### 2️⃣ Forward Pass (전진 계산)

프로젝트 시작 → 끝 방향 계산

```text
ES = 이전 작업 EF 중 최대값
EF = ES + Duration
```

의미
→ **가장 빨리 시작 / 종료 가능한 시간 계산**

---

###### 3️⃣ Backward Pass (후진 계산)

프로젝트 끝 → 시작 방향 계산

```text
LF = 다음 작업 LS 중 최소값
LS = LF - Duration
```

의미
→ **늦어도 시작 / 종료해야 하는 시간 계산**

---

###### 4️⃣ 약어 정리

| 약어  | Full Name    | 의미             |
| --- | ------------ | -------------- |
| ES  | Early Start  | 가장 빨리 시작 가능 시간 |
| EF  | Early Finish | 가장 빨리 끝나는 시간   |
| LS  | Late Start   | 늦어도 시작해야 하는 시간 |
| LF  | Late Finish  | 늦어도 끝나야 하는 시간  |

---

###### 5️⃣ 여유시간 (Float / Slack)

### Total Float (총 여유시간)

```text
TF = LS - ES
TF = LF - EF
```

의미
→ **작업이 지연되어도 프로젝트 완료일에 영향 없는 시간**

✔ **TF = 0 → Critical Activity**

---

### Free Float (자유 여유시간)

```text
FF = 다음 작업 ES - 현재 작업 EF
```

의미
→ **다음 작업 시작에 영향을 주지 않고 지연 가능한 시간**

---

###### 6️⃣ 여유시간 관계

```text
Free Float ≤ Total Float
```

---

###### 7️⃣ Critical Path

```text
Float = 0 인 경로
```

특징

- 프로젝트 **최장 경로**

- 지연 시 **프로젝트 전체 지연**

- 프로젝트 **최소 완료기간 결정**

---

###### 8️⃣ 시험용 핵심 기억

```text
Forward Pass → ES EF
Backward Pass → LS LF
Float = LS - ES = LF - EF
Float = 0 → Critical Path
```

---

###### 9️⃣ 시험에서 자주 나오는 포인트

|문제 유형|답|
|---|---|
|프로젝트 최소 기간|Critical Path|
|여유시간 계산|LS − ES|
|Forward Pass|ES EF 계산|
|Backward Pass|LS LF 계산|

---

✔ 이 정도가 **감리사 시험 CPM 범위에서 실제로 쓰는 핵심 정리 수준**입니다.
(PMI, _PMBOK Guide_)

---

원하면 제가 **CPM 계산 문제에서 사람들이 가장 많이 틀리는 포인트 3개**도 정리해 줄게요.
감리사 시험에서 **진짜 자주 틀립니다.**

```text
EF = ES + Duration
LS = LF - Duration
Float = LS - ES = LF - EF
```

✔ Float(여유시간) = 0 → Critical Path

|약어|풀네임|의미|계산식|
|---|---|---|---|
|ES|Early Start|가장 빨리 시작할 수 있는 시간|이전 작업 EF 중 최대값|
|EF|Early Finish|가장 빨리 끝날 수 있는 시간|ES + 기간|
|LS|Late Start|늦어도 시작해야 하는 시간|LF - 기간|
|LF|Late Finish|늦어도 끝나야 하는 시간|다음 작업 LS 중 최소값|

### 3️⃣ PERT 핵심 계산식

| 공식 종류           | 계산식              | 특징                                                     |
| --------------- | ---------------- | ------------------------------------------------------ |
| **삼각분포**        | ((O + M + P)/3)  | 단순평균, 모든 값에 동일 가중치, 빠르고 간단함 >> μ                       |
| **PERT (베타분포)** | ((O + 4M + P)/6) | M에 높은 가중치, 현실적 예측에 유리  >> μ                            |
| **표준편차** (PERT) | ((P - O)/6)      | 예측의 불확실성 정도 측정 >> σ                                    |
| 분산 (Variance)   | ((P - O)/6)²     | 분산은 최소값(O)과 최대값(P)을 사용하여 작업 시간의 **변동성**을 측정하는 데 사용됩니다. |
| Z스코어            | Z = (X-μ)/σ      | 3점기법의 계산 평균 =  μ   ,표준편차 = σ   X = 관측값                 |
|                 |                  |                                                        |

✔ O 최적시간 M 가능성있는시간 P 비관적시간
✔ 베타분포 가정
✔ 3점 추정
✔ Z=(값−평균)/표준편차
#### 🔥 **정규분포에서 표준편차에 따른 확률 값**

| **시그마 (σ)**   | **범위** | **확률 값 (%)**  |
| ------------- | ------ | ------------- |
| **1시그마 (1σ)** | μ ± 1σ | **68.23%**    |
| **2시그마 (2σ)** | μ ± 2σ | **95.5%**     |
| **3시그마 (3σ)** | μ ± 3σ | **99.7%**     |
| **4시그마 (4σ)** | μ ± 4σ | **99.9937%**  |
| **5시그마 (5σ)** | μ ± 5σ | **99.99994%** |
|**6시그마 (6σ)**|μ ± 6σ|**99.9999998%**|

### 4️⃣ ==일정== 단축 기법

| 기법            | 핵심 개념       |
| ------------- | ----------- |
| Crashing      | 비용 ↑ → 기간 ↓ |
| Fast Tracking | 순차 작업 병렬 수행 |

| 구분    | Crashing              | Fast Tracking             |
| ----- | --------------------- | ------------------------- |
| 개념    | **추가 자원을 투입하여 일정 단축** | **순차 작업을 병렬로 수행하여 일정 단축** |

| 비용    | **증가**                | 증가하지 않을 수도 있음             |
| 위험    | 비교적 낮음                | **재작업 위험 증가**             |
| 적용 대상 | Critical Path 작업      | 순차 의존성이 있는 작업             |
| 핵심 특징 | **돈으로 시간 단축**         | **동시 수행으로 시간 단축**         |

### 5️⃣ 자원 조정 기법

| 기법                       | 특징                    |
| ------------------------ | --------------------- |
| Resource Leveling 자원평준화  | 자원 초과 해결, 일정 늘어날 수 있음 |
| Resource Smoothing 자원평활화 | 일정 유지, 자원만 조정         |
#### 1️) Resource Leveling (자원 평준화)

| 구분    | 내용                                              |
| ----- | ----------------------------------------------- |
| 정의    | **자원(Resource) 제약을 기준으로 일정(Schedule)을 조정하는 기법** |
| 목적    | 특정 시점에 **자원이 부족하거나 과다 배치된 문제 해결**               |
| 일정 변경 | **가능 (프로젝트 종료일이 늘어날 수 있음)**                     |
| 적용 대상 | 자원이 절대적으로 부족할 때                                 |
| 특징    | Critical Path(주경로)도 변경될 수 있음                    |
#### 1️) Resource Smoothing (자원 평활화)

| 구분    | 내용                                       |
| ----- | ---------------------------------------- |
| 정의    | **일정 여유(Float) 범위 내에서 자원을 균등하게 배치하는 기법** |
| 목적    | 자원 사용량의 **변동을 완화**                       |
| 일정 변경 | **불가 (프로젝트 종료일 유지)**                     |
| 적용 대상 | 일정 여유가 존재할 때                             |
| 특징    | Critical Path는 **절대 변경되지 않음**            |

### 1️⃣ 기본 개념 (3개만 기억)

|항목|의미|한 줄 이해|
|---|---|---|
|PV|계획가치|원래 이만큼 해야 함|
|EV|획득가치|실제로 번 성과|
|AC|실제비용|실제로 쓴 돈|

### 2️⃣ EVM 전체 공식

| 항목  | 공식        | 기준  | 해석        |
| --- | --------- | --- | --------- |
| SV  | EV - PV   | 0   | +면 일정 빠름  |
| CV  | EV - AC   | 0   | +면 비용 절감  |
| SPI | EV / PV   | 1   | 1↑ 일정 양호  |
| CPI | EV / AC   | 1   | 1↑ 비용 양호  |
| EAC | BAC / CPI | -   | 완료 예상 총비용 |
| ETC | EAC - AC  | -   | 남은 필요 비용  |

<a id="eac-공식-4가지-시험-핵심"></a>
## EAC 공식 4가지 (시험 핵심)

| 상황                               | 공식                                      | 의미                    |
| -------------------------------- | --------------------------------------- | --------------------- |
| **현재 비용 성과(CPI)가 계속 유지된다고 가정**⭐⭐ | **EAC = BAC / CPI**                     | 가장 자주 출제되는 공식         |
| **앞으로는 계획대로 진행된다고 가정**⭐⭐         | **EAC = AC + (BAC − EV)**               | 과거 문제 무시, 남은 일은 정상 진행 |
| **비용과 일정 영향 모두 반영**⭐             | **EAC = AC + (BAC − EV) / (CPI × SPI)** | 일정 영향까지 고려            |
| **새로운 재추정 수행**                   | **EAC = AC + ETC**                      | ETC를 새로 계산            |

### 5️⃣값이 안 주어지면 직접 계산

|AC & CV|**EV = CV + AC**|

|---|---|
|BAC & 계획%|**PV = BAC × 계획%**|
|EV & SPI|**PV = EV / SPI**|
|EV & SV|**PV = EV - SV**|

|주어진 값|AC 공식|
|---|---|
|(직접 제시)|**AC = 실제 사용비용**|

|EV & CV|**AC = EV - CV**|

<a id="상황적-리더십-정확-구조"></a>
## 📘 상황적 리더십 정확 구조

성숙도 = 능력 + 의지

성숙도 낮음 → 능력도 없고 의지도 낮음
👉 **지시형 (Telling)**
= 구체적으로 “이렇게 해” 알려줘야 함

성숙도 조금 상승 → 능력 부족하지만 의지는 있음
👉 **설득형 (Selling / Coaching)**
= 가르치면서 왜 해야 하는지 설명

성숙도 더 상승 → 능력 있음, 자신감 부족
👉 **참여형 (Participating / Supporting)**
= 의사결정에 참여시켜 동기 강화

성숙도 높음 → 능력도 의지도 충분
👉 **위임형 (Delegating)**
= 맡기고 간섭 최소

✅ **지시형(Directing/Telling)** → **설득형(Coaching/Selling)** → **참여형(Supporting/Participating)** → **위임형(Delegating)**

## 📘 인도측정(Delivery Measurements)

> “얼마나 빠르고 안정적으로 가치를 전달하고 있는가?”를 보는 지표

| 구분    | 지표                      | 의미                  | 핵심 포인트      |

| 시간 기반 | **주기 시간 (Cycle Time)**  | 작업 시작 → 완료까지 걸린 시간  | 짧을수록 효율 ↑   |
| 시간 기반 | **리드 타임 (Lead Time)**   | 요청 접수 → 완료까지 전체 시간  | 고객 관점 시간    |

| 흐름 기반 | **배치 크기 (Batch Size)**  | 한 번에 처리하는 작업 묶음 수   | 작을수록 빠른 피드백 |
| 속도 기반 | **속도 (Velocity)**       | 스프린트당 완료한 스토리 포인트 합 | 예측 가능성 판단   |
| 시각화   | **번다운 차트**              | 남은 작업량 감소 추이        | 일정 예측       |
| 시각화   | **번업 차트**               | 완료된 작업량 증가 추이       | 범위 변화 확인    |
| 시각화   | **누적 흐름도(CFD)**         | 작업 상태별 흐름 시각화       | 병목 구간 확인    |

## 📘 연동 기획 (Rolling Wave Planning)

### 🔥 정의

> 단기 작업은 상세 계획,
> 장기 작업은 개략 계획 후 **점진적으로 구체화**하는 기법

### 📌 적용 상황

- 프로젝트 초기에 모든 세부사항을 알 수 없음

- 불확실성이 높음

- 점진적 상세화 필요

### 📊 특징

|구분|내용|
|---|---|
|단기 계획|상세하게 수립|
|장기 계획|개략적으로 수립|
|시간 경과|점차 구체화|
|적용 영역|일정관리 (계획 수립)|

### 🎯 시험 포인트

- 일정 계획 기법

- “가까운 것은 상세, 먼 것은 거칠게”

- 애자일/적응형 환경과도 연결 가능

## 원가관리 분석

### PERT 핵심 계산식

| 공식 종류           | 계산식              | 특징                                                     |
| --------------- | ---------------- | ------------------------------------------------------ |
| ==**삼각분포**==        | ==((O + M + P)/3)==  | ==단순평균, 모든 값에 동일 가중치, 빠르고 간단함 >> μ==                       |
| ==**PERT (베타분포)**== | ==((O + 4M + P)/6)== | ==M에 높은 가중치, 현실적 예측에 유리  >> μ==                            |
| ==**표준편차** (PERT)== | ==((P - O)/6)==      | ==예측의 불확실성 정도 측정 >> σ==                                    |
| 분산 (Variance)   | ((P - O)/6)²     | 분산은 최소값(O)과 최대값(P)을 사용하여 작업 시간의 **변동성**을 측정하는 데 사용됩니다. |
| Z스코어            | Z = (X-μ)/σ      | 3점기법의 계산 평균 =  μ   ,표준편차 = σ   X = 관측값                 |
|                 |                  |                                                        |

✔ O 최적시간 M 가능성있는시간 P 비관적시간
✔ 베타분포 가정
✔ 3점 추정
✔ Z=(값−평균)/표준편차
#### 🔥 **정규분포에서 표준편차에 따른 확률 값**

| ------------- | ------ | --------------- |

| ==**2시그마 (2σ)**== | ==μ ± 2σ== | ==**95.5%**==       |
| ==**3시그마 (3σ)**== | ==μ ± 3σ== | ==**99.7%**==       |

## EVM
#### 획득일정 (Earned Schedule, ES) 최소 정리

|항목|내용|
|---|---|
|정의|**EV가 계획상 달성되어야 하는 시점(시간)**|
|목적|EVM의 **일정 성과를 시간 단위로 분석**|
|필요 데이터|**PV(계획가치) 누적 일정**|
|핵심 특징|**비용이 아니라 시간 기반 일정관리**|

#### 관련 지표

| 항목  | 의미                      |
| --- | ----------------------- |
| EV  | 실제 완료된 작업의 예산 가치        |
| PV  | 계획된 작업의 예산 가치           |
| ES  | **EV가 계획 일정에서 달성되는 시점** |
| AT  | 현재 실제 시간                |

| 지표     | 공식              | 의미                        |
| ------ | --------------- | ------------------------- |
| ES     | Earned Schedule | EV에 해당하는 계획 일정 시점         |
| SV(t)  | ES − AT         | 시간 기준 일정 편차               |
| SPI(t) | ES / AT         | 시간 기준 일정 성과지수 (비용기준과 동일함) |
pv rate = bac / 총기간

ES = EV/ (총기간/ pv rate)

---

### 투자평가 기법 정리

|기법|영어|계산 방법|의미|판단 기준|
|---|---|---|---|---|
|PV|Present Value|미래 현금흐름 ÷ (1+r)^t|미래 돈의 현재 가치|값이 클수록 좋음|
|NPV|Net Present Value|PV 합 − 초기투자비|순현재가치|**NPV > 0 투자 가능**|
|EPV|Equivalent Present Value|반복사업을 동일기간 기준 PV 환산|반복 투자 비교|값이 큰 사업 선택|
|회수기간법|Payback Period|투자금 회수까지 걸리는 기간|투자금 회수 속도|짧을수록 좋음|
|IRR|Internal Rate of Return|NPV=0 되는 할인율|투자 수익률|**IRR > 요구수익률**|
|BCR|Benefit Cost Ratio|편익 PV ÷ 비용 PV|비용 대비 편익 비율|**BCR > 1**|

<a id="1-미래-돈을-지금-가치로-바꾼다"></a>
## 1️⃣ 미래 돈을 지금 가치로 바꾼다

### PV (현재가치, Present Value)

건우 생각

> "3년 뒤 50만원은 지금 50만원이랑 같지 않지?"

은행 금리 **10%** 기준으로 계산합니다.

|시점|실제 받는 돈|현재 가치|
|---|---|---|
|1년 후|50|45|
|2년 후|50|41|
|3년 후|50|37|

현재 가치 합

[
45 + 41 + 37 = 123
]

즉

> 이 사업이 만들어내는 **현재 가치 = 123만원**

---

<a id="2-투자보다-이익인지-확인"></a>
## 2️⃣ 투자보다 이익인지 확인

### NPV (순현재가치, Net Present Value)

건우 생각

> "투자 100인데 현재 가치가 얼마지?"

[
NPV = 123 - 100 = 23
]

의미

> 이 사업 하면 **23만원 가치가 생긴다**

판단

**NPV > 0 → 투자**

---

<a id="3-언제-투자금-회수되는지-확인"></a>
## 3️⃣ 언제 투자금 회수되는지 확인

### 투자회수기간 (Payback Period)

건우 생각

> "돈 언제 회수하지?"

|연도|누적수익|
|---|---|
|1년|50|
|2년|100|
|3년|150|

투자 회수

**2년**

---

<a id="4-이-사업-수익률이-몇-인지"></a>
## 4️⃣ 이 사업 수익률이 몇 %인지

### IRR (내부수익률, Internal Rate of Return)

건우 생각

> "이 사업은 연 몇 %짜리 사업이지?"

계산하면

IRR ≈ **23%**

판단

은행 금리

**10%**

즉

> 은행보다 훨씬 높은 사업

---

<a id="5-편익이-비용보다-큰지"></a>
## 5️⃣ 편익이 비용보다 큰지

### BCR (편익비용비, Benefit Cost Ratio)

건우 생각

> "1만원 투자하면 얼마 가치 생기지?"

[
BCR = 123 / 100 = 1.23
]

의미

> **1만원 투자 → 1.23만원 가치**

판단

**BCR > 1 → 투자**

---

<a id="6-다른-기간-사업과-비교"></a>
## 6️⃣ 다른 기간 사업과 비교

### EPV (연간균등현재가치, Equivalent Present Value)

건우가 고민합니다.

- 커피머신 A → 3년

- 커피머신 B → 5년

기간이 다르므로

> **연간 가치 기준으로 환산해서 비교**

이때 사용하는 것이 **EPV**

---

<a id="전체-흐름-직관적-순서"></a>
## 전체 흐름 (직관적 순서)

건우가 투자 판단하는 순서

1️⃣ 미래 돈을 현재 가치로 환산 → **PV**
2️⃣ 투자하고 남는 가치 확인 → **NPV**
3️⃣ 언제 투자금 회수 → **투자회수기간**
4️⃣ 사업 수익률 확인 → **IRR**
5️⃣ 편익/비용 비율 확인 → **BCR**
6️⃣ 기간 다른 사업 비교 → **EPV**

## 📘자원 조정 기법

#### 1️) Resource Leveling (자원 평준화)

| 구분    | 내용                                              |
| ----- | ----------------------------------------------- |
| 정의    | **자원(Resource) 제약을 기준으로 일정(Schedule)을 조정하는 기법** |
| 목적    | 특정 시점에 **자원이 부족하거나 과다 배치된 문제 해결**               |
| 일정 변경 | **가능 (프로젝트 종료일이 늘어날 수 있음)**                     |
| 적용 대상 | 자원이 절대적으로 부족할 때                                 |
| 특징    | Critical Path(주경로)도 변경될 수 있음                    |
#### 1️) Resource Smoothing (자원 평활화)

| 구분    | 내용                                       |
| ----- | ---------------------------------------- |
| 정의    | **일정 여유(Float) 범위 내에서 자원을 균등하게 배치하는 기법** |
| 목적    | 자원 사용량의 **변동을 완화**                       |
| 일정 변경 | **불가 (프로젝트 종료일 유지)**                     |
| 적용 대상 | 일정 여유가 존재할 때                             |
| 특징    | Critical Path는 **절대 변경되지 않음**            |
### 📘 ==일정== 단축 기법

| 기법            | 핵심 개념       |
| ------------- | ----------- |
| Crashing      | 비용 ↑ → 기간 ↓ |
| Fast Tracking | 순차 작업 병렬 수행 |

| 구분    | Crashing                  | Fast Tracking             |
| ----- | ------------------------- | ------------------------- |
| 개념    | **추가 자원을 투입하여 일정 ==압축==** | **순차 작업을 병렬로 수행하여 일정 단축** |
| 방법    | 인력·장비·비용 증가               | 작업을 겹쳐서 수행                |
| 비용    | **증가**                    | 증가하지 않을 수도 있음             |
| 위험    | 비교적 낮음                    | **재작업 위험 증가**             |
| 적용 대상 | Critical Path 작업          | 순차 의존성이 있는 작업             |
| 핵심 특징 | **돈으로 시간 ==압축==**         | **동시 수행으로 시간 단축**         |

<a id="eac-공식-3가지-시험-핵심"></a>
## 📘 EAC 공식 3가지 (시험 핵심)

| 상황                               | 공식                                      | 의미                    |
| -------------------------------- | --------------------------------------- | --------------------- |
| **현재 비용 성과(CPI)가 계속 유지된다고 가정**⭐⭐ | **EAC = BAC / CPI**                     | 가장 자주 출제되는 공식         |
| **앞으로는 계획대로 진행된다고 가정**⭐⭐         | **EAC = AC + (BAC − EV)**               | 과거 문제 무시, 남은 일은 정상 진행 |
| **비용과 일정 영향 모두 반영**⭐             | **EAC = AC + (BAC − EV) / (CPI × SPI)** | 일정 영향까지 고려            |
| **완료시차이**                        | **VAC = BAC - EAC**                     | ETC를 새로 계산            |
| **남은 필요 비용**                     | **ETC** =**EAC - AC**                   | 남은 필요 비용              |
| **완료성과지수**                       | **TCPI = (BAC-EV) / (BAC-AC) **         |                       |

## 📘  버지니아 변화 모델 (Virginia Satir Change Model)

|단계|영문|핵심 특징|성과 수준|
|---|---|---|---|
|1. 현재 상태|Late Status Quo|익숙하고 예측 가능, 안정적. 규칙과 관행이 지배|안정적|
|2. 외부 요소|Foreign Element|변화 촉발 사건 발생. 저항·부정·회피 시작. 성과 하락 시작|저하 시작|
|3. 혼돈|Chaos|기존 방식 붕괴, 방향 상실. 불안·혼란 극대화. 역설적으로 창의적 아이디어 폭발|최저|
|4. 변환 아이디어|Transforming Idea|외부 요소를 이해하는 돌파구 발견. 흥분과 희망 생김|반등 시작|
|5. 실무 및 통합|Practice & Integration|새 방식 적용·연습. 성과 빠르게 회복|회복·상승|
|6. 새로운 상태|New Status Quo|새 방식이 일상화. 성과 기존 이상|기존 이상|

## 📘 리스크 유형 (가변성 vs 모호성)

<a id="흐름속도시간대기-개념-정리"></a>
## 📘 흐름/속도/시간/대기 개념 정리

| 개념     | 영문               | 정의              | 공식/관계       | 핵심 포인트         |
| ------ | ---------------- | --------------- | ----------- | -------------- |
| 작업 진척  | Progress         | 전체 작업 대비 완료된 비율 | 완료량 / 전체량   | % 개념           |
| 리드타임   | Lead Time        | 요청 ~ 완료까지 전체 시간 | 처리시간 + 대기시간 | 고객 기준          |
| 주기시간   | Cycle Time       | 실제 작업 처리 시간     | 작업 시작 ~ 완료  | 작업 기준          |
| 대기시간   | Waiting Time     | 작업이 대기하는 시간     | 리드타임 - 주기시간 | 병목 발생 지점       |
| 대기열 크기 | Queue Size       | 처리 대기 중 작업 수    | WIP 일부      | 병목 지표          |
| 속도     | Velocity         | 일정 기간 처리한 작업량   | 스프린트 완료량    | 애자일            |
| 처리시간   | Processing Time  | 실제 작업 수행 시간     | 주기시간과 동일 개념 | Value-added 시간 |
| 처리량    | Throughput       | 단위시간당 완료량       | 완료건수 / 시간   | 시스템 생산성        |
| WIP    | Work In Progress | 진행 중 작업 수       | 시스템 내 작업 수  | 많을수록 지연 증가     |
